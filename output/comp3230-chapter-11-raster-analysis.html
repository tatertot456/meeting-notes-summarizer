Map Algebra & Raster Queries

▶

Rasters are matrices of numbers georeferenced to earth locations. Because they share the same spatial grid, rasters with the same extent and resolution can be analyzed using cell-by-cell operations called map algebra. A new raster is created to store the result.

// Map algebra examples:
precip_cm = precip_inches * 2.54      // multiply raster by constant
combined  = raster_A + raster_B       // add two rasters cell by cell
diff      = raster_A - raster_B       // find difference between rasters

// Raster query (Boolean raster):
result = DEM > 1400                   // 1 where elevation > 1400m, 0 elsewhere

// Boolean operators in map algebra:
AND result  = (condition_A == 1) * (condition_B == 1)  // same as multiply
OR  result  = (condition_A == 1) + (condition_B == 1) > 0
NOT result  = 1 - condition_A

A Boolean raster contains only 0 (False) and 1 (True). Raster queries produce Boolean rasters. These are the building blocks of raster overlay analysis.

Always store rasters in a PROJECTED coordinate system before analysis. Distance and area calculations are meaningless in geographic coordinates (degrees). Raster analysis is performed using the Raster Calculator tool (requires Spatial Analyst extension — additional license required).

Boolean operator equivalents to vector operations:

  • AND → vector Intersect (areas meeting ALL conditions)
  • OR → vector Union (areas meeting ANY condition)
  • NOT → vector Erase (areas NOT meeting a condition)
  • XOR → no simple vector equivalent (Intersect then Erase)
  • AND is equivalent to MULTIPLYING two Boolean rasters (0×0=0, 1×0=0, 1×1=1)

Boolean Overlay & Additive Overlay

▶

Raster overlay uses Boolean rasters and map algebra to find areas where multiple conditions hold simultaneously.

// Snail habitat example:
ElevRange    = (DEM > 800) * (DEM < 1200)   // suitable elevation
LimestoneBoo = (geology == "limestone")      // limestone areas
DenseTrees   = (canopy > 40)                 // dense canopy

// AND overlay (multiply = all conditions must be met):
SnailHabitat = ElevRange * LimestoneBoo * DenseTrees
// Result: 1 only where ALL three conditions are 1

// Additive overlay (ranking instead of strict yes/no):
HabitatRank = ElevRange + LimestoneBoo + DenseTrees
// Result: 0 = no conditions, 1 = one condition, 2 = two, 3 = ideal

Boolean Overlay

Strict yes/no result. AND operator (multiply). Output = 1 where ALL conditions met, 0 elsewhere. Simple but treats all conditions as equal and abrupt boundaries as meaningful.

Additive Boolean Overlay

Sum the Boolean rasters instead of multiplying. Output = count of how many conditions are present. Provides ranking (0, 1, 2, 3...) instead of binary yes/no. Better for showing gradations.

Boolean overlay drawback: the boundary is completely abrupt — elevation 1500m is unsuitable, 1501m is ideal. This rarely reflects reality. Use WEIGHTED OVERLAY for more realistic, graduated suitability analysis.

Weighted Overlay

▶

Weighted overlay ranks conditions on a graduated scale (not just 0/1) and assigns weights based on relative importance — producing a more realistic suitability index.

// General formula:
Output = W1 * Rank1 + W2 * Rank2 + ... + Wk * Rankk
// Where: W1 + W2 + ... + Wk = 1.0 (weights must sum to 1)
// Ranks: 1 to N (1 = worst, N = best)

// Landfill suitability example (K=4 conditions, N=3 ranks):
// Condition ranks (1=worst, 3=best):
//   Slope:      high=1, medium=2, low=3
//   Infiltration: high=1, medium=2, low=3  (most important!)
//   Road distance: far=1, medium=2, close=3
//   Stream distance: close=1, medium=2, far=3

// Weights (must sum to 1.0):
//   Slope: 0.10 (least important)
//   Infiltration: 0.40 (most important — prevents groundwater contamination)
//   Road distance: 0.25
//   Stream distance: 0.25

LSI = 0.10*slope_rank + 0.40*infilt_rank + 0.25*road_rank + 0.25*stream_rank
// Output range: 1.0 to 3.0 (Landfill Suitability Index)

Each condition raster must be reclassified into ranked values (1 to N) before weighted overlay. The reclassification maps original data values to suitability ranks — this is where gradations can be specified (e.g., 0-1000m=1, 1000-1500m=2, >1500m=3).

Distance Functions

▶

Euclidean Distance

Creates a raster where each cell value = straight-line distance to the nearest input feature (e.g., nearest stream). Units match the raster coordinate system (meters/feet). Used for: buffers, proximity analysis, suitability.

Euclidean Direction

Optional output from the distance tool. Each cell value = direction (0–360°) you would travel to reach the nearest feature. N=0°/360°, E=90°, S=180°, W=270°.

Raster Buffers

Apply a logical query to a distance raster: [streamdistance] < 500. Cells within 500m = 1, outside = 0. Creates a Boolean raster buffer usable in overlay analysis.

Least Cost Path

Finds the easiest route between two points using a COST raster (e.g., slope). Combines cost with distance to determine the path of least resistance — not necessarily straight-line shortest.

Neighborhood Statistics & Zonal Functions

▶

Neighborhood functions calculate statistics for a window of cells around each target cell and assign the result to that cell.

Focal (Moving Window)

Window moves ONE CELL at a time — neighborhoods overlap. Each cell is evaluated in multiple windows. Produces a smoother result. Common use: majority filter to simplify complex rasters before converting to vector.

Block

Window moves to a COMPLETELY NEW set of cells — no overlap. Each cell evaluated in exactly one block. Produces a blocky, generalized result.

Common neighborhood statistics: mean, majority, minority, sum, max, min, range, std dev. Window shapes: square (most common), circle, wedge, annulus.

Focal majority use case — simplifying a complex land cover raster before converting to polygons: run 2 passes of focal majority with a 4×4 window → eliminates small isolated patches → cleaner polygon output.

Zonal Functions — neighborhood defined by geographic zones (features or raster regions with unique values):

  • Zones = areas with a unique value (e.g., each watershed has a unique ID)
  • Zones need NOT be contiguous — all cells with value 3 are zone 3 regardless of location
  • Zonal Statistics calculates statistics for each zone applied to another raster
  • Example: average slope for each watershed — watersheds are zones, slope raster is input
  • Use a unique ID field (like Feature ID) to ensure each zone is distinct

Reclassify & Density Functions

▶

Reclassify — changes raster cell values to a new set of values based on specified rules:

  • Convert continuous elevation values to suitability ranks (1/2/3)
  • Convert continuous values to Boolean (suitable = 1, unsuitable = 0) for overlay
  • Generalize a dataset by collapsing many categories into fewer
  • Required step before weighted overlay — all input rasters must be on the same scale

Simple Density

Counts features within a circle of specified radius, divides by circle area. Each feature can be weighted by an attribute (e.g., population). Assumes all weight is concentrated at the point location.

Kernel Density

Spreads each feature's value over an area using a Gaussian (bell curve) distribution before summing. More realistic for phenomena like population (not truly concentrated at one point). Produces smoother surfaces.

Environment Settings for Raster Analysis

▶

Setting environment settings BEFORE analysis ensures consistent, correct output. Set these in the Environments tab of each tool or globally via the Analysis ribbon → Environments button.

SettingControlsBest Practice
WorkspaceWhere permanent output is saved (defaults to project geodatabase)Create a "junk" geodatabase for temporary intermediate rasters — easier cleanup. Scratch workspace holds truly temporary rasters during tool execution.
Output Coordinate SystemCS of output rastersLEAVE BLANK for raster analysis. Changing CS during processing requires resampling, which degrades data quality. Project rasters BEFORE analysis, not during.
Processing ExtentOutput raster extent when inputs don't matchSet to a specific input raster/layer to force all outputs to match it. Default (intersection) is safe but may miss data. Avoids unnecessary resampling.
Cell SizeOutput cell size when inputs have different resolutionsExplicitly set to match your input rasters. Default (maximum/coarsest cell size) is appropriate since the largest cell size limits overall accuracy. Avoid automatic resampling.
MaskClips all outputs to an area of interestUse a watershed polygon or study area raster to limit outputs to the region of interest. Cells outside mask = NoData. More efficient than clipping after analysis.

AUTOMATIC RESAMPLING WARNING: If input rasters have mismatched cell sizes or extents, ArcGIS resamples them automatically — silently degrading your data without warning you. Always use consistent cell sizes and extents. Set environment settings explicitly to prevent invisible data quality loss. The analysis result may appear correct but be less accurate than it should be.
