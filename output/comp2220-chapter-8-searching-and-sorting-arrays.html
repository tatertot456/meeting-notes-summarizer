Linear Search

▶

Also called sequential search. Starting at index 0, examines each element one by one until the target value is found or the array ends. Works on any array — sorted or unsorted.

int linearSearch(int arr[], int size, int value)
{
    int index = 0;
    int position = -1;    // -1 means "not found"
    bool found = false;

    while (index < size && !found)
    {
        if (arr[index] == value)
        {
            found = true;
            position = index;
        }
        index++;
    }
    return position;   // returns index if found, -1 if not
}

Returning -1 for "not found" is a common convention — -1 is never a valid array index, so it unambiguously signals failure. The caller checks: if (result == -1) ... else use result as the index.

Benefits

Simple to understand and implement. Array can be in any order — no sorting required.

Disadvantages

Inefficient for large arrays. On average examines N/2 elements when found, N elements when not found. O(n) complexity.

Binary Search

▶

Requires a sorted array. Repeatedly halves the search space by comparing the target to the middle element. Dramatically faster than linear search on large arrays.

Algorithm:

  1. Set first = 0, last = size - 1
  2. Calculate middle = (first + last) / 2
  3. If array[middle] == value → found!
  4. If array[middle] > value → search lower half (last = middle - 1)
  5. If array[middle] < value → search upper half (first = middle + 1)
  6. Repeat until found or first > last (not in array)
int binarySearch(int array[], int size, int value)
{
    int first = 0,
        last = size - 1,
        middle,
        position = -1;
    bool found = false;

    while (!found && first <= last)
    {
        middle = (first + last) / 2;

        if (array[middle] == value)
        {
            found = true;
            position = middle;
        }
        else if (array[middle] > value)
            last = middle - 1;    // search lower half
        else
            first = middle + 1;   // search upper half
    }
    return position;
}

Benefits

Extremely efficient. For an array of N elements, performs at most log₂N comparisons. A 1 million element array needs at most 20 comparisons. O(log n) complexity.

Disadvantages

Array must be sorted first. If the array changes frequently, keeping it sorted adds overhead. More complex to implement than linear search.

Bubble Sort

▶

Repeatedly compares adjacent pairs and swaps them if out of order. Larger values "bubble up" to the end of the array with each pass. Stops when a full pass completes with no swaps.

Example trace on {17, 23, 5, 11}:

  • Pass 1: 17,23 ok → swap 23,5 → {17,5,23,11} → swap 23,11 → {17,5,11,23}
  • Pass 2: swap 17,5 → {5,17,11,23} → swap 17,11 → {5,11,17,23} → 17,23 ok
  • Pass 3: 5,11 ok → 11,17 ok → 17,23 ok → no swaps → done!
void bubbleSort(int array[], int size)
{
    bool swap;
    int temp;

    do
    {
        swap = false;
        for (int count = 0; count < (size - 1); count++)
        {
            if (array[count] > array[count + 1])
            {
                temp = array[count];
                array[count] = array[count + 1];
                array[count + 1] = temp;
                swap = true;
            }
        }
    } while (swap);
}

Benefits

Easy to understand and implement. Good for nearly-sorted arrays where few swaps are needed.

Disadvantages

Very inefficient for large arrays. O(n²) complexity — doubling the data quadruples the work. One of the slowest sorting algorithms.

Selection Sort

▶

Finds the smallest element and swaps it into position 0. Then finds the next smallest and swaps it into position 1. Continues until the entire array is sorted. Fewer swaps than bubble sort.

Example trace on {11, 2, 29, 3, 17}:

  • Find smallest (2) → swap with position 0 → {2, 11, 29, 3, 17}
  • Find next smallest (3) → swap with position 1 → {2, 3, 29, 11, 17}
  • Find next smallest (11) → swap with position 2 → {2, 3, 11, 29, 17}
  • Find next smallest (17) → swap with position 3 → {2, 3, 11, 17, 29}
void selectionSort(int array[], int size)
{
    int minIndex, minValue;

    for (int start = 0; start < (size - 1); start++)
    {
        minIndex = start;
        minValue = array[start];

        for (int index = start + 1; index < size; index++)
        {
            if (array[index] < minValue)
            {
                minValue = array[index];
                minIndex = index;
            }
        }
        // Swap the minimum with the start position
        array[minIndex] = array[start];
        array[start] = minValue;
    }
}

Benefits

More efficient than bubble sort — performs exactly N-1 swaps regardless of initial order. Fewer writes to memory.

Disadvantages

Still O(n²) comparisons. Slightly harder to understand than bubble sort. Not as fast as advanced sorts (quicksort, mergesort) on large datasets.

Asymptotic Analysis (Big-O)

▶

Asymptotic analysis quantifies algorithm efficiency by describing how runtime grows as data size grows. We throw a massive amount of data at the problem and measure the impact.

NotationNameImpact of doubling dataExample
O(log n)LogarithmicNegligible — adds one stepBinary search
O(n)LinearRuntime doublesLinear search
O(n²)PolynomialRuntime quadruplesBubble sort, selection sort
O(2ⁿ)ExponentialRuntime squaresBrute-force combinatorics

O(log n) is incredibly powerful — binary search on 1 billion sorted elements takes at most 30 comparisons. O(n²) becomes painfully slow fast — sorting 1 million elements with bubble sort requires ~500 billion comparisons. O(2ⁿ) is effectively unusable beyond tiny inputs. Always choose the best complexity available for the problem.
