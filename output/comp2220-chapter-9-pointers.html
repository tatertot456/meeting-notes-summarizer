Memory Addresses

▶

Every variable in a program is stored at a unique memory address. Use the address operator & to get the address of any variable.

int num = -99;
cout << &num;   // prints the memory address in hexadecimal
                // e.g., 0x7ffee4bc

You've already worked with addresses without knowing it. When you pass an array to a function, C++ automatically passes the array's starting address. When you use a reference parameter (&), C++ automatically stores the address of the argument. Pointers make this mechanism explicit and give you direct control over it.

Pointer Variables

▶

A pointer variable holds the address of another variable. The * in the definition means "this is a pointer to" — it does NOT dereference here, it's part of the type declaration.

// Definition — all three mean the same thing
int *intptr;       // preferred style
int * intptr;
int* intptr;

// Assign an address
int num = 42;
intptr = &num;     // intptr now "points to" num

// Always initialize pointers — use nullptr if no address yet
int *ptr = nullptr;    // C++11: nullptr means "points to nothing"

// Test for invalid/null pointer
if (!ptr)
    cout << "Pointer is null\n";
if (ptr == nullptr)
    cout << "Also null\n";

An uninitialized pointer contains garbage — a random address that could point anywhere in memory. Using an uninitialized pointer causes undefined behavior, often crashing the program. ALWAYS initialize pointers to nullptr or a valid address before using them.

Rules: pointer type must match the type it points to — you cannot store the address of a double in an int pointer. Pointers to different types are incompatible.

The Indirection (Dereference) Operator

▶

The * operator, when used with a pointer variable in an expression (not a definition), dereferences the pointer — it accesses the value at the address the pointer holds.

int x = 25;
int *intptr = &x;

cout << intptr;    // prints the ADDRESS (e.g., 0x7ffee4bc)
cout << *intptr;   // prints the VALUE at that address: 25

// Dereferencing lets you modify the original variable
*intptr = 99;
cout << x;         // prints 99 — x was changed through the pointer!

* means two different things depending on context: in a definition (int *ptr), it declares a pointer variable. In an expression (*ptr), it dereferences — accesses the value at the pointer's address. Context is everything.

Arrays and Pointers

▶

An array name IS a pointer constant — it holds the address of the first element. This is why arrays behave differently from regular variables when passed to functions.

int vals[] = {4, 7, 11};

cout << vals;       // prints starting address (e.g., 0x4a00)
cout << vals[0];    // prints 4
cout << *vals;      // also prints 4 — dereference the array name

// Pointer can be used like an array name
int *valptr = vals;
cout << valptr[1];  // prints 7
cout << valptr[2];  // prints 11

// vals[i] is EXACTLY equivalent to *(vals + i)
cout << vals[2];         // 11
cout << *(vals + 2);     // also 11

vals[i] and *(vals + i) are identical — the compiler converts one to the other internally. No bounds checking happens either way. The array name is a CONSTANT pointer — you can't change what it points to (vals = something_else is illegal), but a pointer variable assigned to it can be moved freely.

Pointer Arithmetic

▶

Adding or subtracting integers from pointers moves them by multiples of the pointed-to type's size — not by raw bytes.

int vals[] = {4, 7, 11};
int *valptr = vals;        // points to vals[0]

// valptr + 1 means address + (1 * sizeof(int)) = next element
cout << *(valptr + 1);    // 7
cout << *(valptr + 2);    // 11

// Move the pointer itself
valptr++;                  // now points to vals[1]
cout << *valptr;          // 7

valptr--;                  // back to vals[0]
OperationAllowed?
ptr + n / ptr - nYes — moves n elements forward/back
ptr++ / ptr--Yes — moves one element
ptr1 - ptr2Yes — gives number of elements between them
ptr1 == ptr2Yes — compares addresses
ptr * n / ptr + ptrNo — not meaningful

ptr1 == ptr2 compares ADDRESSES (are they pointing to the same location?). *ptr1 == *ptr2 compares CONTENTS (do both locations hold the same value?). These are very different questions — always know which one you need.

Pointers as Function Parameters

▶

Passing a pointer to a function allows the function to modify the original variable — like pass by reference, but explicit. This is how C (which has no reference parameters) achieves pass-by-reference behavior.

// Prototype: * declares pointer parameter
void getNum(int *ptr);

// Definition: * dereferences to access original
void getNum(int *ptr)
{
    cin >> *ptr;    // stores input directly into the pointed-to variable
}

// Call: & passes the address of the variable
int num;
getNum(&num);       // pass address of num

// Classic swap using pointers
void swap(int *x, int *y)
{
    int temp = *x;
    *x = *y;
    *y = temp;
}

int num1 = 2, num2 = -3;
swap(&num1, &num2);  // pass addresses
// num1 = -3, num2 = 2 after the call

Functions should only return pointers to: (1) data passed in as an argument, or (2) dynamically allocated memory. NEVER return a pointer to a local variable — local variables are destroyed when the function ends, leaving a dangling pointer to invalid memory.

Const Pointers

▶

Three combinations of const with pointers — each restricts something different:

int value = 22;
const double payRates[] = {18.55, 17.45, 12.85};

// 1. Pointer to const: cannot change the VALUE through the pointer
//    (but can change where the pointer points)
const int *ptr = &value;
*ptr = 99;          // ERROR — can't modify through pointer-to-const
ptr = &otherVar;    // OK — can point elsewhere

// Use for array parameters you don't want modified:
void display(const double *rates, int size)  // protects the array

// 2. Constant pointer: cannot change WHERE it points
//    (but can change the value at the address)
int * const ptr2 = &value;
*ptr2 = 99;         // OK — can modify the value
ptr2 = &otherVar;   // ERROR — can't repoint

// 3. Constant pointer to constant: cannot change either
const int * const ptr3 = &value;
*ptr3 = 99;         // ERROR
ptr3 = &otherVar;   // ERROR

When passing arrays to functions that should NOT modify them, use const pointer parameters. This protects the original array from accidental changes and documents the function's intent clearly.

Dynamic Memory Allocation

▶

Dynamic allocation lets you request memory at runtime rather than compile time. The new operator allocates memory and returns its address. The delete operator releases it.

// Allocate a single variable
double *dptr = nullptr;
dptr = new double;     // allocate, get address back
*dptr = 3.14;          // use it
delete dptr;           // free it when done
dptr = nullptr;        // good practice — avoid dangling pointer

// Allocate an array
const int SIZE = 25;
double *arrayptr = new double[SIZE];

for (int i = 0; i < SIZE; i++)
    arrayptr[i] = i * i;    // use like a regular array

delete [] arrayptr;          // free array — note the []
arrayptr = nullptr;

Memory leaks happen when you allocate with new but never call delete — the memory is reserved for the program's lifetime but inaccessible. In long-running programs this is serious. Rules: always pair new with delete, always pair new[] with delete[] (never mix them), only call delete on dynamically allocated memory — never on local/global variables.

Smart Pointers (C++11)

▶

Smart pointers automatically release memory when they go out of scope — no manual delete needed, no memory leaks. Requires #include <memory>.

#include <memory>

// unique_ptr: one owner, auto-deleted when out of scope
unique_ptr<int> ptr(new int);
*ptr = 42;
cout << *ptr;
// No delete needed — memory freed automatically when ptr goes out of scope

// unique_ptr for an array
unique_ptr<int[]> arrPtr(new int[10]);
arrPtr[0] = 5;   // use like array

unique_ptr

Only one pointer can own the resource. Auto-deletes when the pointer goes out of scope. Cannot be copied — only moved. Best default choice.

shared_ptr

Multiple pointers can share ownership. Memory freed when the last shared_ptr releases it. Uses reference counting. More overhead than unique_ptr.

weak_ptr

Non-owning reference to a shared_ptr's resource. Doesn't prevent deletion. Used to break circular reference cycles between shared_ptrs.

Why Smart Pointers?

Eliminate memory leaks automatically. Make ownership semantics explicit. Exception-safe — memory freed even if an exception is thrown. Prefer over raw pointers in modern C++.
