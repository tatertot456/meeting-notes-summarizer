What Is HTML?

▶

HTML is the language of the World Wide Web — used primarily by browsers to render web pages.

Hypertext

Text and other resources (images, video, etc.) connected by "links." Makes web pages interactive and connected to each other.

Markup

A way to annotate text with "tags" that define how a page is structured and presents itself. Tags wrap content to give it meaning.

Language

Not technically a programming language — it's a set of specifications defining a web page's contents and presentation. No logic, no loops, no decisions.

In Blazor

Everything outside "@" lines is pure HTML and CSS. Lines with "@" are treated as Blazor/C# code. HTML = content. CSS = presentation. C# = behavior.

The three layers of a web page: HTML defines what's on the page (content), CSS defines how it looks (presentation), and C# or JavaScript defines what it does (behavior). Always implement elements first, then formatting, then behaviors.

How HTML Works in the Browser

▶

HTML and CSS serve as the blueprint for a web page. They are plain text files that the browser translates into visual elements rendered on screen.

  • The browser's job is to read the HTML/CSS text and turn it into visual objects
  • Communication happens between a client (browser) and a server
  • The internet accepts requests and returns responses
  • The server may interact with databases and other web services to build the response

The browser is essentially an HTML/CSS interpreter. The same HTML file may look slightly different across browsers — they each have their own rendering engines. This is why cross-browser testing matters in web development.

The Structure of an HTML Page

▶

Every HTML page follows the same fundamental structure:

<!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="utf-8">
        <title>Page Title</title>
        <style>
            /* CSS goes here */
        </style>
    </head>
    <body>
        <!-- Page content goes here -->
    </body>
</html>
ElementPurpose
<!DOCTYPE html>Instruction (not an element) — tells the browser this is an HTML5 document. Each version of HTML has its own DOCTYPE.
<html>Container for the entire page. Surrounds everything else. The lang attribute specifies the document language.
<head>Surrounds elements associated with the document as a whole — metadata, title, styles. NOT displayed on the page.
<body>Surrounds elements that actually display in the web page — all visible content goes here.

Container elements must be properly nested — if you open a container inside another container, you must close the inner one before closing the outer one. Improper nesting causes rendering bugs that can be difficult to track down.

head element contains: title (required by spec), meta elements, style (CSS), and links to external resources.

<head>
    <meta charset="utf-8">
    <meta name="author" content="NNU Student">
    <meta name="description" content="My first page">
    <meta name="keywords" content="Blazor, awesome">
    <title>Hello World</title>
</head>

Tags, Elements & Content

▶

Elements define the structure of content. They surround their content with a start tag and end tag.

<h1>Hello World!</h1>
<!--  ↑ start tag    ↑ end tag (starts with /) -->

Dr. McCarty was <strong>very</strong> disappointed.
<!-- strong makes text bold/important -->

Container Elements

Have a start tag AND an end tag. Content lives between them. Examples: <h1>, <p>, <div>, <body>, <html>. Most HTML elements are containers.

Void Elements

Have just ONE tag — content is stored within the tag itself as attributes. No end tag. Examples: <meta>, <br>, <hr>, <img>.

Common heading and text elements:

TagPurpose
<h1> to <h6>Headings — h1 is largest, h6 is smallest
<p>Paragraph — adds blank lines above and below
<strong>Important/bold text
<br>Line break (void — no end tag)
<hr>Horizontal rule/line (void — no end tag)
<div>Generic container — groups related content, single line breaks above/below

When discussing a tag in general (not a specific instance), drop the angled brackets and call it an element. <h1> = a specific tag. h1 element = the concept. When should you use <p> vs <div>? Use <p> for actual prose paragraphs. Use <div> as a structural container for grouping and layout purposes.

Attributes

▶

Attributes are properties of an element that provide additional information. They appear inside the start tag and almost always have a value assigned with =.

<meta charset="utf-8">
<!--  ↑ attribute name  ↑ attribute value (always in quotes) -->

<html lang="en">
<!-- lang attribute specifies the page language -->

<meta name="author" content="NNU Student">
<meta name="description" content="My first program.">
<meta name="keywords" content="Blazor, awesome">

Always surround attribute values with quotes — this forms a string (zero or more characters surrounded by quotes). The charset attribute tells the browser what character encoding to use. UTF-8 supports virtually every character in every language and is the standard choice for modern web pages.

Most meta elements use the name attribute to specify what type of information is provided, and the content attribute to provide the actual value. Meta elements help search engines find and categorize your page.

Key Body Elements

▶ 
<!-- Horizontal rule: renders a full-width line -->
<hr>

<!-- Paragraph: blank lines above and below -->
<p>Welcome to your first HTML page.</p>

<!-- Line break: forces a new line mid-paragraph -->
<p>
    High of 95 degrees today.<br>
    With 30% humidity it will feel like 102.
</p>
<!-- Note: pressing Enter in your code does NOT create a
     line break on screen — only <br> does -->

<!-- Div: generic container, single line breaks -->
<div>
    Class Days:<br>
    M, W, F
</div>

The <div> element has a very special purpose beyond simple text grouping — it acts as a container for other objects. Use div to create "panels" for layout, isolate components for styling, and organize your page into sections. This is the foundation of modern web layout. Think of divs as invisible boxes you arrange on the page.

CSS Preview

▶

CSS (Cascading Style Sheets) controls presentation. It can be embedded in the <head> section using a <style> element, or placed in a separate .css file.

<head>
    <style>
        body {
            font-family: Arial, sans-serif;   /* CSS property: value */
            background-color: #ffffff;
            color: #333333;
            text-align: center;
            margin-top: 20px;
        }
    </style>
</head>

CSS works with properties and values. The property names what you're changing (font, color, size). The value specifies what to change it to. In Blazor, component-specific CSS goes in a .css file with the same name as the .razor file.

CSS is what separates a professional-looking site from a plain one. However, always implement your HTML structure first, then apply CSS styling. Trying to style elements before they exist leads to confusion. Structure first, then presentation, then behavior.

Brief History of HTML

▶

The web didn't appear out of nowhere — it evolved from decades of research into document publishing:

  • 1960s: Researchers recognized the need for a common text-based language for publishing electronic documents, regardless of the reader's display tool
  • 1980/1986: SGML (Standard Generalized Markup Language) published — introduced DOCTYPE declarations and element tags
  • 1989: Tim Berners-Lee at CERN proposed adding "hypertext links" to research papers to make related content easily accessible
  • 1989–1991: Berners-Lee expanded SGML into HTML, introduced HTTP (so any server could exchange HTML with a client), and created a prototype browser
  • 1993: First formal proposal for HTML submitted to the Internet Engineering Task Force (IETF)
  • 1994: Berners-Lee founded the World Wide Web Consortium (W3C) at MIT — which has maintained the HTML standard ever since

Mozilla Firefox got its name from a combination of "Mosaic" (an early browser) and "Godzilla." The lang attribute (e.g., <html lang="en">) specifies the language of the page's content — not a meta element type attribute.
