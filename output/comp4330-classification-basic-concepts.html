Classification vs. Numeric Prediction

▶

Classification

Predicts categorical class labels (discrete/nominal). Constructs a model from training data to classify new data.

Numeric Prediction (Regression)

Models continuous-valued functions — predicts unknown or missing numeric values.

Classification is a two-step process:

  • Model Construction: Learn a classifier from labeled training tuples
  • Model Usage: Apply classifier to unlabeled data — only if accuracy is acceptable

Test set must be independent of training set to avoid overfitting. Accuracy = % of test samples correctly classified.

Decision Tree Induction

▶

Decision trees use a greedy, top-down, divide-and-conquer approach:

  • All training examples start at the root
  • Attributes are selected based on information gain or entropy
  • Examples are partitioned recursively based on selected attributes

Stopping conditions:

  • All samples belong to the same class
  • No remaining attributes — use majority voting
  • No samples left

Overfitting prevention:

Prepruning

Halt tree construction early if goodness measure falls below a threshold. Difficult to choose the threshold.

Postpruning

Grow the full tree, then remove branches using a validation set to find the best pruned version.

Attribute Selection: Information Gain & Gini

▶

Information Gain (ID3/C4.5)

Selects attribute with highest gain. Entropy measures variability. Biased toward multivalued attributes.

Gain Ratio (C4.5)

Normalizes information gain by SplitInfo to overcome multivalued bias. Tends to prefer unbalanced splits.

Gini Index (CART)

Measures impurity. Selects attribute with smallest gini split. Biased to multivalued attributes, difficulty with many classes.

General Comparison

Most measures return good results. No single measure is significantly superior across all datasets.

For the buys_computer example: Gain(age)=0.246, Gain(income)=0.029, Gain(student)=0.151, Gain(credit_rating)=0.048 — age is selected first.

Bayesian Classification

▶

Bayesian classifiers perform probabilistic prediction using Bayes' Theorem:

posteriori = likelihood × prior / evidence

The Naïve Bayesian classifier assumes conditional independence between attributes — greatly reducing computation. Despite this simplification, it performs comparably to decision trees and neural networks in many cases.

Advantages

Easy to implement, incremental learning, each example can adjust probabilities, good baseline performance.

Disadvantages

Assumes attribute independence (often violated). Dependencies (e.g., patient symptoms) can't be modeled. Use Bayesian Belief Networks for dependencies.

Zero-probability problem: Use Laplacian correction (add 1 to each case) to avoid any conditional probability being zero, which would zero out the entire product.

Rule-Based Classification

▶

Rules take the form: IF condition THEN class. Easier to understand than large trees.

  • Coverage: Fraction of tuples the rule applies to
  • Accuracy: Fraction of covered tuples correctly classified

Conflict resolution strategies:

  • Size ordering: toughest requirement (most attribute tests) wins
  • Class-based ordering: by prevalence or misclassification cost
  • Rule-based ordering (decision list): priority list by rule quality

Sequential covering algorithms (FOIL, RIPPER) learn rules one at a time, removing covered tuples before learning the next rule.

Validation methods: Holdout (2/3 train, 1/3 test), k-fold cross-validation (k=10 most popular), Bootstrap (.632 method for small datasets).

ROC curves plot Sensitivity vs (1-Specificity). Area under ROC = 1.0 is perfect. Closer to 0.5 (diagonal) = worse. Used to compare classifiers M1 vs M2 statistically.

Ensemble Methods

▶

Bagging

Bootstrap Aggregation — train k classifiers on k bootstrap samples, combine by majority vote. Robust to noise, proven accuracy improvement.

Boosting (AdaBoost)

Iteratively trains classifiers, increasing weight of misclassified tuples. Generally higher accuracy than bagging but risks overfitting.

Random Forest

Ensemble of decision trees using random attribute selection at each node. Comparable to AdaBoost, more robust to outliers, faster than bagging/boosting.

Class Imbalance

Handle rare classes via oversampling, under-sampling, threshold-moving, or ensemble techniques.
