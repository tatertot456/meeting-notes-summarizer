Parts of a C++ Program

▶ 
// sample C++ program          ← comment
#include <iostream>           ← preprocessor directive
using namespace std;           ← which namespace to use
int main()                     ← beginning of function named main
{                              ← beginning of block for main
    cout << "Hello World!";   ← output statement
    return 0;                  ← send 0 to operating system
}                              ← end of block for main
ElementPurpose
// commentIgnored by compiler — for human readers only

The #include directive is a preprocessor directive — not part of C++ itself. The preprocessor handles it before the compiler sees the file. Never put a semicolon at the end of a #include line.

Output with cout

▶

The cout object displays output on the screen. Use the stream insertion operator << to send data to it.

cout << "Programming is fun!";

// Chain multiple items
cout << "Hello " << "there!";

// Two separate statements produce the same single line
cout << "Programming is ";
cout << "fun!";

Starting a new line — two ways:

// Using endl manipulator (outside the string)
cout << "Programming is" << endl;
cout << "fun!";

// Using \n escape sequence (inside the string)
cout << "Programming is\n";
cout << "fun!";

endl vs \n: both start a new line, but endl also flushes the output buffer. The last character in endl is a lowercase L — not the number 1. \n must be placed INSIDE the string quotes. endl goes OUTSIDE the quotes, chained with <<.

Variables, Literals & Identifiers

▶

A variable is a named storage location in memory. It has a name and a data type, and must be defined before it can be used.

int item;            // defines a variable named item of type int
int length = 12;     // define AND initialize in one step
int length, width;   // define multiple variables of the same type

A literal is a value written directly into code:

"hello world"   // string literal
12              // integer literal
3.14            // floating-point literal
'A'             // character literal — single quotes!

Identifier rules — names you make up for variables, functions, etc.:

  • First character must be a letter or underscore _
  • After that: letters, digits, or underscores are all fine
  • Case sensitive — Area and area are different identifiers
  • Cannot use C++ reserved keywords (int, double, return, etc.)
  • Should be descriptive: itemsOrdered not x

The auto keyword (C++11) lets the compiler infer the type from the initialization value: auto amount = 100; → int, auto rate = 12.0; → double, auto code = 'D'; → char. Use when the type is obvious from context.

Assignment: the variable receiving a value MUST be on the LEFT side of =.

item = 12;    // correct
12 = item;    // ERROR — literals cannot receive values

Character literals use SINGLE quotes ('A'). String literals use DOUBLE quotes ("Hello"). Mixing them up is a common beginner mistake. Strings are stored with a null terminator \0 at the end — "Hello" takes 6 bytes, not 5.

Arithmetic Operators

▶
OperatorOperationExampleResult
+Addition7 + 310
-Subtraction7 - 34
*Multiplication7 * 321
/Division13 / 52 (integer!) or 2.6 (float)
%Modulus (remainder)13 % 53

Integer division: if BOTH operands are integers, / returns only the quotient with no remainder — 13/5 = 2, not 2.6. If EITHER operand is floating point, the result is floating point — 13/5.0 = 2.6. The % (modulus) operator requires both operands to be integers — 13 % 5.0 is an error.

cout << 13 / 5;    // 2  (integer division — remainder dropped)
cout << 13 / 5.0;  // 2.6 (float division — one operand is double)
cout << 13 % 5;    // 3  (remainder of 13 ÷ 5)

Operator types by number of operands: unary (1 operand, e.g. -5), binary (2 operands, e.g. 13 - 7), ternary (3 operands, e.g. exp1 ? exp2 : exp3).

Comments

▶

Comments document the program for human readers — they are completely ignored by the compiler. Use them to explain the purpose of the program, describe variables, and clarify complex sections.

// Single-line comment: from // to end of line
int length = 12; // length in inches

/* Multi-line comment: from /* to */
   Can span multiple lines.
   Useful for longer explanations. */

int area; /* can also be on one line */

Good commenting habit: explain the WHY, not the WHAT. "// multiply length by width" is redundant — the code already shows that. "// area formula for rectangular room" adds context the code can't express.

Named Constants

▶

A named constant (constant variable) is a variable whose value cannot be changed after it is set. Use the const keyword. By convention, named constants are written in ALL_CAPS.

const double TAX_RATE = 0.0675;
const int NUM_STATES = 50;

// Attempting to change a const causes a compile error:
TAX_RATE = 0.08;  // ERROR!

Why use named constants instead of magic numbers? If your tax rate appears as 0.0675 in 50 places throughout your code and it changes, you have to update all 50. With const double TAX_RATE = 0.0675, you update one line. Named constants also make code self-documenting — TAX_RATE is far more readable than 0.0675.

Scope & Programming Style

▶

Scope defines the part of the program where a variable can be accessed. A variable cannot be used before it is defined — the compiler will catch this.

int main() {
    cout << x;  // ERROR — x hasn't been defined yet
    int x = 5;
}

Programming style is the visual organization of source code — use of spaces, tabs, and blank lines. Style does NOT affect how the program runs (it doesn't change syntax) but dramatically affects readability. Consistent indentation, meaningful variable names, and strategic blank lines make code far easier to understand and maintain.
