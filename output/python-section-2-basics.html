Case Sensitivity & Magic Commands

▶

Python is case-sensitive — var and Var are completely different variables. This is a common source of errors for beginners.

myVar = "Python is case-sensitive"
print(MyVar)   # NameError — MyVar doesn't exist
print(myVar)   # OK — prints the string

Convention in this course: use camelCase (myVar) rather than the Python standard snake_case (my_var) — to stay consistent with C# conventions used in other courses.

What Are Variables?

▶

Variables are bits of computer memory used to store state information. State information is what allows programs to make informed decisions rather than arbitrary ones.

Think of it this way: "Are you hungry? Bored? Cold?" — your answer describes your current state. Programs need to track similar states to make decisions: stop when the light is red, go when it's green.

Numbers

int, float, complex

Text

str (strings)

Collections

list, tuple, range, dict, set

Other

bool, bytes, objects, and more

# A program is lines of code that, when run, "does something"
print("Hello World!")

# Multiple commands on one line (legal but avoid)
print("First") ; print("Second")

In Python, commands are (mostly) just text on a line. The command terminator (semicolon) is optional — unlike C++ or Java where it's required. Keep one command per line for readability.

Binary Representation & Variable Types

▶

Computers represent all data internally in binary (base-2) because electronic systems have two reliable states: on and off.

# Base 10: each place is a power of 10
1943 = 1×10³ + 9×10² + 4×10¹ + 3×10⁰

# Base 2: each place is a power of 2
1101 = 1×2³ + 1×2² + 0×2¹ + 1×2⁰ = 13 (base 10)

# With k bits, largest unsigned integer = 2^k - 1
# 4 bits → max = 2^4 - 1 = 15
# Arithmetic overflow: 1110 + 0010 = 10000 (needs 5 bits!)

Floating-point numbers are split into mantissa and exponent — not stored directly like integers. (More detail in Computer Architecture.)

Dynamic Typing & Value vs Reference

▶

Strongly Typed (C++)

Must declare variable type before using it. Compiler checks all type usage. Type cannot change.

Weakly Typed (JavaScript)

Doesn't care about types at all. No enforcement. Easy to write but easy to get wrong.

Dynamically Typed (Python)

No declaration needed. Python determines type from the assignment. Type IS enforced after assignment, but can change at runtime.

type() function

Use type(var) to check a variable's current type at any time.

a = 10
print(type(a))      # <class 'int'>
a = "Python"
print(type(a))      # <class 'str'>
a = False           # capitalization matters!
print(type(a))      # <class 'bool'>

Pass by Value vs Pass by Reference:

# Pass by VALUE — basic types (int, float, str, bool)
# Changes to var2 do NOT affect var1
var1 = 1
var2 = var1
var1 = 2
# var2 is still 1

# Pass by REFERENCE — complex types (list, dict, etc.)
# Changes to var2 DO affect var1 (same memory location)
var1 = list(range(3))   # [0, 1, 2]
var2 = var1             # var2 points to SAME list
var1.append(10)
# var2 is now [0, 1, 2, 10] too!

A reference is not the value itself — it's the memory address where the value lives. Think of it like a home address: the address isn't the house, but it tells you where to find it. Basic types (int, str, bool) are passed by value — copies. Complex types (list, dict) are passed by reference — same object in memory.

Strings

▶ 
# Single or double quotes (must match)
var1 = "This is a string."
var2 = 'Also a string.'

# Triple quotes for multiline strings
var3 = """This is a
multiline string
"""

print(var1)   # This is a string.
print(var3)   # prints across multiple lines

# Strings are IMMUTABLE — cannot change characters in place
var1[2] = "c"   # TypeError — strings can't be modified

# Type conversion to string
x = 10
var5 = str(x)   # "10"

# Windows path — backslash problem
print("C:\Data\project")    # \p and \D may be interpreted as escapes

print("C:\\Data\\b1")       # escaped backslashes — same result

Strings are immutable BUT iterable — you can loop through them character by character. They are both a simple type (str) and a complex type (sequence/tuple-like). The r prefix before a string makes it a "raw string" — no escape processing, perfect for Windows paths.

Operators

▶

Operators are polymorphic — they behave differently depending on what they operate on. The + operator adds numbers but concatenates strings.

Binary / Arithmetic

Work on pairs of variables. +, -, *, /, //, %, **

Unary

Handle a single variable. Note: Python does NOT have ++ or --. Use += 1 instead.

Logical

Test true/false conditions. and, or, not. Note: AND precedes OR in precedence.

Assignment

Assign and sometimes modify values. =, +=, -=, *=, /=

Ternary

Three-variable conditional: value_if_true if condition else value_if_false

Bitwise

Operate on individual bits. &, |, ^, ~, <<, >>

When operator precedence is unclear, use parentheses — they always make intent explicit and code more readable. Multiplication/division precede addition/subtraction. AND precedes OR. When in doubt, add parentheses.

Capturing User Input

▶

The input() function prompts the user and returns their response as a string. Always convert if you need a number.

# Basic input
player1 = input("What is your name, player? ")
print("Welcome, ", player1)

# Numeric input — input() ALWAYS returns a string
num1 = input("Enter a number to square: ")
num2 = float(num1)      # convert string to float first!
print("The result is ", num2 * num2)

# Exercise — try this yourself:
firstName = input("Your first name is: ")
lastName = input("Your last name is: ")
print("Your name is: " + firstName + " " + lastName)

input() always returns a string — even if the user types a number. If you try to do math with it without converting first, you'll get a TypeError. Use int() or float() to convert. This is one of the most common beginner mistakes in Python.

Numeric, Boolean & Null Types

▶ 
# Numeric types
x = 43
y = 9
x / y           # 4.777... (Python 3 always does true division)
                # large ints auto-convert to long integer
                # floats are 64-bit double precision

# Boolean — True and False (capitalization matters!)
x < 50                      # True
x < 50 and y > 15           # False (y is 9, not > 15)
x < 50 or y > 15            # True

# Booleans can be numeric — 0 is False, everything else is True
print(bool(12))   # True
print(bool(1))    # True
print(bool(0))    # False
print(bool(-1))   # True (non-zero!)

if x:
    print("x is not empty!")  # any non-zero number is truthy

# Null — None is Python's reserved null value
z = None
print("not empty" if x else "empty!")   # not empty
print("not empty" if z else "empty!")   # empty!
bool(z)                                  # False

In Python, truthiness extends beyond True/False: 0, None, empty strings (""), empty lists ([]), and empty dicts ({}) are all falsy. Everything else is truthy. This is extremely useful in loops and conditionals — you can write "if myList:" instead of "if len(myList) > 0:".

Dates & Times

▶

Dates and times are tricky in most languages. Python's datetime module handles this. Because datetime is a class, it has properties and methods to work with.

import datetime

# Current date and time
x = datetime.datetime.now()
print(x)            # 2025-01-15 14:30:00.123456
print(x.year)       # 2025
print(x.day)        # 15
print(x.strftime("%A"))   # Wednesday (day name)

# Specific date
start_date = datetime.date(2023, 1, 1)
print(start_date.strftime("%A"))   # Sunday

# Find the day you were born
birthday = datetime.date(2004, 6, 15)   # replace with your date
print(birthday.strftime("%A"))

# Common strftime format codes:
# %A = full weekday name    %B = full month name
# %d = day of month         %Y = 4-digit year
# %H = hour (24hr)          %M = minute

Reference: docs.python.org/3/library/datetime.html — Python's documentation is excellent and should be your first stop for anything datetime-related. strftime() formats a date as a string using format codes. strptime() does the reverse — parses a string into a date object.
