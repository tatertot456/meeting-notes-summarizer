Increment & Decrement Operators

▶

++ adds 1 to a variable. -- subtracts 1. Both can be used in prefix (before) or postfix (after) position — and the difference matters in expressions.

ModeSyntaxBehavior
Prefix++val / --valIncrement/decrement FIRST, then return new value
Postfixval++ / val--Return current value FIRST, then increment/decrement
int num, val = 12;

cout << val++;   // displays 12, THEN val becomes 13
cout << ++val;   // val becomes 14, THEN displays 14
num = --val;     // val becomes 13, stores 13 in num
num = val--;     // stores 13 in num, THEN val becomes 12

Prefix vs postfix only matters when the operator is used INSIDE a larger expression. As a standalone statement (val++;), both are identical. Cannot apply ++ or -- to an expression — only to a variable with a memory location. (num1 + num2)++ is illegal.

The while Loop

▶

The while loop is a pretest loop — the condition is evaluated BEFORE each iteration. If the condition is false from the start, the body never executes.

// General format
while (expression)
    statement;   // or block in { }

// Example: count from 1 to 5
int number = 1;
while (number <= 5)
{
    cout << number << endl;
    number++;     // must update or infinite loop!
}

// Input validation pattern
cout << "Enter a number less than 10: ";
cin >> number;
while (number >= 10)
{
    cout << "Invalid! Enter a number less than 10: ";
    cin >> number;
}

Infinite loops happen when the expression never becomes false — always make sure the loop body contains code that eventually makes the condition false. The while loop is ideal for input validation (repeat until valid) and reading data terminated by a sentinel value.

The do-while Loop

▶

The do-while loop is a posttest loop — the body executes FIRST, then the condition is checked. The body always executes at least once.

// General format — note semicolon after while()
do
{
    statement;
} while (expression);

// Example: always runs at least once
int x = 1;
do
{
    cout << x << endl;
} while (x < 0);   // false, but still ran once!

// Classic use: menu-driven programs
do
{
    displayMenu();
    cin >> choice;
    processChoice(choice);
} while (choice != 4);  // repeat until user chooses quit

The semicolon after while(expression) is required — don't forget it. The do-while is the natural choice for menus because you always need to show the menu at least once before you can test the user's choice.

The for Loop

▶

The for loop is a pretest loop designed for counter-controlled iteration. It bundles initialization, test, and update into one clean line.

// General format
for (initialization; test; update)
    statement;   // or block in { }

// Example: count from 1 to 5
for (int count = 1; count <= 5; count++)
    cout << "Hello" << endl;

// Execution order:
// 1. initialization (once only)
// 2. test — if false, exit loop
// 3. body executes
// 4. update
// 5. repeat from step 2

Modifications:

// Multiple initializations and updates (comma-separated)
for (int x=1, y=1; x <= 5; x++, y++)
    cout << x + y << endl;

// Omit initialization if already done
int sum = 0, num = 1;
for (; num <= 10; num++)
    sum += num;

// Variable declared inside for — scope is the loop only
for (int i = 0; i < 10; i++)
    cout << i;

No semicolon after the update expression or after the closing parenthesis. Variables declared in the initialization expression only exist inside the loop — they go out of scope when the loop ends. Use the for loop when you know exactly how many iterations are needed.

Loop Techniques: Counters, Totals & Sentinels

▶

Counter

A variable incremented or decremented each iteration — used to control how many times the loop runs. Must be initialized before the loop.

Running Total (Accumulator)

A variable that accumulates a sum across iterations. Initialize to 0 before the loop, then add each value inside the loop.

Sentinel

A special value that signals the end of input — chosen so it can't be confused with valid data (e.g., -999 for test scores). Loop continues until sentinel is entered.

break & continue

break exits the loop immediately. continue skips to the end of the current iteration (test/update for while/for). Use sparingly — both make logic harder to follow.

// Running total example
int sum = 0, num = 1;
while (num <= 10)
{
    sum += num;   // accumulate
    num++;
}
cout << "Sum = " << sum;

// Sentinel example
int score, total = 0;
cout << "Enter score (-999 to stop): ";
cin >> score;
while (score != -999)
{
    total += score;
    cout << "Enter score (-999 to stop): ";
    cin >> score;
}

Nested Loops

▶

A nested loop is a loop inside the body of another loop. The inner loop completes ALL its iterations for each single iteration of the outer loop.

// Multiplication table
for (int row = 1; row <= 3; row++)        // outer: 3 iterations
{
    for (int col = 1; col <= 3; col++)    // inner: 3 iterations each
    {
        cout << row * col << "\t";
    }
    cout << endl;
}
// Output:
// 1  2  3
// 2  4  6
// 3  6  9

Total iterations = outer iterations × inner iterations. A 3×3 nested loop runs the inner body 9 times total. When break is used inside a nested loop, it only exits the INNER loop — the outer loop continues normally.

Choosing the Right Loop

▶
LoopTypeBest Used When
whilePretestNumber of iterations unknown; validating input; reading sentinel-terminated data
do-whilePosttestMust execute at least once; menu-driven programs; repeat-until patterns
forPretestExact number of iterations known; counter-controlled loops; clean init/test/update

File I/O

▶

Files allow data to persist between program runs. Requires #include <fstream>. Three steps: open → use → close.

#include <fstream>

// Define file stream objects
ifstream infile;    // for reading
ofstream outfile;   // for writing

// Open files
infile.open("data.txt");
outfile.open("report.txt");  // created if it doesn't exist
                              // existing file is ERASED first!

// Test for open failure
if (!infile)
{
    cout << "Error: could not open file!\n";
    return 1;
}

// Write to file (same as cout)
outfile << "Inventory Report\n";
outfile << partNum << " " << qty << endl;

// Read from file (same as cin)
infile >> partNum;
infile >> qtyInStock >> qtyOnOrder;

// Loop through entire file
int number;
while (infile >> number)   // returns false at end of file
{
    cout << number << endl;
}

// Close files when done
infile.close();
outfile.close();

// Let user specify filename (C++11)
string filename;
cout << "Enter filename: ";
cin >> filename;
infile.open(filename);             // C++11
infile.open(filename.c_str());     // pre-C++11 (c_str converts to null-terminated)

Always close files explicitly — don't rely on the OS to close them. Buffered output data may not be written to disk until the file is properly closed. Opening an output file that already exists silently erases its contents — make sure you're opening the right file. Always test for open failure before reading or writing.
