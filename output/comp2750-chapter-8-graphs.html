Graph Definitions & Terminology

▶

A graph is a generalized tree with no restriction on connections — it can express any relationship, not just hierarchical ones. A simple graph G = (V, E) consists of a finite set of vertices V and a collection E of unordered pairs of distinct vertices (edges).

Vertex / Node

Each element of V. The order of a graph is |V| — the number of vertices.

Edge / Arc

Each element of E (a pair of vertices). The size of a graph is |E| — the number of edges.

Simple Graph

No self-loops, no multiple edges between the same pair of vertices.

Multigraph

Multiple edges allowed between the same pair of vertices. aᵢⱼ = number of edges between vᵢ and vⱼ.

Digraph

Directed graph — edges have direction. A circuit revisits a vertex; a cycle does not revisit vertices.

Complete Graph Kₙ

Every pair of vertices is connected by an edge. Kₙ has n(n-1)/2 edges.

Graph Representation

▶

Adjacency List

For each vertex, list all adjacent vertices. Space-efficient for sparse graphs. Good for traversal. Can be a table (star representation) or linked list.

Adjacency Matrix

|V|×|V| binary matrix. aᵢⱼ = 1 if edge(vᵢ,vⱼ) exists. n! possible orderings. Fast edge lookup O(1) but O(V²) space.

Incidence Matrix

|V|×|E| binary matrix. Row = vertex, Column = edge. Entry = 1 if vertex is endpoint of edge. Columns with one 1 represent loops (multigraph).

Choice of representation depends on the operations needed. Adjacency lists are best for sparse graphs and traversal. Adjacency matrices excel at edge-existence queries and dense graphs. Incidence matrices are useful for certain theoretical analyses but rare in practice.

Graph Traversals — DFS & BFS

▶

Graph traversals cannot simply reuse tree algorithms — graphs can have cycles and isolated vertices.

Depth-First Search (DFS)

Visit a vertex, then recursively visit all unvisited adjacent vertices. Backtrack when stuck. Uses a stack (implicitly via recursion). Forward edges = tree edges; back edges = edges to already-visited nodes. Assigns unique numbers to vertices during traversal.

Breadth-First Search (BFS)

Visit a vertex, enqueue all unvisited neighbors, process queue. Explores level by level — finds shortest path (unweighted). Uses a queue explicitly. Visits closer nodes before farther ones.

// DFS pseudocode
DFS(vertex v):
    mark v as visited
    assign v a number
    for each neighbor u of v:
        if u not visited:
            DFS(u)   // recursive — uses runtime stack

// BFS pseudocode
BFS(start vertex s):
    enqueue s, mark s as visited
    while queue not empty:
        v = dequeue
        visit v
        for each neighbor u of v:
            if u not visited:
                mark u as visited
                enqueue u

DFS creates a spanning tree (or forest). Edges included = forward edges; omitted = back edges. If vertices remain unvisited after DFS completes, restart at an unvisited vertex — this handles disconnected graphs. DFS is the foundation for cycle detection, topological sort, and finding strongly connected components.

Shortest Paths — Dijkstra & Ford

▶

Weighted graph edges represent distance, time, cost, etc. label(v) = (currDist(v), predecessor(v)). Two families of algorithms:

Label-Setting (Dijkstra, 1956)

Once a vertex is assigned a distance, it never changes. Selects the unvisited vertex with the smallest current distance each iteration. CANNOT handle negative edge weights. O(V²) basic, O(E log V) with priority queue.

Label-Correcting (Ford)

Any label can be updated at any time. Handles negative edge weights (but NOT negative cycles). Processes entire graph repeatedly until no improvement. Basis for Bellman-Ford algorithm.

// Dijkstra's Algorithm
DijkstraAlgorithm(graph, source):
    dist[source] = 0; dist[all others] = ∞
    toBeChecked = all vertices
    while toBeChecked not empty:
        v = vertex in toBeChecked with min dist[v]  // key step
        remove v from toBeChecked
        for each neighbor u of v:
            if dist[v] + weight(v,u) < dist[u]:
                dist[u] = dist[v] + weight(v,u)
                pred[u] = v

// Ford's Algorithm (label-correcting)
// Repeatedly scans all edges, updating distances
// Terminates when no more updates occur
// Queue variant: only add updated vertices to queue
// Deque variant: first-time vertices added to back, revisits to front (~60% faster avg)

Warshall-Floyd — All-Pairs Shortest Path:

// O(V³) — computes shortest path between ALL pairs
// Works with negative weights (but not negative cycles)
// Initialize adjacency matrix with edge weights (∞ if no edge, 0 on diagonal)
for k = 1 to |V|:
    for i = 1 to |V|:
        for j = 1 to |V|:
            if dist[i][k] + dist[k][j] < dist[i][j]:
                dist[i][j] = dist[i][k] + dist[k][j]
// Cycle detection: if diagonal of dist gets changed, graph has a cycle

Dijkstra fails with negative edge weights because once a vertex is "settled" its distance is assumed final — but a negative edge discovered later could provide a shorter path. Use Ford/Bellman-Ford for graphs with negative edges. Note: negative weights are legitimate in some problems (profit/discount graphs). The efficiency of both algorithms depends heavily on the data structure used for toBeChecked.

Cycle Detection & Union-Find

▶

Many algorithms require detecting cycles. Warshall-Floyd can detect them (O(V³)), but that's too slow for many uses.

DFS-Based (Undirected)

During DFS, if we encounter an already-visited vertex via a back edge (not the direct parent), a cycle exists. Simple and O(V+E).

DFS-Based (Digraph)

Side edges between subtrees complicate detection. After all descendants are visited, assign a number larger than any descendant. If a vertex is about to be connected to a vertex with a lower number via a forward edge, a cycle exists.

Union-Find — efficient cycle detection for spanning tree algorithms:

// Each vertex starts in its own set (circular linked list)
// union(u, v): merge the sets containing u and v
//   - Interchange next pointers to merge lists
//   - Update root[] array for shorter list's vertices
//   - length[] array tracks list sizes for efficiency
// find(v): return root[v] — O(1) after union

// Cycle detection with Union-Find:
// For each edge (u,v):
//   if find(u) == find(v): adding this edge creates a cycle!
//   else: union(u, v) — safe to add

Spanning Trees — Kruskal & Dijkstra Methods

▶

A spanning tree of a connected graph includes all vertices with the minimum number of edges (|V|-1). A minimum spanning tree (MST) minimizes the total edge weight.

Kruskal's Algorithm (1956)

Sort all edges by weight. Add each edge to the tree if it doesn't create a cycle (use Union-Find to check). Stop when |V|-1 edges are added. Greedy — always picks globally cheapest safe edge. O(E log E).

Dijkstra's MST Method (1960)

Add edges one at a time in ANY order. If a cycle forms, remove the highest-weight edge from that cycle. No pre-sorting needed. More flexible but requires cycle detection per step.

// Kruskal's Algorithm
KruskalAlgorithm(graph):
    tree = empty
    edges = all edges sorted by weight (ascending)
    for each edge eᵢ in edges:
        if eᵢ does not form a cycle with edges already in tree:
            add eᵢ to tree
        if |tree| == |V| - 1: break   // spanning tree complete

// Dijkstra's MST Method
DijkstraMethod(graph):
    tree = empty
    for each edge eᵢ (any order):
        add eᵢ to tree
        if tree now contains a cycle:
            remove the maximum-weight edge from that cycle

Connectivity

▶

Undirected graphs: A graph is connected if a path exists between any two vertices. An n-connected graph has at least n vertex-disjoint paths between any two vertices.

Cut-Vertex (Articulation Point)

A vertex whose removal disconnects the graph. Found via DFS: a vertex v is an articulation point if it's the root with 2+ DFS subtrees, OR if some subtree has no back edge to v's ancestors.

Cut-Edge (Bridge)

An edge whose removal disconnects the graph. If the graph can be split on an edge, that edge is a bridge.

Biconnected (2-connected)

At least two non-overlapping paths between any two vertices. No articulation points or bridges. Subgraphs with no articulation points = blocks (biconnected components).

Directed graphs — three types of connectivity:

Weakly Connected

The underlying undirected graph (ignoring edge direction) is connected. Weakest form.

Strongly Connected

For every pair of vertices u, v there is a directed path from u to v AND from v to u. Strictest form.

Strongly Connected Components (SCCs)

Maximal subsets of vertices that are mutually reachable. Found via Tarjan's DFS algorithm using a stack and the pred(v) parameter.

Topological Sort

▶

A topological sort of a DAG (Directed Acyclic Graph) linearly orders vertices so that for every directed edge u→v, u appears before v. Only possible if the graph has NO directed cycles.

// Algorithm: repeatedly find and remove a sink (minimal vertex — no outgoing edges)
topologicalSort(DAG):
    for i = 1 to |V|:
        find vertex v with no outgoing edges (sink)
        assign num(v) = i
        remove v and all edges pointing to v

// DFS-based (more efficient — no actual deletion):
// When DFS backtracks from vertex v (all successors processed),
// assign the next number to v.
// Vertices are numbered in reverse topological order.
// Reverse the numbering at the end for correct order.

Applications: task scheduling (prerequisite ordering), build systems (compile dependencies), course scheduling. The output is a valid sequence where all prerequisites come before their dependents. If a cycle exists (a task depends on itself, directly or indirectly), no topological sort is possible.

Networks — Maximum Flow & Min Cost

▶

A network is a digraph where each edge has capacity and flow. Flow in = flow out at every vertex (except source s and sink t). Goal: maximize flow from s to t.

Max-Flow Min-Cut Theorem: The maximum flow from s to t equals the minimum capacity of any cut separating s from t. A cut is a partition of vertices into two sets with s on one side and t on the other — cut capacity = sum of capacities of edges crossing from s-side to t-side.

Matching

▶

A matching in a graph is a set of edges with no shared vertices. Used for job-applicant assignment, scheduling, resource allocation.

Maximum Matching

Matching with the fewest unmatched vertices. Found by repeatedly finding augmenting paths (paths alternating matched/unmatched edges between two unmatched vertices).

Perfect Matching

All vertices are matched. Exists only if |V| is even and the graph is sufficiently connected.

Bipartite Matching

Graph vertices split into two sets; edges only cross between sets. Hungarian tree (BFS-based) finds augmenting paths efficiently.

Stable Matching (Gale-Shapley, 1962)

Two sets with preference rankings. A matching is unstable if two elements prefer each other over current partners. Algorithm always finds a stable matching — favors proposing set's preferences.

Berge's Theorem (1957): A matching M is maximum if and only if there is no augmenting path in G. This gives us the algorithmic approach: start with any matching, find augmenting paths to grow it, stop when none exist. For non-bipartite graphs, odd cycles (blossoms) complicate the search — Edmonds' blossom algorithm (1965) handles this by contracting odd cycles into single vertices.

Eulerian & Hamiltonian Graphs

▶

Eulerian Trail

Visits every EDGE exactly once. Exists if the graph has exactly 0 or 2 vertices with odd degree. Named for Euler's 1736 Seven Bridges of Königsberg solution.

Eulerian Circuit

Eulerian trail that starts and ends at the same vertex. Exists if and only if ALL vertices have even degree. Fleury's algorithm (1883) finds it.

Hamiltonian Path

Visits every VERTEX exactly once. No simple characterization — determining existence is NP-complete. All complete graphs are Hamiltonian.

Hamiltonian Cycle

Hamiltonian path returning to start. Foundation of the Travelling Salesman Problem (TSP). NP-complete — no efficient exact algorithm known.

Chinese Postman Problem — find shortest closed path visiting every edge (mail delivery analogy). If the graph is Eulerian, the Eulerian circuit is optimal. If not, add minimum-weight duplicate edges to make all vertices even-degree, then find an Eulerian circuit.

Travelling Salesman Problem (TSP) — find minimum Hamiltonian cycle. Exact solution: (n-1)!/2 possible tours. Approximation using minimum spanning tree: preorder traversal of MST gives a tour at most 2× optimal (triangle inequality assumed).

Graph Coloring

▶

Assign colors to vertices so no two adjacent vertices share the same color. Minimum colors needed = chromatic number χ(G). Applications: scheduling (tasks sharing a resource can't run simultaneously), register allocation in compilers, frequency assignment.

Graph TypeChromatic Number
Complete graph Kₙχ = n
Even cycle C₂ₙχ = 2
Odd cycle C₂ₙ₊₁χ = 3
Bipartite graphχ ≤ 2
General graphNP-complete to determine exactly

Approximation algorithms:

Sequential Coloring

Order vertices, then assign the lowest-numbered color that doesn't conflict with neighbors. Simple but result depends heavily on vertex ordering.

Largest-First Ordering

Order vertices by degree (highest first). Gives a better bound: χ(G) ≤ largest degree ordered + 1. Often closer to optimal.

Brélaz Algorithm (DSatur, 1979)

Always color the vertex with the highest saturation degree (most differently-colored neighbors). Ties broken by largest uncolored degree. Often finds optimal coloring in practice.

NP-Complete Problems in Graph Theory

▶

Several important graph problems are NP-complete — no polynomial-time algorithms are known. They are shown NP-complete through reduction: showing that a known NP-complete problem can be transformed into the new problem in polynomial time.

ProblemDescriptionReduced From
Clique ProblemDoes G contain a complete subgraph Kₘ for given m?3-SAT (satisfiability)
3-ColorabilityCan G be colored with only 3 colors?3-SAT via 9-subgraph gadget
Vertex CoverDoes G have a set of ≤k vertices covering all edges?Clique problem (complement graph)
Hamiltonian CycleDoes G have a cycle visiting all vertices exactly once?Vertex cover via 12-subgraph gadget
Travelling SalesmanDoes G have a Hamiltonian cycle of total weight ≤k?Hamiltonian cycle

The chain of reductions: 3-SAT → Clique → Vertex Cover → Hamiltonian Cycle → TSP. Since 3-SAT is the foundational NP-complete problem (Cook's theorem, 1971), and each subsequent problem can be transformed into the next, all inherit NP-completeness. A polynomial solution to ANY of these would imply P=NP and solve all NP problems — the most important open question in computer science.
