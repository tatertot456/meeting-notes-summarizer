Abstract Data Types

▶

An Abstract Data Type (ADT) specifies what values can be stored and what operations can be performed on those values — without specifying how it's implemented internally. Users of an ADT don't need to know the implementation details.

Abstraction captures general characteristics without details. An abstract triangle is "a 3-sided polygon" — the specific type (scalene, isosceles, equilateral) is an implementation detail. This principle drives all of object-oriented programming. Structures are our first step toward creating our own data types.

Structures

▶

A struct groups multiple variables of different types together under one name. It's C++'s way of creating a custom composite data type.

// struct declaration (defines the TYPE — no memory allocated yet)
struct Student
{
    int studentID;
    string name;
    short yearInSchool;
    double gpa;
};                      // semicolon required after closing brace!

// Define variables of the struct type
Student bill;           // now memory is allocated
Student stu1, stu2;

// Initialize at definition
Student s = {11465, "Joan", 2, 3.75};

// Partial initialization — unset members get 0/empty
Student bill2 = {14579};   // only studentID set

// Cannot skip members in the list
// Student bad = {1234, "John", , 2.83};  // ILLEGAL

The struct declaration only defines a new TYPE — like a blueprint. It doesn't create any variables or allocate any memory. You then use that type name (Student) to define actual variables, just like you use int or double. Cannot initialize members inside the struct declaration itself for the same reason.

Accessing Structure Members

▶

Use the dot operator (.) to access individual members of a struct variable. Members can be used like any normal variable of their type.

Student stu1;

// Read into members
cin >> stu1.studentID;
getline(cin, stu1.name);

// Assign to members
stu1.gpa = 3.75;
stu1.yearInSchool = 2;

// Display members — must print each one separately
cout << stu1;              // WRONG — can't print whole struct
cout << stu1.studentID << endl;
cout << stu1.name << endl;
cout << stu1.yearInSchool << " " << stu1.gpa;

// Compare members — cannot compare whole structs directly
if (bill == william)       // WRONG
if (bill.studentID == william.studentID)  // correct

Arrays of Structures

▶

Arrays of structs are a cleaner alternative to parallel arrays — all related data for one entity stays together in one struct.

const int NUM_STUDENTS = 20;
Student stuList[NUM_STUDENTS];   // array of 20 Student structs

// Access: subscript for array element, dot for member
cout << stuList[5].studentID;
stuList[0].gpa = 3.9;
getline(cin, stuList[3].name);

// Process all students with a loop
for (int i = 0; i < NUM_STUDENTS; i++)
{
    cout << stuList[i].name << ": " << stuList[i].gpa << endl;
}

Arrays of structs replace parallel arrays elegantly. Instead of id[5], gpa[5], name[5] all referring to student 5, you have stuList[5].id, stuList[5].gpa, stuList[5].name — all grouped in one struct. Much easier to pass to functions and harder to accidentally misalign.

Nested Structures

▶

A struct can contain another struct as a member — use the dot operator multiple times to reach nested fields.

struct PersonInfo
{
    string name;
    string address;
    string city;
};

struct Student
{
    int studentID;
    PersonInfo pData;    // nested struct member
    short yearInSchool;
    double gpa;
};

// Access nested members: chain dot operators
Student s;
s.pData.name = "Joanne";
s.pData.city = "Tulsa";
s.pData.address = "123 Main St";

cout << s.pData.name << " in " << s.pData.city;

Structures as Function Arguments

▶ 
// Pass individual member
computeGPA(stu.gpa);

// Pass entire struct by value (copy — original unchanged)
void showData(Student stu);
showData(stu1);

// Pass by reference (efficient — no copy, can modify original)
void modifyData(Student &stu);

// Pass by const reference (efficient — no copy, read-only)
void showItem(const Student &stu);   // best for display functions
showItem(stu1);

// Return a struct from a function
Student getStudentData();    // prototype
stu1 = getStudentData();     // call

Student getStudentData()
{
    Student tempStu;         // local struct
    cin >> tempStu.studentID;
    getline(cin, tempStu.pData.name);
    cin >> tempStu.gpa;
    return tempStu;          // return the local struct
}

Passing a struct by value copies ALL its members — expensive for large structs. Pass by reference for efficiency. Pass by const reference when the function only needs to read the struct — you get the speed of reference without risking accidental modification. This is the recommended pattern for display/output functions.

Pointers to Structures

▶

A pointer can hold the address of a struct. Two syntaxes exist for accessing members through a pointer.

Student stu1;
Student *stuPtr;
stuPtr = &stu1;        // point to stu1

// Access member through pointer — must parenthesize the dereference
cout << (*stuPtr).studentID;    // (*ptr).member

// Structure pointer operator: cleaner, preferred syntax
cout << stuPtr->studentID;     // ptr->member (equivalent)
stuPtr->gpa = 3.75;
cin >> stuPtr->yearInSchool;

// Pointer as function parameter
void printStudent(Student *ptr)
{
    cout << ptr->name << " " << ptr->gpa;
}
printStudent(&stu1);

The arrow operator (->) is just shorthand for (*ptr).member. Always use -> when accessing struct members through a pointer — it's cleaner and less error-prone than the parenthesized dereference form. The parentheses in (*stuPtr).member are required because . has higher precedence than * — without them, *stuPtr.studentID would try to dereference the member, not the pointer.

Enumerated Data Types

▶

An enumerated type is a programmer-defined type consisting of named integer constants called enumerators. The compiler assigns integer values starting at 0.

// Declaration
enum Day { MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY };
// MONDAY=0, TUESDAY=1, WEDNESDAY=2, THURSDAY=3, FRIDAY=4

// Define a variable of that type
Day workDay;
workDay = WEDNESDAY;    // assign an enumerator

// Enumerators print as their integer values
cout << MONDAY;      // 0
cout << WEDNESDAY;   // 2
cout << FRIDAY;      // 4

// Can assign enumerator to int
int x = THURSDAY;    // x = 3

// Cannot assign int directly to enum — must cast
workDay = 3;                           // ERROR
workDay = static_cast<Day>(3);         // OK — THURSDAY

// Compare enumerators
if (FRIDAY > MONDAY)   // true (4 > 0)
    cout << "Friday is later\n";

// Declare type and variable in one statement
enum Car { PORSCHE, FERRARI, JAGUAR } sportsCar;

Enumerators are named integer constants — not strings. They make code more readable and self-documenting: workDay = WEDNESDAY is far clearer than workDay = 2. However, they print as integers (2), not as their names ("WEDNESDAY"). You need your own logic to display the name if needed.

Enum Operations, Arrays & Strongly Typed Enums

▶ 
// Using enum as array subscript
enum Day { MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY };
double sales[5];
sales[MONDAY]    = 1525.00;
sales[WEDNESDAY] = 1975.63;

// Looping with enum — cannot use ++ directly, must cast
Day workDay;
for (workDay = MONDAY; workDay <= FRIDAY;
     workDay = static_cast<Day>(workDay + 1))
{
    cout << "Sales for day " << workDay << ": ";
    cin >> sales[workDay];
}

// Math requires casting too
Day day1 = TUESDAY;
Day day2 = static_cast<Day>(day1 + 1);  // WEDNESDAY

Strongly Typed Enums (C++11) — fix the duplicate name problem and add type safety:

// Without strongly typed: ROOSEVELT is ambiguous — ERROR
// enum Presidents { MCKINLEY, ROOSEVELT, TAFT };
// enum VicePresidents { ROOSEVELT, FAIRBANKS, SHERMAN }; // conflict!

// With strongly typed enum class: no conflict
enum class Presidents { MCKINLEY, ROOSEVELT, TAFT };
enum class VicePresidents { ROOSEVELT, FAIRBANKS, SHERMAN };

Presidents prez = Presidents::ROOSEVELT;
VicePresidents vp = VicePresidents::ROOSEVELT;

// Must cast to get integer value
int x = static_cast<int>(Presidents::ROOSEVELT);  // 1

Enumerators must be unique within the same scope — declaring the same name in two regular enums in the same scope causes a compiler error. Strongly typed enum class solves this by requiring the enum name as a prefix (Presidents::ROOSEVELT vs VicePresidents::ROOSEVELT). Prefer enum class in modern C++ code.
