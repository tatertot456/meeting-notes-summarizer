Introduction & Building a Window

▶

GUIs make programs easier to use, allow more functionality, and are more visually appealing. Python provides several GUI libraries — we use tkinter, which is included with Anaconda.

Window

The main interface for the user. Handles menus and base visualizations. Created with tk.Tk().

Widgets

Controls inside the window — buttons, labels, text boxes, frames, etc. Everything the user interacts with.

import tkinter as tk

# Minimal window
window = tk.Tk()
window.mainloop()   # mainloop() = primary event handler
                    # listens for events: button clicks, keypresses, redraws

# Window with attributes
window = tk.Tk()
window.title("A Nicer Looking Window")
window.geometry("300x200")         # width x height in pixels
window.config(bg="#5FB691")        # background color
window.mainloop()

mainloop() is the heartbeat of a tkinter app. It runs continuously, listening for events and redrawing the window when needed. Your program stays alive inside mainloop() — code after it only runs after the window is closed. Always call it last.

Widgets Overview

▶

Classic tkinter widgets:

Label

Displays read-only text or images.

Button

Clickable — triggers an event handler when clicked.

Entry

Single-line text input field.

Text

Multi-line text input area.

Frame

Container for organizing other widgets.

Checkbutton / Radiobutton / Combobox

Selection controls for boolean and multi-choice inputs.

Labels & Buttons

▶ 
import tkinter as tk
window = tk.Tk()

# Label — read-only text
greeting = tk.Label(text="Hello, Tkinter")
greeting.pack()   # pack() binds widget to window, sizes window to fit

# Label with colors and size
label = tk.Label(
    text="Hello, Tkinter",
    fg="white",        # foreground (text) color
    bg="black",        # background color
    width=10,          # in TEXT UNITS (not pixels!)
    height=10          # text units are taller than wide
)
label.pack()

# Button
button = tk.Button(
    text="Click me!",
    width=25,
    height=5,
    bg="blue",
    fg="yellow",
)
button.pack()
window.mainloop()

Width and height in tkinter labels/buttons are measured in TEXT UNITS — not pixels. One text unit horizontally ≈ the width of the character "0" in the current font. Vertically, one unit ≈ the height of one line. This ensures consistent proportions across different platforms and screen resolutions.

Messagebox & Events

▶

A messagebox is a popup window for displaying info, errors, warnings, or questions. Attach functions to buttons using the command attribute.

import tkinter as tk
from tkinter import messagebox

window = tk.Tk()
window.title("Python Message Boxes")
window.geometry("300x200")
window.config(bg="#5FB691")

def msg1():
    messagebox.showinfo("Hello World!", "Hi! Welcome to Python GUIs.")

tk.Button(window, text="Click Me", command=msg1).pack(pady=50)
window.mainloop()
MethodShowsReturns
showinfo()Info messageNone
showwarning()Warning messageNone
showerror()Error messageNone
askyesno()Yes/No questionTrue / False
askquestion()Yes/No question"yes" / "no"
askokcancel()OK/CancelTrue / False
askretrycancel()Retry/CancelTrue / False
askyesnocancel()Yes/No/CancelTrue / False / None

command=msg1 passes the FUNCTION OBJECT — not a call. Don't write command=msg1() with parentheses — that would call the function immediately at setup time and pass its return value (None) as the command. Write command=msg1 to let tkinter call it when the button is clicked.

Entry & Text Widgets

▶ 
import tkinter as tk
from tkinter import messagebox
window = tk.Tk()

# Entry — single line input
label = tk.Label(text="Name")
entry = tk.Entry(fg="yellow", bg="blue", width=50)
label.pack()
entry.pack()
entry.insert(0, "Hello World!")   # pre-fill with text

# Entry methods
def showName():
    name = entry.get()                      # get all text
    messagebox.showinfo("Got", "You typed: " + name)

def helloWorld():
    entry.insert(0, "Hello World!")         # insert at position 0

def testDelete():
    entry.delete(0)                         # delete char at index 0
    # entry.delete(0, tk.END)              # delete everything

# Text — multi-line input
text_box = tk.Text()
text_box.pack()

# Text get — requires start and end positions: "line.charpos"
def showAll():
    content = text_box.get("1.0", tk.END)  # line 1, char 0 to end
    messagebox.showinfo("Got", content)

# Text delete
def deleteRange():
    text_box.delete("1.0", "1.5")          # line 1, chars 0-4
    text_box.delete("1.0", tk.END)         # delete everything

# Text insert
def insertBeginning():
    text_box.insert(1.0, "Hello")          # at line 1, start
def insertEnd():
    text_box.insert(tk.END, "World!")      # at the end

window.mainloop()

Entry.get() needs no arguments — returns the whole line. Text.get() needs "line.charposition" format — "1.0" means line 1, character 0. Lines start at 1 (not 0!), characters start at 0. tk.END is a special constant meaning "the very end of content." Use it whenever you want to operate on everything up to the end.

Frames

▶

Frames are containers that organize other widgets, giving your app a cleaner, more professional layout.

import tkinter as tk window = tk.Tk() # Create a frame and pack it frame_a = tk.Frame(master=window, relief=tk.RAISED, borderwidth=2) frame_a.pack() # Assign widget to a frame using master= label_a = tk.Label(master=frame_a, text="I'm in a frame!") label_a.pack() # Multiple frames — order of pack() determines order on screen frame_1 = tk.Frame(master=window) frame_2 = tk.Frame(master=window) label_1 = tk.Label(master=frame_1, text="First frame") label_2 = tk.Label(master=frame_2, text="Second frame") label_1.pack() label_2.pack() frame_1.pack() # frame_1 appears first (top) frame_2.pack() # frame_2 appears below # Frame relief styles — add visual borders border_effects = { "flat": tk.FLAT, "sunken": tk.SUNKEN, "raised": tk.RAISED, "groove": tk.GROOVE, "ridge": tk.RIDGE } for relief_name, relief in border_effects.items(): frame = tk.Frame(master=window, relief=relief, borderwidth=5) frame.pack(side=tk.LEFT) tk.Label(master=frame, text=relief_name).pack() window.mainloop()

Geometry Managers: pack(), place(), grid()

▶

pack()

Simplest — stacks widgets top to bottom by default. Good for quick layouts. Use fill and side for more control.

place()

Pixel-precise positioning from top-left origin (0,0). Full control but fragile — layouts break if window resizes.

grid()

Best for most apps — rows and columns like a spreadsheet. More work but most flexible and predictable.

## PACK frame1.pack() # stack vertically (default) frame1.pack(fill=tk.X) # fill full width frame1.pack(fill=tk.Y, side=tk.LEFT) # fill full height, left column frame1.pack(fill=tk.BOTH, side=tk.LEFT, expand=True) # expand in both directions ## PLACE — pixel coordinates from top-left (0,0) label1.place(x=0, y=0) # top-left corner label2.place(x=75, y=75) # 75 pixels right and down ## GRID — row/column layout for i in range(3): for j in range(3): frame = tk.Frame(master=window, relief=tk.RAISED, borderwidth=1) frame.grid(row=i, column=j, padx=5, pady=5) label = tk.Label(master=frame, text=f"Row {i}\nColumn {j}") label.pack(padx=5, pady=5) # Configure column/row resize behavior window.columnconfigure(i, weight=1, minsize=75) window.rowconfigure(i, weight=1, minsize=50) # Sticky — align widget within its cell (compass directions) label1.grid(row=0, column=0, sticky="ne") # top-right label2.grid(row=1, column=0, sticky="sw") # bottom-left label3.grid(row=0, column=0, sticky="nsew") # fill entire cell

Never mix geometry managers (pack, place, grid) within the same container — it causes unpredictable layouts and can freeze your app. You CAN use different managers in different frames. grid() is usually the best choice for professional layouts because it gives the predictability of place() with the flexibility of pack().

Events & Event Handlers

▶

mainloop() maintains a queue of all events. Widgets fire events (clicks, keypresses). Attach event handlers with command= or bind().

import tkinter as tk window = tk.Tk() # Counter app — demonstrates event handlers modifying widget state def increase(): value = int(lbl_value["text"]) lbl_value["text"] = f"{value + 1}" # update label text directly def decrease(): value = int(lbl_value["text"]) lbl_value["text"] = f"{value - 1}" btn_decrease = tk.Button(master=window, text="-", command=decrease) btn_decrease.grid(row=0, column=0, sticky="nsew") lbl_value = tk.Label(master=window, text="0") lbl_value.grid(row=0, column=1) btn_increase = tk.Button(master=window, text="+", command=increase) btn_increase.grid(row=0, column=2, sticky="nsew") window.mainloop() # Two ways to attach event handlers: # 1. command= attribute at widget creation btn = tk.Button(text="Click", command=my_handler) # 2. bind() method — more powerful, can handle any event btn.bind("<Button-1>", my_handler) # left mouse click entry.bind("<Return>", on_enter) # Enter key pressed

Event handlers modify widget state by updating their properties directly (lbl["text"] = "new"). This triggers a redraw.
