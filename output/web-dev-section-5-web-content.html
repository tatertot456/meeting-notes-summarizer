Lists — Unordered & Ordered

▶

Lists are fundamental content structures. Use CSS's list-style-type property rather than relying on browser defaults.

<!-- Unordered list -->
<ul>
    <li>Get dressed</li>
    <li>Eat a good breakfast</li>
    <li>Take an apple for my professor</li>
</ul>

<!-- Ordered list -->
<ol>
    <li>God the Father</li>
    <li>Jesus, His Son</li>
    <li>The Holy Spirit</li>
</ol>
PropertyUnordered ValuesOrdered Values
list-style-typedisc, circle, square, nonedecimal, upper-alpha, lower-alpha, upper-roman, lower-roman

Descendant selectors for nested list styling:

ul ul     { list-style-type: square; } /* second level */
ul ul ul  { list-style-type: none; }   /* third level */

/* Descendant selector: element inside another */
p strong  { color: red; }   /* strong inside a paragraph */

/* Child selector: immediately inside (not just any descendant) */
section > h2 { background-color: palegreen; }
aside > h3   { margin: 0; }

The only element allowed immediately inside <ul> or <ol> is <li> — no plain text or other elements directly inside. Always specify list-style-type explicitly rather than relying on browser defaults which vary by nesting level across browsers.

Organizational & Structural Elements

▶

Organizational elements group content into meaningful sections without obvious visual manifestation. All are block elements by default — they span the full width of their container.

section

Groups a section of a page. Typically contains a heading and one or more paragraphs.

article

Groups one or more sections. Forms an independent part of the page or website.

aside

Content related to but not part of the main flow. Position with float and width CSS properties.

nav

Groups navigation links. Use with <a> elements and href attributes.

header

Groups header content at the top of the page or a section.

footer

Groups footer content. Often contains address, contact info, and copyright.

<nav>
    <a href="#dining">Dining</a>
    <a href="#hours">Hours</a>
</nav>

<section>
    <h2 id="dining">Dining</h2>
    <p>Great food here...</p>
</section>

<footer>
    <address display="inline">123 Main St</address>
</footer>

/* display: inline overrides address element's block default */
address { display: inline; }

User agent style sheets are the browser's built-in defaults — they form the lowest priority in the CSS cascade. The address element displays as block by default — use display: inline to override this. In Blazor, the <a> anchor technique for page navigation requires JavaScript interop — it doesn't work the same way as in plain HTML.

Figures group self-contained content with optional captions:

<figure>
    <img src="images/photo.png" alt="Description of photo">
    <figcaption>The Final HTML Tag</figcaption>
</figure>
<!-- figcaption MUST be inside figure -->
<!-- alt attribute is REQUIRED by HTML5 standard -->

Data Tables

▶

Use tables ONLY for tabular data — not for page layout. HTML5 is explicit: table elements are for data, CSS is for layout.

<table>
    <caption>My Semester Course Schedule</caption>
    <thead>
        <tr>
            <th>Course #</th>
            <th>Course Name</th>
            <th>Classroom</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>COMP2040</td>
            <td>Web Development</td>
            <td>Room 101</td>
        </tr>
    </tbody>
</table>

/* Table styling */
table, th, td { border: thin solid; }
th { text-align: center; font-weight: bold; }  /* default */
td { text-align: left; }                         /* default */

Cell spanning:

<th colspan="2">Events</th>   /* spans 2 columns */
<th rowspan="2">Mesozoic</th> /* spans 2 rows    */

Center a table horizontally:

/* Apply to the PARENT container, not the table itself */
body { display: flex; justify-content: center; }

/* In Blazor, wrap in a div */
<div style="display:flex; justify-content:center;">
    <table>...</table>
</div>

thead/tbody separate header rows from data rows — use them when you have column headers at top AND row headers at left. The vertical-align property positions text vertically within a cell. Default: th = center/bold, td = left/normal, both have zero padding. Use CSS to override all of these.

Layout with CSS — display, flex, position

▶

CSS display property with table values — for layouts that grow/shrink with content:

.table   { display: table; border-spacing: 20px; }
.caption { display: table-caption; text-align: center; }
.row     { display: table-row; }
.row > * { display: table-cell; text-align: center; }

/* Usage */
<div class="table">
    <div class="caption">Ancient Wonders</div>
    <div class="row">
        <span><img src="pyramid.jpg" alt=""></span>
        <span><img src="gardens.jpg" alt=""></span>
    </div>
</div>
/* Use span wrappers for images — display:table-cell
   doesn't work well with embedded elements directly */
/* border-spacing adds space OUTSIDE cells */
/* padding adds space INSIDE cells          */
/* margin does NOT work with table cells    */

Flex container — the preferred Blazor approach:

/* A div with display:flex is a flex container */
<div style="display:flex;">
    <div>Item 1</div>
    <div>Item 2</div>
    <div>Item 3</div>
</div>
/* Flex components align horizontally (row) or vertically (column) */
/* Blazor's rendering engine handles resizing automatically        */
/* Much easier to add/remove items than HTML tables or div tables  */

CSS position: absolute — for fixed-position layouts:

.sidebar {
    position: absolute;
    left: 20px;   /* distance from containing block's left edge */
    top: 100px;   /* distance from containing block's top edge  */
}
/* position:absolute removes the element from normal flow */
/* Normal flow: block elements top-to-bottom, phrasing left-to-right */

Links & Navigation

▶ 
<!-- Basic link -->
<a href="http://www.nnu.edu">Northwest Nazarene University</a>

<!-- Link to same server (relative URL) -->
<a href="../hw2/weather.html">Weather Page</a>

<!-- Link within current page (anchor) -->
<a href="#pledge-drive">Pledge Drive</a>
<h3 id="pledge-drive">Pledge Drive</h3>

<!-- Download link -->
<a download href="/files/catalog.pdf">Download Catalog</a>

<!-- Open in new tab -->
<a href="https://youtube.com" target="_blank">YouTube</a>
<!-- target="_self" = overlay current page (default)    -->
<!-- target="_blank" = open in new tab                  -->

CSS for links:

a:link    { color: #00ff00; }   /* unvisited */
a:visited { color: #ff0000; }   /* visited   */
a         { text-decoration: none; }      /* remove underline */
a:hover   { text-decoration: underline; } /* re-add on hover  */
/* hover is more specific than a, so it overrides */

Relative URL rules: use / to separate directories, use .. to go up to parent directory. Path-absolute URLs start with / and are relative to the web server's root.

In Blazor, anchor navigation (jumping within a page) requires JavaScript interop — inject IJSRuntime, create onclick events, add a JS file under wwwroot, and reference it in App.razor. It's significantly more work than plain HTML. Always include an index.html in every directory that might be specified in a URL — without it, users see either directory listings or error pages.

Images & Image Formats

▶ 
<img src="/images/photo.png"
     alt="Description of photo"
     width="400" height="300">
<!-- alt is REQUIRED by HTML5 -- used by screen readers -->
<!-- Specify natural width/height for faster download    -->
<!-- Mismatched w/h degrades image quality              -->

<!-- Float image to position text around it -->
<img src="photo.jpg" alt="photo"
     style="float: left; margin: 10px;">

<!-- Shortcut icon (favicon) in head section -->
<link rel="icon" href="../images/tree.png">
<!-- In Blazor, use <HeadContent> element instead -->
FormatBest ForNotes
GIF (.gif)Line drawings, icons, cartoons8-bit (256 colors). Small files. Transparency: 2 levels only.
JPEG (.jpg)Photographs24-bit (16M colors). Lossy compression. Quality vs size tradeoff.
PNG (.png)Both photos & limited-colorFlexible bit depth. 256 transparency levels. Open source.
SVG (.svg)Icons, logos, diagramsVector — infinite scaling, no pixelation. XML-based. Not for photos.

Bitmap images (GIF, JPEG, PNG) store a color per pixel — resizing adds/drops pixels and degrades quality. Vector images (SVG) are mathematical formulas — scaling redraws with no quality loss. Use <img> with an SVG src, or embed SVG code directly using an <svg> element. iframe elements display embedded web documents — useful for photo galleries where clicking thumbnails shows larger versions.

Audio & Video

▶

HTML5 provides native audio and video elements — no plugins required.

<!-- Audio -->
<audio src="/audio/sound.mp3" preload="auto" controls></audio>

<!-- Multiple sources for browser compatibility -->
<audio autoplay controls loop>
    <source src="/audio/rain.mp3" type="audio/mpeg">
    <source src="/audio/rain.wav" type="audio/wav">
    <source src="/audio/rain.ogg" type="audio/ogg">
</audio>
<!-- Include .mp3, .wav, .ogg for maximum compatibility -->

<!-- Video -->
<video width="397" height="225"
       preload="metadata" controls
       poster="/images/thumbnail.jpg">
    <source src="/video/cats.mp4" type="video/mp4">
    <source src="/video/cats.webm" type="video/webm">
</video>
<!-- Include .mp4 and .webm for maximum compatibility -->

<!-- Alternative: embed YouTube via iframe -->
<iframe width="400" height="225" allowfullscreen
    src="//www.youtube.com/embed/VIDEO_ID"></iframe>
AttributeEffect
controlsShows play/pause/volume controls
autoplayStarts playing on page load
loopRepeats continuously
preload="auto"Downloads file on page load
preload="metadata"Downloads only dimensions, duration, first frame
poster (video)Image shown while video isn't playing

Quiz answers: controls attribute shows the play/pause/volume control bar. Default background image behavior: tiles multiple copies to fill the viewport. The alt attribute provides fallback text read by screen readers and displayed when images fail to load — required by HTML5.

Background Images

▶ 
/* HTML/CSS approach */
body {
    background-image: url(../images/pumpkin.jpg);
    background-size: cover;        /* scale to fill viewport */
    background-attachment: fixed;  /* don't scroll with page */
    background-position: center;   /* center horizontally    */
}
/* cover + fixed: scales image so one dimension fills viewport
   exactly, crops the other — viewport is fully covered       */
/* Without background-size: images tile by default            */

/* Blazor approach: wrap page in a div */
<div style="background-image: url(./images/bg.jpg);
             background-size: cover;
             background-attachment: fixed;
             background-position: center;
             min-height: 100vh;">
    @Body
</div>

In Blazor, applying background-image to body through a global CSS rule affects all pages. For page-specific backgrounds, wrap the page content in a div and apply the background there. Pick background images that won't blend in with or hide your page's text — contrast is critical for readability.

Web Fonts

▶

Standard fonts (Tahoma, Arial, Courier New) are safe to use directly. Non-standard fonts must be delivered with an @font-face at-rule — the server sends the font file alongside the page.

/* In an external .css file (NOT in a .razor file —
   @ is reserved for C# code in Razor pages) */

@font-face {
    font-family: Buffied;
    src: url(/fonts/Buffied.woff2) format("woff2"),
         url(/fonts/Buffied.woff)  format("woff");
}

h1, h3 { font-family: Buffied, serif; }

In Blazor, put @font-face rules in a .razor.css or external .css file — never in a .razor file because @ is reserved for C# code. Prefer WOFF2 over WOFF (30% smaller files) and both over TrueType (.ttf). Include both WOFF and WOFF2 for compatibility with older browsers. Use a file converter (font-converter.net) to generate WOFF/WOFF2 from TTF files.
