Course Overview

▶

COMP2040 is a 3-credit course focused on designing and building professional web pages. It is the first of a 3-semester full-stack programming course — not a simple web design class. This is a programming course.

  • Focus: designing and building professional-looking web pages
  • Combination of tests, homework assignments, and a project
  • Weekly pop quizzes posted for exactly 2 minutes at the start of class — cannot be made up
  • Canvas overrides the syllabus when there is a conflict
  • Visual appeal may factor into grades, but technical skill is most important

Everyone who fails this class does so due to failure to complete assignments. DO YOUR HOMEWORK. Treat this class like a job — show up on time and do the work when it's supposed to be done.

The Web Stack

▶

Modern websites use a combination of technologies. This course covers all three layers, though the emphasis shifts toward frameworks rather than raw HTML/CSS/JS:

HTML5

Objects and components for input and output. The structure of every web page — defines what's on the page.

CSS

Custom configuration for maintenance, reuse, and visual appeal. Controls how the page looks.

JavaScript

Interactivity and communication with external resources. Makes pages dynamic — in Blazor, we replace this with C#.

Blazor / .NET

The framework we study. Contains prebuilt components that make development faster. Lets us use C# instead of JavaScript.

This course emphasizes frameworks over raw HTML/CSS/JS. Most professional websites aren't built from scratch — they use frameworks. Knowing Blazor makes you more employable and demonstrates a more advanced skill level than basic HTML/CSS alone. Think power tools vs. hand tools.

Why Blazor Instead of Plain HTML?

▶

This course deliberately focuses less on generic HTML tables/lists and more on grids, charts, flex containers, and application programming patterns. Here's why:

  • Employers value framework knowledge over basic HTML/CSS/JS
  • Shows advanced skill level and greater productivity potential
  • People who do regular HTML use tools like WordPress anyway
  • This is a CS course — the focus is on advanced systems programming, design, and architecture
  • Blazor lets us write C# instead of JavaScript, keeping the language consistent with the rest of your CS coursework

We explore the Blazor/.NET framework rather than building pages from scratch. The goal is web APPLICATION programming, not basic web page construction. This distinction matters — it's the difference between knowing how to build something and knowing how to build it professionally at scale.

The Project

▶

The main deliverable is a website supporting an online pet store and personal profile. The instructor provides back-end web and database services while you build the front-end pages.

  • Multiple iterations throughout the semester — first due by end of week 2
  • Each iteration must include specified milestones
  • You're welcome to add your own features on top of the requirements
  • Work alone preferred — ask for help if stuck (TA, instructor, other students)
  • Do NOT share code

Best Website Contest: TAs and class vote on the best site. Winners receive Amazon gift cards ($50 / $25 / $10) plus 5 extra grade points. Be creative and have fun — but don't count on this to compensate for weak overall performance.

Why Web Development?

▶

What's great about it:

  • No headaches from program updates — deploy once, everyone gets the latest version
  • Little to install on the user's end
  • Works across multiple operating systems and browsers
  • Centralized storage of multimedia
  • Your audience can be as small as one person or as large as the entire internet (~6 billion connected users)

What's challenging about it:

  • Stateless programming — generally slower, requires workarounds
  • Interactive/event-driven programming is trickier than desktop
  • Visuals are more important — users judge quickly
  • Security is significantly more complex — anyone with a code editor and internet access can attempt to exploit your site
  • Visual design tooling still lags behind desktop development
