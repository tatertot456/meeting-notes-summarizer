What Is a Query?

▶

Queries are fundamental GIS analysis techniques that extract a subset of features or records based on a condition. Three types:

Interactive (Manual) Query

User visually identifies desired features by clicking or drawing a selection shape on the map. Best when you know where the feature is. Fastest for simple, visual selection.

Attribute Query

Expression tests table values: counties with population > 100,000. Uses SQL (Structured Query Language). Finds records meeting specified numeric or text conditions.

Spatial Query

Tests spatial relationships between two layers: cities within 50 miles of a volcano. Uses spatial operators (intersect, contains, proximity). Evaluates geometry, not attribute values.

Operator types used in GIS queries:

  • Relational operators: =, >, <, ≥, ≤ — test conditions, return True or False
  • Logical operators: AND, OR, XOR, NOT — combine True/False conditions
  • Spatial operators: intersect, contains, within, proximity — test geometric relationships

Attribute Queries & SQL

▶

Attribute queries use expressions applied to a table to find records meeting one or more conditions. Executed using SQL (Structured Query Language). ArcGIS Pro provides the Clause Builder — an intuitive interface that converts simple input into SQL automatically.

-- Simple attribute query examples:
STATE = 'NJ'            -- text values in single quotes
Population > 100000     -- numeric values without quotes
GPA >= 3.0 AND MAJOR = 'Geography'
RockType LIKE '%stone%' -- partial match: limestone, sandstone, etc.

-- Clause Builder intuitive input:
"RockType" contains the text "stone"
↓ converts to SQL:
RockType LIKE '%stone%'
-- The % wildcard means "any characters can appear here"

The % wildcard in LIKE queries matches any characters on either side of the search term. '%stone%' finds limestone, sandstone, brownstone, and any other value containing "stone" anywhere. Use the SQL button in the query builder to see and directly edit the underlying SQL expression.

Boolean Logic & Order of Precedence

▶
OperatorReturns True WhenExample
ANDBOTH conditions are trueMAJOR = 'Geography' AND STATE = 'NY' → Geography majors from NY only
OREITHER or BOTH conditions are trueSTATE = 'NY' OR STATE = 'NJ' → students from either state
XORONE condition is true but NOT bothTrue for NY-only OR NJ-only, but NOT both simultaneously
NOTFirst condition true AND second condition falseSTATE = 'NJ' NOT MAJOR = 'Geography' → NJ students who aren't Geography majors

STATE = 'NJ' AND STATE = 'NY' returns NOTHING — a field cannot hold two values simultaneously. Use OR when testing the same field for different values. AND, OR, and XOR are commutative (A AND B = B AND A). NOT is NOT commutative — order matters.

Order of precedence — evaluated left to right unless parentheses override:

Example 1: ZONE = 5 OR ZONE = 6 AND OWNER = 'Public'
Evaluates as: ZONE = 5 OR (ZONE = 6 AND OWNER = 'Public')
→ Selects parcels 5,6,7,8

Example 2: (ZONE = 5 OR ZONE = 6) AND OWNER = 'Public'
→ Selects parcels 1,2,5,6,7,8

Use parentheses to control evaluation order and make intent explicit!

Venn diagrams help visualize Boolean logic. The whole table T is the outer rectangle. Circle A = records meeting condition A. Circle B = records meeting condition B. Overlap = records meeting both. Shaded region = what the expression selects.

Spatial Queries

▶

Spatial queries test geometric relationships between two layers using the Select Layer By Location tool. The target layer is what gets selected; the source layer is what you compare against.

Spatial OperatorTests ForExample
IntersectFeatures touch or overlapCounties that touch (intersect) rivers
ContainsFeature is fully inside another (boundary counts)Counties contained by Oregon (can share state boundary)
Completely ContainsFeature is fully inside, no shared boundaryCounties completely within Oregon (no boundary sharing)
WithinFeature is inside the source featureCities within a county
Proximity (Within Distance)Feature is within a specified distanceCities within 50 miles of a volcano

Spatial queries select ENTIRE features — you cannot select only the part of a river inside Texas. "Select rivers that intersect Texas" returns the full length of each river, including sections outside Texas. "Select rivers within Texas" returns nothing because no river is entirely within Texas. To isolate the portions inside a boundary, use Clip or Intersect geoprocessing tools instead.

Data quality affects spatial queries:

  • Poor logical consistency — Congressional district boundaries that don't align with county boundaries cause contains/within queries to fail unexpectedly
  • Poor accuracy/precision — generalized points and lines for Minneapolis and the Mississippi River may not intersect in the data even though they do in reality. Use a search distance to compensate.

Buffers & Extraction Functions

▶

Buffers delineate the area within a specified distance of features. They are a type of spatial query result stored as a new polygon layer.

  • Drug-free zones: 100m ring buffers around schools
  • Logging restrictions: buffers around streams
  • Hazard zones: 50-mile buffers around active volcanoes

By default, each input feature gets its own buffer — overlapping buffers produce inflated area calculations. Use the DISSOLVE option when creating buffers to merge overlapping buffers into a single, clean region. Always check whether dissolved or individual buffers are appropriate for your analysis.

Extraction functions — create new datasets by spatially cutting:

Clip

Cookie cutter — keeps features INSIDE the boundary polygon. Roads clipped to city limits = only roads inside the city. Creates a new dataset.

Erase

Keeps features OUTSIDE the boundary polygon. Opposite of clip. Roads erased by city limits = only roads outside the city. Creates a new dataset.

After clipping or erasing, features that crossed the boundary now have different lengths or areas. Update any user-defined length or area fields using the Calculate Geometry Attributes tool.

Selection Methods & The Logical Two-Step

▶

Most geoprocessing tools honor the selected set — they operate only on selected features, not the whole layer. This allows queries to be chained and combined. Use List By Selection in Contents pane to control which layers are selectable and see counts of selected features.

Selection TypeEquivalent ToEffect
New Selection(replaces)Drops existing selection, creates new one
Add to SelectionORAdds newly found features to existing selection
Remove from SelectionNOTRemoves newly found features from existing selection
Select Subset from SelectionANDKeeps only features in BOTH old and new selections
Switch Selection(invert)Trades selected for unselected — selects everything NOT currently selected
Clear Selection(deselect all)Removes all selections

The Logical Two-Step — combining attribute and spatial queries using selection types. Example: Find earthquakes in Mineral County, Nevada with magnitude > 5.0:

Step 1: Select Layer By Location (Spatial Query)
→ New Selection: Select earthquakes that INTERSECT Mineral County
→ Result: All earthquakes in the county selected (green)

Step 2: Select Layer By Attribute (Attribute Query)
→ Change selection type to: "Select Subset from Current Selection" (= AND)
→ Expression: Magnitude > 5.0
→ Result: Only earthquakes in the county WITH magnitude > 5.0 selected

This is equivalent to: (IN Mineral County) AND (Magnitude > 5.0)
Can mix spatial and attribute queries using selection types!

Managing Query Results

▶

Several ways to preserve selected features for future use:

Make Feature Layer Tool

Creates a new temporary layer based on an attribute query. Optionally change which fields are shown. The new layer can be symbolized differently and used as input to other tools.

Make Layer from Selected Features

Menu option: right-click layer → Selection → Make Layer from Selected Features. Works for both attribute and spatial selections. Creates a temporary in-memory layer.

Definition Query

Set a persistent attribute filter on a layer in its properties. The layer only displays and participates in analysis with the filtered records. Survives project saves.

Export Selected Features

Saves selected features as a new permanent dataset. Most permanent option. Useful when you need a standalone copy of the subset for sharing or further analysis.

Interactive selection (clicking on features) is fastest when you know where something is. For reproducible, documented analysis, use attribute queries (Select Layer By Attribute) or spatial queries (Select Layer By Location) — these can be saved as tools, automated, or repeated consistently. Export to a new dataset when you need a permanent, shareable result.
