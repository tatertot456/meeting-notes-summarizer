K-Means Clustering

▶

K-Means partitions a dataset of n objects into k clusters by minimizing the sum of squared distances to cluster centroids.

Algorithm:

  • Partition objects into k non-empty subsets arbitrarily
  • Compute centroid (mean point) for each cluster
  • Reassign each object to the cluster with the nearest centroid
  • Repeat until no objects change clusters

Strengths

Efficient: O(tkn) where t=iterations, k=clusters, n=objects. Normally k,t << n.

Weaknesses

Only for continuous n-dimensional space. Must specify k in advance. Sensitive to outliers. Can't find non-convex clusters.

K-Means often terminates at a local optimum, not the global one. Multiple restarts with different initial centroids can help.

Variations: K-modes for categorical data (uses frequency-based modes). K-prototype for mixed categorical + numerical data.

K-Medoids (PAM)

▶

Instead of using the mean, K-Medoids uses the most centrally located actual object as the cluster representative — making it more robust to outliers.

PAM (Partitioning Around Medoids):

  • Arbitrarily choose k objects as initial medoids
  • Assign remaining objects to nearest medoid
  • Randomly select a non-medoid object and compute cost of swapping
  • If swap improves total cost, perform it
  • Repeat until no improvement

PAM

Works effectively for small datasets. O(k(n-k)²) — doesn't scale to large datasets.

CLARA

PAM on samples — scales better by working on subsets of data.

CLARANS

Randomized re-sampling — further improves scalability.

vs. K-Means

K-Medoids applies to wider range of data types and is more robust to outliers, but slower.

Genetic Algorithms

▶

Genetic Algorithms (GAs) are based on an analogy to biological evolution — survival of the fittest rules.

  • An initial population of randomly generated rules is created
  • Rules are encoded as bit strings (one-hot encoding for multi-value attributes)
  • Fitness = classification accuracy on training examples
  • New generations formed via crossover and mutation
  • Continue until rules satisfy a prespecified accuracy threshold

GAs are slow but easily parallelizable — each individual in the population can be evaluated independently on separate processors.

Fuzzy Sets & Rough Sets

▶

Fuzzy Logic

Uses truth values between 0.0 and 1.0 to represent degree of membership. Example: $49K income belongs to "medium" with value 0.15 and "high" with value 0.96. Values don't need to sum to 1.

Fuzzy Rules

Define fuzzy expressions using AND = min(A,B), OR = max(A,B), NOT = 1-A. Defuzzification applies the rule with the maximum expression value.

Rough Sets

Approximate class membership using lower approximation (certainly in C) and upper approximation (cannot be excluded from C).

Feature Reduction

Finding minimal subsets of attributes (reducts) is NP-hard. Discernibility matrix reduces computation by storing differences between attribute values for each pair of tuples.

Real-World Applications: Remote Sensing & Burn Severity

▶

Hamilton's research applied ML to mapping wildfire burn severity using drone and satellite imagery:

  • Challenge: Mixed pixels at 30m Landsat resolution contain multiple land cover classes
  • Solution: Train SVM at 5cm drone resolution, use Fuzzy Control to generate 30m training data
  • Fuzzy Control Pipeline: Capture 5cm class density → Fuzzify inputs → Apply fuzzy rules → Defuzzify → Label 30m pixels
  • Result: SVM more accurate than kNN; 5cm resolution significantly better than 30m (Burn Extent p=0.018, Consumption p=0.006)

Published: Hamilton et al. (2020) — "Mapping Burn Extent of Large Wildland Fires from Satellite Imagery Using Machine Learning Trained from Localized Hyperspatial Imagery." Remote Sensing, 12(24), 4097.

Active Learning & Additional Topics

▶

Active Learning

Query a human oracle for labels on the most informative unlabeled tuples. Goal: high accuracy with as few labeled examples as possible. Strategy: uncertainty sampling — query least certain tuples.

Transfer Learning

Extract knowledge from source tasks and apply to a target task. Instance-based: reweight source data. Negative transfer: when source knowledge hurts target performance.

Contrastive Learning

SimCLR and FireCLR use data augmentation to learn representations without labels. Augmentations: random crop, Gaussian blur, flip, rotation, grayscale/color change.

Reinforcement Learning

Q-Learning: Q(s,a) estimates expected reward. Update rule balances current reward with future expected rewards. Deep RL replaces Q-table with neural network weight matrices.
