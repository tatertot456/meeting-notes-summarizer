Relational Operators

▶

Relational operators compare values and produce a Boolean result — true (1) or false (0).

OperatorMeaningExampleResult
>Greater than12 > 5true
<Less than7 < 5false
>=Greater than or equal5 >= 5true
<=Less than or equal7 <= 5false
==Equal tox == 10true (if x is 10)
!=Not equal tox != 8true (if x is 10)
// Relational expressions can be assigned to variables
result = x <= y;   // stores 0 or 1

Never confuse = (assignment) with == (equality test). Writing if (x = 5) assigns 5 to x and always evaluates as true — a very common bug. Always use == inside conditions.

The if Statement

▶

The if statement conditionally executes or skips a statement based on whether an expression is true or false.

// Single statement
if (score > 90)
    grade = 'A';    // executed only if score > 90

// Block of statements using { }
if (score > 90)
{
    grade = 'A';
    cout << "Good Job!\n";
}

Important rules: never put a semicolon after the condition (if (score > 90); would make the if do nothing). Use curly braces {} any time you need more than one statement to execute. 0 is false; any nonzero value is true. Be careful comparing floats/doubles for equality — floating point rounding can cause unexpected results.

The if/else Statement

▶

if/else provides two execution paths — one when the condition is true, another when it's false. Exactly one path always executes.

if (expression)
    statement1;   // executes when true
else
    statement2;   // executes when false

// Example: check if number is even or odd
if (number % 2 == 0)
    cout << "Even\n";
else
    cout << "Odd\n";

Nested if & if/else if

▶

Nested if — an if statement inside another if statement. Used to test multiple conditions in sequence.

if/else if — tests a series of conditions until one is true. Cleaner than deeply nested ifs for multiple alternatives.

// if/else if chain
if (score >= 90)
    grade = 'A';
else if (score >= 80)
    grade = 'B';
else if (score >= 70)
    grade = 'C';
else if (score >= 60)
    grade = 'D';
else
    grade = 'F';   // trailing else catches everything else

The trailing else at the end of an if/else if chain is optional but highly recommended — use it to catch invalid input or unexpected values. Always use consistent indentation so the structure is visually clear. Flags (bool variables) are useful for signaling conditions: set them before the if chain, test them inside.

// Flag example
bool isValid = false;
if (score >= 0 && score <= 100)
    isValid = true;

if (isValid)
    cout << "Processing score...\n";

Logical Operators

▶
OperatorMeaningTrue when...
&&ANDBoth sides are true
||ORAt least one side is true
!NOTThe operand is false
int x = 12, y = 5, z = -4;

// AND: both must be true
if (x > 0 && y > 0)       // true (both positive)
if (x > 0 && z > 0)       // false (z is negative)

// OR: at least one must be true
if (x > 0 || z > 0)       // true (x is positive)

// NOT: reverses true/false
if (!( x == y))            // true (x does not equal y)

// Range testing
if (grade >= 0 && grade <= 100)    // inside range
    cout << "Valid grade";

if (grade < 0 || grade > 100)     // outside range
    cout << "Invalid grade";

// WRONG — mathematical notation doesn't work in C++
if (0 <= grade <= 100)    // always true! Don't do this.

Precedence: ! has highest precedence, then &&, then ||. Short-circuit evaluation: for &&, if the left side is false, the right side is never evaluated. For ||, if the left side is true, the right side is never evaluated. This matters when the right side has side effects or could cause errors (like division by zero).

Comparing Characters & Strings

▶

Characters are compared by their ASCII values. Uppercase letters (A=65) have lower ASCII values than lowercase (a=97), so 'a' > 'Z'.

// Character comparison
'A' < 'B'    // true (65 < 66)
'1' < '2'    // true (49 < 50)
'a' > 'Z'    // true (97 > 90)

String objects are also compared character by character using ASCII values:

string name1 = "Mary";
string name2 = "Mark";

name1 != name2   // true  (y != k)
name1 > name2    // true  ('y' > 'k')
name1 <= name2   // false
name1 < "Mary Jane"  // true (Mary is a prefix of Mary Jane)

String comparison is case-sensitive and goes character by character. "Mary" and "mary" are NOT equal. Two strings are equal only when every character matches exactly, including length.

The Conditional Operator

▶

The conditional operator ?: is a compact if/else expression — the only ternary operator in C++ (takes 3 operands).

// Format: condition ? value_if_true : value_if_false
x < 0 ? y = 10 : z = 20;

// Common use: assign based on condition
int abs_val = (x < 0) ? -x : x;

// Use in cout
cout << (score >= 60 ? "Pass" : "Fail");

The conditional operator is useful for short, simple decisions — especially inline in expressions or cout statements. For anything more complex, a full if/else is clearer and easier to maintain. Parentheses may be needed due to the low precedence of the conditional operator.

The switch Statement

▶

switch selects among multiple alternatives based on the value of an integer (or char) expression. It's a natural fit for menu-driven programs.

switch (expression)   // must be integer or char
{
    case 1:
        cout << "One\n";
        break;           // exit the switch — required!
    case 2:
        cout << "Two\n";
        break;
    case 3:
        cout << "Three\n";
        break;
    default:             // optional but recommended
        cout << "Invalid choice\n";
}

The break statement is critical — without it, execution "falls through" into the next case and keeps running until it hits a break or the end of the switch. This is almost never what you want. Case values must be constant integer expressions or literals — variables and ranges are not allowed. default catches anything that doesn't match any case.

// switch for a menu system
cout << "1. Add\n2. Subtract\n3. Quit\n";
cin >> choice;
switch (choice)
{
    case 1: doAdd(); break;
    case 2: doSubtract(); break;
    case 3: cout << "Goodbye\n"; break;
    default: cout << "Invalid selection\n";
}

Scope & Input Validation

▶

The scope of a variable is the block {} in which it is defined — from the point of definition to the closing brace. A variable cannot be used outside its scope.

int main()
{
    int x = 10;         // x is in scope for all of main

    {                   // inner block
        int y = 20;     // y only exists inside this block
        cout << x;      // ok — x is still in scope
        cout << y;      // ok
    }

    cout << y;          // ERROR — y is out of scope
}

You CAN define a variable with the same name in an inner block as in an outer block. Inside the inner block, the outer variable becomes inaccessible — the inner one shadows it. This compiles but is confusing and not recommended. Always give variables unique, descriptive names.

Input validation — always check that user input is acceptable before processing it. Test for range, reasonableness, valid menu choices, and potential divide-by-zero conditions. Bad output from bad input ("garbage in, garbage out") is one of the most common program failures.

// Validate before using
cout << "Enter a score (0-100): ";
cin >> score;
if (score < 0 || score > 100)
    cout << "Error: score out of range\n";
else
    // process valid score
