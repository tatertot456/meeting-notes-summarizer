Map Design Process

▶

Map design is an iterative five-step process. Every step refers back to the map's objective — the objective is the anchor for all decisions.

1. Select

Choose which data layers tell the story. Identify lead players, supporting layers, and distracting layers to omit.

2. Arrange

Position the map and all supporting elements (title, legend, scale bar) on the layout. Apply visual hierarchy principles.

3. Symbolize

Choose symbols, colors, and classification methods appropriate for the data type and audience. Match visual variables to data meaning.

4. Review

Evaluate the map against the objective. Does it tell the story clearly? Is anything missing, misleading, or distracting?

5. Edit

Make refinements based on review. Repeat as needed — map design is iterative, not linear.

Map Objective — Four Key Questions

▶

Determining the objective is the most important first step. Answer these four questions before touching ArcGIS:

QuestionConsiderations
What is the purpose?Locate points of interest? Navigate? Compile collected data? Convince/persuade? Compare data? Analyze to learn new information?
Who is the audience?Expected subject knowledge? Map literacy? Special requirements (color blindness, large print, Section 508 accessibility)?
What is the medium?Paper (report, wall map, poster)? Electronic file (PDF, GIF — format, file size, delivery)? Computer screen (size, types)? Projection? Mobile devices?
Under what conditions will it be viewed?Indoor/outdoor? Screen glare on mobile? Viewer attention span? Competing with other information?

Paper maps vs. web maps require fundamentally different design approaches. Paper maps are static, fixed-size, high-resolution, and can be complex — the viewer is committed. Web maps are interactive, variable-size, viewed in poor lighting, compete for attention, and should do ONE THING really well with short viewer attention span.

Selecting Data & Cartographic Generalization

▶

A good map tells a story. Not every available layer should be included.

Lead Players

The main subject of the map. Symbolized prominently to draw the eye first. Example: volcanoes on a volcanic hazard map.

Supporting Layers

Context that helps the reader interpret the story without distracting from it. Example: county boundaries, highways.

Distracting Layers

Data that competes with the story without adding value. Remove or de-emphasize. Example: rivers on a volcanic hazard map.

Privacy & Sensitive Data

Consider: parcel owner names, private well tests, fossil/archaeological sites (poaching), pipelines (vandalism), fault lines near parcels (property value implications).

Cartographic Generalization — simplifying data for map display. Techniques include:

  • Exaggerate — make small features larger so they're visible
  • Displace — move features slightly so they don't overlap
  • Typify — represent a pattern with fewer symbols
  • Reclassify — merge categories for simpler display
  • Collapse — replace a polygon with a point at small scales
  • Aggregate — combine multiple features into one
  • Simplify — reduce vertices in a line or polygon
  • Refine — improve overall appearance and readability

Visual Hierarchy & Design Principles

▶

Visual hierarchy is the order in which a reader perceives map elements. The cartographer controls this through design choices.

Visual Center

The visual center of a page is ~5% ABOVE the geometric center. Maps centered on the geometric center feel slightly "heavy" at the bottom.

Rule of Thirds

Divide the page into thirds horizontally and vertically. Place important elements at the intersections of those dividing lines. Borrowed from photography composition.

Alignment

Place elements in ordered columns. Align edges of boxes and frames exactly using guides and snapping. Haphazard arrangement looks unprofessional and confuses the reader's eye.

Balance

Avoid crowded or open areas. Use negative space (empty area) as a design tool — more subtle than neatlines for separating or combining elements.

Foreground & Background

Dark colors and high contrast bring features to the foreground. Light, low-contrast symbols recede to the background. Important features = bold, prominent. Supporting features = subtle, light.

Pattern & Perception

The brain seeks patterns. Color is more effective than shape for highlighting categories. Changes in size communicate changes in quantity. Experiment with different portrayals.

Study basemaps (Esri, Google, OpenStreetMap) to learn from professional cartographers. Notice how they use light, low-contrast backgrounds to support overlaid data without competing with it. The background should be present but invisible — the user shouldn't notice it, just benefit from it.

Common bad map issues to avoid:

  • Haphazard arrangement of elements
  • Uneven negative space
  • Scale bar unnecessarily large
  • Decorative boxes/shadows that add no information
  • Symbols that don't clearly show quantities
  • Unrelated layers that distract from the objective
  • Cryptic file names or field names instead of real words
  • Too many significant figures (or too few)
  • Neatlines crowding the elements

Map Elements & Grids

▶

Map elements support the reader's understanding. Include only what serves the objective — unnecessary elements clutter and distract.

Required Elements

Title, legend, citations/data sources. These are almost always needed to make the map understandable and credible.

Often Needed

Scale bar, north arrow, location map (inset showing where the map area is in a broader context).

Sometimes Needed

Neatlines (border frames), graphics or photos, graticule or measurement grids.

When to Omit

North arrow if north is obvious. Scale bar if the map is for general reference only. If removing an element makes the map cleaner without losing information — remove it.

Map Grids:

  • Graticule grids — longitude/latitude marks around the map frame. Show geographic coordinates.
  • Measurement grids — show x-y projected coordinates (UTM, State Plane, etc.)
  • Index grids — lettered/numbered squares for locating features (like a street atlas grid reference)

Layouts, Text & Scale Bars

▶

A layout is the print/export page that contains one or more map frames plus supporting elements. The map frame is independent of the layout — its extent and scale can be set separately.

Map Frame

A map placed inside a layout. Activated by double-clicking to set its extent and scale. Scaling options: fixed center, fixed extent, or fixed scale.

Scale Bars

Sectioned into user-defined divisions. When the map zooms, behavior options: adjust bar width (keep division value), keep bar constant (change division size), or add/remove whole divisions.

Reference Scale

Without reference scale: symbols always appear at the assigned size. With reference scale: symbols and labels grow/shrink as map scale changes — useful for layouts at specific print scales.

Legends

Inserted for the active map frame. Edited via the ribbon or formatting pane. Should include only layers relevant to the map objective. Label with real words, not field names.

Types of text in ArcGIS Pro:

  • Graphic text — manually placed on a layout: straight, curved, or multiline in rectangle/polygon/circle/ellipse shapes
  • Dynamic labels — set as properties of a layer; automatically placed and updated when data or extent changes
  • Annotation — created and stored as features in a geodatabase; fully editable position, style, and content; most flexible but most work to create and maintain

When the layout is active, the Contents pane shows all layout elements (map frames, titles, legends, scale bars, etc.). Select elements there to edit their properties in the ribbon or formatting panes. This is separate from the map view — switching between layout and map view changes what the Contents pane shows.
