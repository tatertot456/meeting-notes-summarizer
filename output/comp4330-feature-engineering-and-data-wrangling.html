Data Cleaning

▶

Real-world data is dirty — incomplete, noisy, and inconsistent. Cleaning is the first step.

Missing Data

Handle by ignoring the tuple, filling with mean/median, using the most probable value via Bayesian inference or decision tree, or global constant imputation.

Noisy Data

Handle via binning (smooth by mean/median/boundary), regression, clustering to detect outliers, or combined human inspection.

Inconsistent Data

Detect using metadata, uniqueness/null rules, and commercial auditing tools. Resolve via ETL transformations.

Binning

Sort data and partition into equal-frequency or equal-width bins, then smooth by bin means, medians, or boundaries.

Data auditing uses correlation and clustering to detect violations of data rules and identify outliers automatically.

Data Integration

▶

Combining data from multiple sources introduces challenges:

  • Schema integration: A.cust-id ≡ B.cust-# — matching equivalent attributes across sources
  • Entity identification: "Bill Clinton" = "William Clinton" across databases
  • Data value conflicts: Same entity, different representations or scales (metric vs. imperial)
  • Redundancy detection: Use correlation analysis (χ²) and covariance analysis to find derivable/duplicate attributes

Correlation does not imply causality — hospitals and car theft are correlated because both depend on population size.

Key measures for numeric correlation:

  • Pearson's r > 0: Positive correlation
  • Pearson's r = 0: Independent
  • Pearson's r < 0: Negative correlation
  • Covariance > 0: Both variables tend to be above their means together

Data Reduction

▶

Obtain a smaller representation of data that produces the same analytical results.

PCA

Principal Component Analysis — projects data onto orthogonal eigenvectors capturing maximum variance. Reduces dimensions while preserving information.

Wavelet Transform

Decomposes signals into frequency subbands. Preserves relative distances at multiple resolutions. Used for image compression.

Attribute Subset Selection

Remove redundant and irrelevant attributes. Methods: best single attribute, step-wise forward selection, backward elimination.

Numerosity Reduction

Histograms, clustering, sampling, regression models, and data cube aggregation to reduce volume.

Curse of dimensionality: as dimensions increase, data becomes increasingly sparse and distance measures lose meaning.

Data Transformation & Normalization

▶

Transform data so all attributes fall within comparable ranges for fair analysis:

Min-Max Normalization

Maps values to [new_min, new_max]. Ex: $73,600 in [$12K–$98K] → 0.716 in [0,1].

Z-Score Normalization

Standardizes using mean (μ) and standard deviation (σ). Good when min/max are unknown.

Decimal Scaling

Divides by powers of 10 until max absolute value < 1.

Attribute Construction

Create new features from existing ones. Ex: RGB → Hue/Saturation/Intensity, combining sparse features.

Data Discretization

▶

Divide continuous attribute ranges into intervals with labels — reduces data size and prepares for classification.

Equal-Width Binning

Divides range into N equal-sized intervals. Simple but sensitive to outliers and skewed data.

Equal-Frequency Binning

Each bin contains approximately the same number of samples. Better for skewed data.

Decision Tree Discretization

Supervised, top-down split using entropy to find the best split points.

Chi-Merge

Unsupervised, bottom-up merge. Merges neighboring intervals with similar class distributions (low χ² values).

Concept hierarchy generation organizes values hierarchically: street → city → state → country. Supports drill-down and roll-up in data warehouses.
