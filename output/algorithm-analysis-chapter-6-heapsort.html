The Binary Heap

▶

A binary heap is an array that can be viewed as a nearly complete binary tree. Each node corresponds to an array element.

Array index relationships:

  • Parent of i: ⌊i/2⌋
  • Left child of i: 2i
  • Right child of i: 2i + 1

Max-Heap Property

Every node's value ≥ its children's values. The root is the maximum element. Used for heapsort and max-priority queues.

Min-Heap Property

Every node's value ≤ its children's values. The root is the minimum element. Used for min-priority queues.

The heap is stored as a flat array but behaves like a tree. This gives O(1) access to parent/child relationships via index arithmetic — no pointers needed.

MAX-HEAPIFY

▶

MAX-HEAPIFY maintains the max-heap property. It assumes the subtrees rooted at left and right children are max-heaps, but A[i] may be smaller than its children.

MAX-HEAPIFY(A, i)
l = LEFT(i)
r = RIGHT(i)
if l ≤ A.heap-size and A[l] > A[i]
    largest = l
else
    largest = i
if r ≤ A.heap-size and A[r] > A[largest]
    largest = r
if largest ≠ i
    swap A[i] with A[largest]
    MAX-HEAPIFY(A, largest)

Runtime: O(log n) — the element "floats down" the tree at most h = ⌊log n⌋ levels. Each level does O(1) work (compare, possibly swap).

BUILD-MAX-HEAP

▶

Converts an unordered array into a max-heap by calling MAX-HEAPIFY on each non-leaf node from bottom to top:

BUILD-MAX-HEAP(A)
A.heap-size = A.length
for i = ⌊A.length/2⌋ downto 1
    MAX-HEAPIFY(A, i)

Runtime: Θ(n) — NOT O(n log n) as a naive analysis suggests. Most nodes are near the bottom (small height), so the total work sums to a linear bound. This is a tight analysis using the sum of heights in a heap.

We start at ⌊n/2⌋ because nodes at indices ⌊n/2⌋+1 through n are all leaves — they already satisfy the heap property trivially.

The Heapsort Algorithm

▶

Heapsort combines BUILD-MAX-HEAP with repeated extraction of the maximum:

HEAPSORT(A)
BUILD-MAX-HEAP(A)
for i = A.length downto 2
    swap A[1] with A[i]
    A.heap-size = A.heap-size - 1
    MAX-HEAPIFY(A, 1)

How it works:

  • After BUILD-MAX-HEAP, A[1] is the largest element
  • Swap it with A[n] (last position) — it's now in its final sorted position
  • Reduce heap size by 1, restore heap property with MAX-HEAPIFY
  • Repeat n-1 times

Runtime: O(n log n) — BUILD-MAX-HEAP is O(n), then n-1 calls to MAX-HEAPIFY each costing O(log n). Sorts in-place with no extra memory beyond O(1).

Priority Queues

▶

A priority queue maintains a set S of elements with associated keys. A max-priority queue supports:

MAXIMUM

Return the element with the largest key. O(1) — just return A[1].

EXTRACT-MAX

Remove and return the maximum element. O(log n) — swap with last, shrink, heapify.

INCREASE-KEY

Increase key of element x to value k. O(log n) — bubble element up until heap property restored.

INSERT

Insert element with key k. O(log n) — add at end with -∞, then INCREASE-KEY to k.

Applications: OS job schedulers (always run highest priority task), event-driven simulation (process earliest event first), Dijkstra's shortest path algorithm.
