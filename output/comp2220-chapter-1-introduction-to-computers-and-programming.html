Why Program?

▶

A computer is a programmable machine designed to follow instructions. A program is a set of instructions stored in computer memory that makes it do something useful. A programmer is the person who writes those instructions.

Without programmers, there are no programs. Without programs, a computer cannot do anything — it's just hardware sitting idle. Programming is what gives computers purpose.

Hardware Components

▶

CPU (Central Processing Unit)

Control Unit: retrieves and decodes instructions, coordinates all other parts.

Arithmetic & Logic Unit (ALU): optimized for high-speed numeric calculations and true/false decisions.

Main Memory (RAM)

Volatile — erased when program ends or computer turns off. Organized into bits (0 or 1) and bytes (8 bits). Each byte has a unique address — like a house number for data.

Secondary Storage

Non-volatile — data persists when powered off. Types: magnetic (hard drives), solid-state (SSD chips, no moving parts), optical (CD/DVD), flash drives (USB).

Input & Output Devices

Input: keyboard, mouse, touchscreen, scanner, microphone, disk drives.

Output: monitor, printer, speakers — anything that presents results to the user.

Memory addressing: every byte in RAM has a unique number called an address. For example, the value 149 might be stored at address 16, and 72 at address 23. The CPU uses these addresses to read and write data precisely.

Software Categories

▶

System Software

Programs that manage the computer hardware and the programs that run on them. Examples: operating systems (Windows, macOS, Linux), utility programs, software development tools.

Application Software

Programs that provide services directly to the user. Examples: word processors, games, browsers, programs written to solve specific business or scientific problems.

Programming Languages

▶

A program starts as an algorithm — a set of well-defined steps for solving a problem. The algorithm is then expressed in a programming language the computer can eventually execute.

Computers only execute machine language — binary instructions like 1011010000000101. Writing directly in binary is impractical, so programmers use higher-level languages that get translated down to machine code.

Low-Level Languages

Communicate directly with hardware. Written in binary machine code (0s and 1s) or assembly language. Extremely fast but difficult to write and not portable across hardware.

High-Level Languages

Closer to human language. Examples: C++, Python, Java, C#. Must be translated to machine code before execution. Much easier to write, read, and maintain.

Example algorithm — calculating gross pay:

  1. Ask "How many hours did you work?" and store the answer
  2. Ask "How much do you get paid per hour?" and store the answer
  3. Multiply hours × rate and store the result
  4. Display the calculated pay to the user

From C++ Source to Executable

▶

A C++ program goes through several steps before it can run:

  1. Write: create the source file (.cpp) in a text editor
  2. Preprocessor: converts source file directives (#include, #define) into actual code
  3. Compiler: translates the preprocessed source into machine instructions (object file)
  4. Linker: connects hardware-specific code and libraries to produce the final executable file

Steps 2–4 are typically triggered by a single button click or command in an IDE. An error at any step prevents all following steps from executing — fix errors in order from the top of the error list.

IDEs (Integrated Development Environments) combine all these tools into a single application: editor, compiler, debugger, and linker. Examples: Microsoft Visual C++, CLion, XCode, Eclipse, NetBeans.

Structure of a C++ Program

▶

Every C++ program is made up of several common elements:

ElementDescriptionExample
KeywordsReserved words with special meaning — cannot be used for anything elseint, double, return, using, namespace
IdentifiersNames made up by the programmer to represent variables, functions, etc.hours, rate, pay
OperatorsSymbols that perform operations on data+, -, *, /, =, <<, >>
PunctuationCharacters that end statements or separate items; and ,
SyntaxThe grammar rules that govern how all elements are combined correctlyRules for using all of the above
// Example C++ gross pay program structure
#include <iostream>
using namespace std;

int main() {
    double hours, rate, pay;   // variable definition

    cout << "How many hours did you work? ";
    cin >> hours;

    cout << "How much do you get paid per hour? ";
    cin >> rate;

    pay = hours * rate;        // processing

    cout << "You earned $" << pay << endl;  // output

    return 0;
}

Variables

▶

A variable is a named storage location in memory for holding a piece of data. Before using a variable you must define it (also called declaring it) — specifying its data type and name.

double hours, rate, pay;   // defines 3 variables of type double

double means the variable holds a double-precision floating point number (a decimal number). You'll learn more data types in Chapter 2.

The three steps most programs follow: Input (gather data from keyboard or file) → Processing (compute results) → Output (display results on screen or write to file). This IPO pattern appears in virtually every program you'll write.

Programming Paradigms

▶

Procedural Programming

Focus is on the process. Procedures (functions) are written to process data step by step. The program is a sequence of instructions. C is the classic procedural language.

Object-Oriented Programming (OOP)

Focus is on objects — self-contained units that combine data and the operations that manipulate that data. You send messages to objects to make them perform actions. C++ supports both paradigms.

C++ is a multi-paradigm language — it supports procedural programming (like C) and object-oriented programming. This course will start procedurally and introduce OOP concepts as you progress.
