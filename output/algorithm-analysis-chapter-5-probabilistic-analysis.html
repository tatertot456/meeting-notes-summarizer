The Hiring Problem

▶

You interview candidates one at a time. After each interview, you must immediately hire or reject. You always hire someone better than your current employee.

Worst Case

Candidates arrive in increasing order — you hire everyone. Cost = n·(hire cost). This happens with probability 1/n! of all orderings.

Average Case

Over all random orderings, the expected number of hires is O(log n). Much better than worst case!

This problem motivates probabilistic analysis: worst case is too pessimistic. By analyzing expected behavior over random inputs, we get a much more realistic picture.

Indicator Random Variables

▶

An indicator random variable I{A} for event A is defined as:

  • I{A} = 1 if event A occurs
  • I{A} = 0 if event A does not occur

Key property (Lemma 5.1): E[I{A}] = Pr{A}

The expected value of the indicator equals the probability of the event. This lets us convert counting problems into probability problems.

For the hiring problem: let Xᵢ = I{candidate i is hired}. Candidate i is hired iff they are the best of the first i candidates — probability 1/i. Total hires E[X] = Σ 1/i = H(n) = Θ(ln n) = O(log n).

Randomized Algorithms

▶

A randomized algorithm makes random choices during execution. Instead of relying on the input being random, we randomize the algorithm itself.

Probabilistic Analysis

Assumes input is drawn randomly. Performance depends on input distribution — requires knowledge of that distribution.

Randomized Algorithm

Algorithm makes random choices. Performance is guaranteed regardless of input — adversary cannot choose a bad input because the algorithm's behavior is random.

For the hiring problem: randomly permute candidates before interviewing. Now any ordering is equally likely regardless of how they arrived — expected cost is O(log n) guaranteed.

The Birthday Paradox

▶

How many people must be in a room before there's a 50% chance two share a birthday?

Surprisingly: only 23 people. With 57 people, the probability exceeds 99%.

Using indicator variables: let Xᵢⱼ = I{person i and j share a birthday}. E[Xᵢⱼ] = 1/365. Total pairs = C(k,2) = k(k-1)/2. Expected matches = k(k-1)/730. Setting this to 1 gives k ≈ 28.

This result is counterintuitive because we're not asking "does anyone share MY birthday" — we're asking "does any pair share a birthday." The number of pairs grows quadratically with k.

Balls and Bins, and Streaks

▶

Balls and Bins

Throw n balls into n bins uniformly at random. Expected balls per bin = 1. Expected empty bins = n/e. Maximum load = Θ(log n / log log n) with high probability.

Streaks

Flip a fair coin n times. What is the longest streak of heads? Expected longest streak = Θ(log n). With high probability, the longest streak is between (1/2)log n and 2 log n.

Online Hiring Problem

▶

What if you can only hire one person total? The strategy:

  • Interview the first k candidates but reject them all (use as a baseline)
  • Hire the next candidate who is better than all k baseline candidates
  • Optimal k = n/e ≈ 0.368n

With k = n/e, the probability of hiring the best overall candidate is at least 1/e ≈ 0.368. This is the best possible strategy for the online setting — a beautiful result from optimal stopping theory.
