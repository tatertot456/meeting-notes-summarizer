CSS Overview

▶

W3C's goal for HTML and CSS: use HTML for content, use CSS for appearance. CSS (Cascading Style Sheets) provides detailed control over a web page's visual presentation. The current standard is CSS3, supported by all major browsers.

Keep style and content separate. If you embed styles directly in each element, changing the appearance later requires editing every single element. With CSS in one place (the head or an external file), one change updates the entire page or site. This is the most important reason CSS exists.

In Blazor, everything outside "@" lines is HTML/CSS. Lines with "@" are C# code. CSS in Blazor can appear as inline styles, embedded styles, external stylesheets, themes, or component-scoped styles (.razor.css files).

Avoid using the style attribute (inline styles) in your pages — it defeats the purpose of CSS. Use embedded or external styles. External CSS files are most appropriate when you have 5+ CSS rules OR when multiple pages share the same styles. Shared external CSS is where CSS truly shines — one file, consistent look everywhere.

CSS Rules & Selectors

▶

A CSS rule tells the browser which elements to style and how. Browsers match rules to elements using selectors.

/* Short rule format */
selector { property1: value; property2: value; }

/* Block format for long rules */
selector {
    property1: value;
    property2: value;
}

/* Type selector — matches all h1 elements */
h1 { color: blue; }

/* Universal selector — matches EVERY element */
* { text-align: center; }

/* Multiple selectors — comma separates them */
h2, p { font-style: italic; color: blue; }

/* hr element — set width to 50% */
hr { width: 50%; }

Class selectors — apply to any element with a matching class attribute:

/* Class selector uses a dot prefix */
.red { background-color: tomato; }
.bold-text { font-weight: bold; }

/* Apply in HTML with class attribute */
<p class="red">This paragraph has a red background.</p>

/* Multiple classes on one element (space-separated) */
<span class="white orange-background">$10</span>

/* Element-type + class for more precision */
q.red { background-color: tomato; }  /* only q elements with class red */
h1.blue { color: blue; }             /* only h1 elements with class blue */

ID selectors — uniquely identify one specific element:

/* ID selector uses # prefix */
#lizards-lounge { color: green; }

/* Applied in HTML */
<h3 id="lizards-lounge">Lizards Lounge</h3>

/* Use hyphens to separate words in id and class names */

span vs div for grouping without semantic meaning:

<div class="orange">
    NNU Halloween on the River.<br>
    Corn maze: <span class="white orange-background">$10</span>
</div>
/* div = block element (line breaks above/below)  */
/* span = inline element (flows within surrounding text) */

Use span when you want to style text WITHIN a line of surrounding text. Use div when you want the styled content to sit on its own line(s). The local span rule overrides the broader div rule — that's cascading in action.

Cascading — Priority Order

▶

"Cascading" means CSS rules from multiple sources can apply to the same element. When rules conflict, priority determines which wins:

  1. Inline style attribute (highest priority — overrides everything)
  2. style container in the <head>
  3. Blazor component file or external file
  4. User's browser settings
  5. Browser's native defaults (lowest priority)

The browser checks for matching CSS rules starting from the highest priority source. The first match wins — the search stops. This is why a local span rule overrides a broader div rule, and why inline styles override everything. Understanding this cascade is essential for debugging unexpected styling behavior.

Color Properties

▶

Color is critical — it sets tone, creates contrast, highlights content, and drives emotion. Two key properties: color (text color) and background-color.

/* Hex: #000000=black, #ffffff=white */
/* #bbbbff = light blue-purple */
/* #005500 = dark green */

/* RGBA opacity: 0=transparent, 1=opaque, .5=halfway */
.yellow-transparent { background-color: rgba(255,255,0,.4); }

/* HSL: hue 0-360 (0/360=red, 120=green, 240=blue) */
/*      saturation: 0%=gray, 100%=full color         */
/*      lightness:  0%=black, 100%=white              */
.sky { background-color: hsl(210, 100%, 70%); }

CSS3 defines 147 named colors. Hex is the most common format in professional code — #rrggbb where 00 = least intensity, ff = greatest. RGBA and HSLA add an opacity (alpha) value from 0 (fully transparent) to 1 (fully opaque). Use RGBA with background-color for overlay effects.

/* serif, sans-serif, monospace, cursive, fantasy       */

/* size/line-height = font size takes up 2.5em of 3em line height */

/* em units: 1em = height of element's current font size */
.disclaimer    { font-size: .5em; }  /* half normal size */
.advertisement { font-size: 3em; }  /* three times normal */

Always end font-family lists with a generic family name (serif, sans-serif, monospace, cursive, fantasy) as a fallback — all browsers recognize them. Generic names are lowercase. Regular font names use Title Case and are case-sensitive on some systems. One em = the height of the current font — it scales relative to context, making it flexible for responsive design.

Border Properties

▶ 
/* border-style: required for border to show */
.coupon { border-style: dashed; }
/* Values: none, solid, dashed, dotted, double */

/* border-width: requires border-style to also be set */
h1 { border-style: solid; border-width: 4px; }
/* px = CSS pixel (1/96th inch at 100% zoom on desktop) */

/* Four-value shorthand: top right bottom left (clockwise) */
h1 { border-style: solid; border-width: 4px 2px 0px 2px; }

/* Two values: top-bottom / left-right */
h1 { border-width: 4px 2px; }

/* border-color: requires border-style to also be set */
h1 { border-color: blue; } /* same color formats as color property */

/* border shorthand: width style color */
.box { border: thin dotted blue; }
.box2 { border: 10px solid; }
/* style is required — without it, border-style defaults to none */

A CSS pixel is approximately 1/96th of an inch on a desktop at 100% zoom. However, actual pixel size varies by monitor and changes when users zoom in/out or change resolution. Using em or percentage units instead of px makes designs more flexible and resilient to user preferences.

The CSS Box Model

▶

Every HTML element has an invisible box around it. The CSS box model defines four concentric layers:

Content

The actual text or image inside the element. Width and height properties control this area.

Padding

Space INSIDE the border between the content and the border. Adds breathing room inside the element. Same background color as content.

Border

The visible line around the element. Controlled by border-style, border-width, and border-color.

Margin

Space OUTSIDE the border between this element and neighboring elements. Transparent — shows the page background.

/* padding: inside the border */
h1 { padding: 5px; }
h1 { padding: 10px 20px; }  /* top-bottom / left-right */

/* margin: outside the border */
p { margin: 10px; }

/* Body has a default 8px margin — override it */
body { margin: 0; }          /* remove all margin */
body { margin: 10%; }        /* percentage — scales with window size */

/* Individual sides */
.box {
    padding-top: 10px;
    padding-right: 20px;
    margin-left: 5px;
    margin-bottom: 15px;
}

The default body margin (8px) is why content doesn't touch the edges of the browser window. Setting margin: 0 on the body removes this. Using percentage values for margin/padding makes them scale proportionally when the user resizes the browser — essential for responsive design. Quiz answers: background-color applies background color (not bgcolor or color attribute); the class attribute attaches CSS to an element.
