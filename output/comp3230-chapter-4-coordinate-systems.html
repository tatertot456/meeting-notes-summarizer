Coordinate Systems Overview

▶

Every GIS dataset uses x, y, and sometimes z values to locate geographic data. The choice of values defines the coordinate system. A dataset using a coordinate system is said to be georeferenced.

Unprojected (Geographic)

Based on spherical coordinates. Measured in degrees of latitude and longitude. The earth is treated as a 3D sphere/ellipsoid. Data stored as decimal degrees.

Projected

Converts spherical coordinates to flat (planar) 2D coordinates. Uses mathematical equations to project 3D coordinates onto a 2D map. Stored in linear units (meters, feet).

Standard coordinate systems allow different datasets to be aligned and overlaid. A USGS topo map may contain three coordinate systems simultaneously: latitude-longitude, UTM, and State Plane — each gives different x-y values for the same point.

On-the-fly projection: ArcGIS automatically converts data stored in any coordinate system to the map's coordinate system for display. This does NOT change the stored coordinates and does NOT fix analysis errors. Always store data in the appropriate coordinate system for your analysis — don't rely on on-the-fly display.

Geographic Coordinate System (GCS)

▶

A GCS stores locations as longitude-latitude values measured in degrees:

  • Longitude — east-west angle measured from the Prime Meridian (Greenwich, England). Range: -180° to +180°.
  • Latitude — north-south angle measured from the equator. Range: -90° to +90°.

Precision requirements:

1 degree at equator ≈ 110 km
0.001°             ≈ 110 meters
0.000001°          ≈ 0.1 meter

→ Keep at least 5-6 decimal places for accurate locations
→ Degrees-Minutes-Seconds (DMS) must be converted to
  Decimal Degrees (DD) before storing in GIS

The earth's shape must be approximated for mapping. A spheroid (ellipsoid) is a smooth mathematical shape with a slightly flattened north-south axis — used for most mapping. A geoid is modeled from gravity differences and most closely matches the true earth shape, but is too complex for mapping calculations. The GCS is based on the spheroid.

Datums

▶

A datum is the reference framework that approximates the earth's shape for mapping. It shifts the ellipsoid relative to the geoid to achieve the best fit. Different datums place the same point at different coordinates.

Local Datum

Optimizes fit for a particular region. May use surveyed control points for additional adjustments. Best accuracy locally but poor accuracy globally.

Geocentric (World) Datum

Optimizes fit for the entire earth. Used by GPS. May be slightly less accurate locally than a regional datum.

DatumSpheroidNotes
NAD27Clarke 1866Common until 1980s. Still used in some legacy datasets. Local datum centered on Kansas.
NAD83GRS80Current standard for most North American mapping. First choice when datum is unknown.
NAD83 HARNGRS80Updates NAD83 with High Accuracy Reference Network. More precise for survey-grade work.
WGS84WGS84Geocentric. Default for GPS units. Essentially identical to NAD83 for most practical purposes.

Datum transformations are NOT exact mathematical formulas — they introduce errors of up to several meters. Errors accumulate with repeated transformations. Convert datums only when necessary, and choose the best transformation method for your region. Always turn on the datum transformation warning in Project options. For GPS users: YOU MUST KNOW AND RECORD THE DATUM used to collect data!

Map Projections

▶

A projection mathematically converts points on the earth's curved surface to a flat 2D plane. Every projection has an underlying GCS — projections based on different datums will be offset from each other.

Cylindrical

Projection surface is a cylinder wrapped around the earth. Usually preserves direction and shape (conformal). Mercator is the most famous example. Creates extreme distortion near poles.

Conic

Projection surface is a cone placed over the earth. Usually preserves area and/or distance. Best for mid-latitude regions (like the continental US). Lambert Conformal Conic, Albers Equal Area.

Azimuthal (Planar)

Projection surface is a flat plane touching the earth. Usually preserves area and/or distance from the center point. Good for polar regions.

Key projection parameters:

  • Central meridian — longitude at center of map where x = 0
  • Standard parallels — where projection surface touches the ellipsoid (lines of zero distortion). Tangent = 1 standard parallel. Secant = 2 standard parallels.
  • Latitude of origin — latitude where y = 0
  • False easting / False northing — offsets added to ensure all coordinates are positive
  • Principal scale — the base scale of the projection

Distortion & Projection Selection

▶

ALL map projections distort one or more properties. No flat map can perfectly represent the curved earth. The distortion type varies by projection family.

PropertyPreserved ByWhen to Choose
Direction (Conformal)Cylindrical (Mercator, Transverse Mercator)Navigation, preserving shape of small areas
ShapeConformal projectionsGeneral reference maps, navigation
Area (Equal Area)Conic (Albers), AzimuthalComparing sizes, density calculations, statistics
Distance (Equidistant)Conic, Azimuthal equidistantDistance measurements from a point or along lines

Maps displayed in a Geographic Coordinate System appear distorted — especially at high latitudes. Always project your data for analysis and for final presentation. The coordinate system used to STORE data directly affects area and distance calculations. Data stored in GCS (degrees) will produce incorrect distance and area results — always project to an appropriate system before running spatial analysis.

Small-scale (continent/country) maps: Distortion is inevitable — choose based on purpose. Cylindrical for direction/shape, conic for distances/areas, compromise projections for general reference.

UTM & State Plane Systems

▶

Universal Transverse Mercator (UTM) — standard for local to regional mapping:

  • 60 N-S oriented zones around the world, each 6° of longitude wide
  • Extends from 84°N to 80°S latitude
  • Secant case Transverse Mercator — standard meridians 180 km east and west of the central meridian
  • No distortion along the central meridian
  • Preserves all four map properties well within a single zone
  • Ideal for local, county, or small state maps within a single zone
  • Default datum: NAD83 (North America) or WGS84 (global)

State Plane Coordinate System (SPCS) — designed for individual US states:

  • Small zones designed for each state — minimal distortion within a zone
  • N-S oriented zones use Transverse Mercator
  • E-W oriented zones use Lambert Conformal Conic
  • Alaska uses Oblique Mercator
  • Other countries have similar national systems

Oregon spans two UTM zones AND two State Plane zones — neither standard system is ideal. This is a case where a custom projection may be warranted. Custom projections give better accuracy but require careful metadata documentation and coordinate system conversions for all data.

Custom & Adapted Projections

▶

When no standard projection fits well, adapt an existing one by modifying its parameters:

Example: Custom projection for Turkey (east-west oriented country)
Base: Lambert Conformal Conic (conic best for mid-latitude E-W regions)
Datum: Same as all other data in the collection
Central meridian: 35° (center of Turkey)
Standard parallels: 37° and 41° (upper/lower sixths of Turkey's extent)
False easting/northing: set if all-positive coordinates needed

Example: Adapted UTM for Oregon
Base: UTM Zone 10
Change: Shift central meridian to center of Oregon
Keep: All other parameters identical to UTM Zone 10
Rename: Give it a distinct name in metadata

Example: Adapted State Plane for Kansas
Base: Kansas State Plane South
Change: Shift standard parallels to upper/lower sixths of the state
Keep: All other parameters identical

Custom projections produce the best accuracy for maps but require careful management: all data must be converted to the custom CS; GPS units likely can't collect data directly in the custom CS; metadata must clearly document the custom parameters so other users can work with the data correctly.

Coordinate Problems, Spatial Reference & Tools

▶

Coordinate problems occur when two datasets should align but don't. The cause is almost always a missing or incorrect coordinate system label.

Discrepancy SizeLikely Cause
100s–1000s of kilometersProjection is mislabeled — the data is actually in a different projection than what's recorded
10s–100s of metersDatum is mislabeled — data collected in NAD27 but labeled as NAD83 (or vice versa)
1–10 metersData accuracy/quality issue — not a coordinate system problem

Spatial Reference — the complete coordinate system label stored with each dataset:

  • Projection name and all parameters (for projected data)
  • Geographic coordinate system and datum (all data have this)
  • Storage units (meters, feet, degrees)
  • Domain (maximum allowable x-y values)
  • Resolution (x-y precision)

CRITICAL: Define Projection vs. Project tool — DO NOT CONFUSE THESE!

Define Projection Tool

Changes ONLY the CS label (metadata). Does NOT change any coordinates in the file. Keeps the original dataset. Use ONLY when the label is missing or incorrect. Most CS problems are caused by misuse of this tool.

Project Tool

Changes the actual coordinate values in the file. Updates the CS label to match. Creates a NEW dataset (original is preserved). Use to permanently convert between coordinate systems. Use when assembling data collections with consistent stored CS.

Diagnosing coordinate problems:

  • Check the layer properties → Source tab to see current coordinate system and extent values
  • Check if the units in the extent match the expected coordinate system (degrees vs. meters)
  • Review metadata/documentation for the original data source
  • If all else fails: guess and test — assign a coordinate system and see if the data aligns
  • Use Map Properties to review all coordinate systems in the current map
