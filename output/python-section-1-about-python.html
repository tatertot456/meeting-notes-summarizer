What Is Computer Science?

▶

A common misconception is that computer science is the study of computers, or of how to write programs, or of applications and software. All of these are incomplete. The real definition:

Computer Science is the study of algorithms — including their formal and mathematical properties, hardware realizations, linguistic realizations, and applications.

A computer scientist designs and develops algorithms to solve problems. The four areas of study:

Formal & Mathematical Properties

Studying the behavior of algorithms to determine whether they are correct and efficient.

Hardware Realizations

Designing and building computer systems that are able to execute algorithms.

Linguistic Realizations

Designing programming languages and translating algorithms into those languages.

Applications

Identifying important problems and designing correct, efficient software to solve them.

Algorithms

▶

An algorithm is a well-ordered collection of unambiguous and effectively computable operations that, when executed, produces a result and halts in a finite amount of time.

Breaking that down:

  • Well-ordered: upon completion of any step, we always know which step comes next
  • Unambiguous: every operation can be understood and carried out directly by the computing agent without further explanation (called a "primitive")
  • Effectively computable: it must not only be understandable — it must be doable
  • Finite: the algorithm must terminate — infinite loops are a common mistake in algorithm design

If we can specify an algorithm to solve a problem, we can automate its solution. The computing agent (machine, robot, person) does not need to understand the concepts behind the solution — it just follows the steps.

Three categories of algorithmic operations:

Sequential

Carry out one well-defined task, then move to the next. Example: "Set x to 1."

Conditional

Ask a question and select the next step based on the answer. Example: "If x ≠ 0, set y = 1/x; otherwise print error."

Iterative

Repeat a block of instructions. Example: "While there are more checks to process, do the following five steps."

Result

The result of an algorithm can be a number, text, error message, image, sound, or change in the computing agent's environment.

What Is Python?

▶

Python is an open-source, high-level, interpreted programming language that looks like pseudocode. It supports object-oriented, functional, and procedural programming styles.

  • Started in 1989/1990
  • Python 2.0 released in 2000
  • Python 3.0 released in 2008 — broke backwards compatibility but removed duplication and added structural improvements
  • #1 programming language in 2025 (TIOBE Index)

Interpreted

Unlike compiled languages, Python code is run through an interpreter line by line. This makes it "slower" than C or C++ but dramatically easier to develop in.

High-Level

Abstracts away low-level details like memory management. Looks almost like English pseudocode — very readable.

Object-Oriented

Everything is an object, including numbers and operators. Every object has a type. Supports full OOP with classes and inheritance.

Module System

Can be extended with libraries written in Python or other languages (like C). This is Python's greatest strength — flexibility through modules.

Python works as a general programming language AND extends into specialized areas: data science, machine learning, blockchain, automation, web development, and more. Often described as "Python as Glue" — it connects different systems and languages together.

Why Python? Why Multiple Languages?

▶

No one has created a "one size fits all" programming language. Different languages excel at different things — speed, flexibility, ease of use, software engineering principles. You should expect to use multiple languages throughout your career.

Why Python specifically:

  • Originally designed as a starter language — easy to learn and understand
  • Promotes good programming habits
  • Multiplatform (Mac, Windows, Linux)
  • Concise — often fewer lines of code than other languages
  • Flexible — object-oriented, functional, or procedural
  • Open source and free with a massive developer community
  • Huge library ecosystem for data science, ML, visualization, web, and more
  • Platform-independent

Don't rely too heavily on AI to write your code. Literally anyone can generate code with AI — what separates you is understanding HOW it works and WHY it's written that way. You may be able to fake it through college, but you won't at your job. Learn by doing, not watching.

Python Basics — Shell, Comments & Working Directory

▶

The IDE sits on top of the Python shell. The shell lets you input commands one at a time — useful for initial development and exploration.

# Single line comment — use the hash mark
""" 
This is a 
multiline comment 
"""

# Variables and output
a_piece_of_code = "This is a string"
print("Output " + a_piece_of_code)
# Output: Output This is a string

# Getting help
help(str)               # full help for str class
help(str.capitalize)    # help for a specific method
str.capitalize?         # Spyder/IPython shorthand

x = "hello"
print(x.capitalize())   # Hello

Setting your working directory:

import os

os.getcwd()                    # get current working directory
os.chdir(r"C:\Project1")       # set working directory
help(os)                       # explore the os module

Setting your working directory matters because Python looks for files relative to it. The r before the string makes it a "raw string" — backslashes are treated literally, not as escape characters. This is important on Windows where paths use backslashes.

Modules

▶

Modules extend Python with new functionality — math, statistics, date/time, graphics, machine learning, etc. Modules can also be user-defined as .py script files.

# Three ways to import modules
import keyword                  # import the module
import keyword as kw            # import with an alias (shorter name)
from keyword import kwlist      # import specific object into namespace

# Using an alias
import keyword as kw
print(kw.kwlist)                # list of all Python keywords

# What NOT to do
from keyword import *           # loads EVERYTHING into namespace
                                # avoid — pollutes the namespace and
                                # makes it unclear where things came from

Use import module as alias for frequently used modules — numpy as np and pandas as pd are standard conventions you'll see everywhere. Avoid from module import * because it loads everything into the local namespace without clear labeling of where each name came from, making debugging much harder.
