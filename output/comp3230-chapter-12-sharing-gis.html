Web GIS

▶

GIS multiplies in value when shared. Web GIS content is delivered over the Internet from powerful GIS Servers working through a Web Server configured to communicate with users (clients). Clients can be desktop computers, laptops, tablets, or smartphones.

Data Downloads

Datasets downloaded and stored locally on desktop or laptop computers. Traditional model — user has a local copy.

Web Maps & Services

Map services, image services, and feature layers consumed on demand. Work on desktop AND mobile devices. Data stays on server.

Geoprocessing Services

Analysis runs ON the server — client doesn't need analysis tools installed. Results returned to the client. Good for complex or data-heavy processing.

Workflows

Stored sequences of analysis steps that can be shared, repeated with new inputs, and executed automatically. Reduces errors and speeds work.

Metadata & Standards

▶

Metadata (data about data) is essential when sharing resources. It lets users determine whether a dataset is suitable for their purpose. Good metadata is a sign of quality data. The thoroughness of metadata is a clue to the professionalism of the provider.

Six basic data quality issues metadata must address:

  • Lineage — where did it come from and how was it processed?
  • Positional accuracy — are features in the right location?
  • Attribute accuracy — are the table attributes correct?
  • Logical consistency — how well are real-world relationships represented?
  • Completeness — was every instance recorded or measurement taken?
  • Temporal accuracy — how long will the data remain valid? Are updates planned?

Metadata standards define required format and content so any GIS can read it. Most stored as text, HTML, or XML.

StandardScopeNotes
CSDGM (FGDC)US Federal StandardContent Standard for Digital Geospatial Metadata. Required for US government data. If all required fields filled = FGDC-compliant.
ISO 19115International (geospatial)Broad international geospatial metadata standard.
ISO 19139International (general)XML implementation of metadata standards.
North American Profile (NAP)US & CanadaCustomization of ISO 19115 for North America. Managed by FGDC.

CSDGM major sections: Identification, Data Quality, Data Organization, Spatial Reference, Entity and Attribute Info, Distribution Information, Metadata Reference.

ArcGIS Pro stores metadata in an internal format and lets you view/edit it in any style (FGDC, ISO, ArcGIS Item Description). Export to any required standard. If you plan to distribute data professionally or fulfill a contract, aim for at minimum FGDC-compliant metadata. Data contracts typically require it.

ArcGIS Online (AGOL)

▶

ArcGIS Online is Esri's cloud-based platform for sharing GIS resources. Accessed through the Portal tab in the Catalog pane in ArcGIS Pro.

Content TypeAccount RequiredCredits Used?
Open / Public ContentFree public account or noneNo
Subscription ContentOrganizational accountMinimal — most daily activities
Premium ContentOrganizational accountYES — warned before access. Examples: geocoding many addresses, downloading large datasets.

Organizational accounts are not free — the organization pays for user accounts AND service credits. Credits are consumed by data storage, premium content access, geocoding, and tile caching. Monitor credit use carefully, especially when caching tiles at many scale levels. If your estimated cache exceeds 100MB, consult your AGOL administrator first.

Service quality warning: Some AGOL services are professionally created and curated. Others are shared by inexperienced users and may have errors, poor metadata, or little value. Always check metadata thoroughness as a quality indicator. Some services provide actual data (analyzable); others provide display-only representations.

Tiling, Services & Optimization

▶

Optimization strategies for delivering large volumes of content efficiently:

  • Restrict geographic extent — only transmit features in the current map view
  • Use scale ranges — deliver only content appropriate to the current zoom level
  • Tile complex vector maps as JPG/PNG image snapshots for fast transmission
  • Tiles can be rendered dynamically (on demand) or pre-cached at standard scales

Tiling divides the map into squares — only the squares in the current view are transmitted. AGOL, Google Earth, and Bing share a common global tiling scheme with 19 scale levels. Only cache the scale levels you actually need.

Primary GIS service types:

Feature Layer

Serves actual vector features. Can be symbolized, queried, and analyzed. Can be configured for editing and mobile collection. Often downloadable.

Map Image Layer

Server-rendered map image. Complex map rendered on server, sent as image. Fast delivery but display-only — cannot analyze individual features.

Imagery Layer

Serves raster/image data. Provides actual data values for analysis (unlike map image layers). Used for satellite imagery, aerial photos, DEMs.

Vector Tile Layer

Tiles that retain vector format. Faster rendering than feature layers, more flexible styling than raster tiles. Display-only — not for analysis.

Ways to Share Data

▶

Hosted Services

User publishes feature layers, imagery, tiles, or geoprocessing to AGOL. Requires organizational account with publishing privileges. Can be configured for feature editing, syncing, and mobile access. Tile caching can consume many credits — configure carefully.

Packages

Compact all data for a layer, map, project, or workflow into a single file. Can use public OR organizational account — no publishing privileges needed. Easy to share. Downloading includes all linked data. Desktop use only — NOT compatible with mobile devices.

Sharing best practices:

  • Do NOT include basemaps in shared maps — consumers add their own
  • Set a default map extent (Full Extent) so the map opens to the right location
  • Keep feature layers as small as possible — large/complex datasets work better as tiles
  • Set appropriate visible scale ranges for all layers
  • Configure viewing, sharing, and editing permissions explicitly
  • Provide quality metadata and good search tags
  • Use the Analyze button before publishing — it identifies issues to fix first

Mobile Data Collection

▶

Mobile devices can only use published hosted feature layers configured for editing. Multiple field workers can collect and edit simultaneously and see each other's edits in real time.

Esri mobile apps:

  • Explorer for ArcGIS — free, view-only
  • Collector for ArcGIS — free to download, requires organizational account. Field data collection with GPS.
  • Survey123 — free to download, requires organizational account. Form-based data collection.

Setup workflow for mobile collection:

  • Create/configure feature classes with appropriate geometry and fields
  • Set up coded domains for attribute fields to aid data entry and reduce errors
  • Publish feature classes to AGOL as hosted layers configured for editing
  • Enable Sync if disconnected collection is needed (live updates disabled when syncing)
  • Create a web map containing the hosted layers and publish to AGOL
  • Install app on mobile device, log in with AGOL account, open web map, collect data

Mobile device location accuracy is generally less than a dedicated GPS but adequate for many purposes. Use large, high-contrast symbols in web maps for mobile collection — small screens and bright sunlight make small symbols hard to see. Coded domains are essential for field collection: faster entry, fewer typos, consistent values. Always create domains before publishing.

Sharing Workflows — ModelBuilder, Scripts & Tasks

▶

A workflow is a stored sequence of data management or analysis steps. Sharing workflows makes analysis faster, more reliable, and reproducible.

ModelBuilder

Graphical canvas to link inputs (blue ovals) and tools (yellow squares) into a sequence. Green ovals = outputs. Stores parameters for automatic execution. Supports iteration, conditions, and variables. Can be configured as a tool with user-specified parameters. Models can be saved as Python scripts for further customization.

Python Scripts

Computer programs that call GIS functions and tools directly. More flexible than ModelBuilder for complex logic. Models can be exported as starting-point scripts. Best for automation, batch processing, and custom tools.

Tasks

Interactive step-by-step workflows where users make decisions. Each step auto-launches the next tool. Provides instructions guiding the user. Good for training, standardizing complex multi-step processes, or workflows requiring human judgment at each step.

Why use ModelBuilder?

  • Build and visualize complex analysis sequences graphically
  • Store processing steps for documentation and future reference
  • Execute the same workflow repeatedly with different inputs
  • Share workflows with colleagues so they can replicate your analysis
  • Set parameters so users can customize inputs without editing the model
