Array Basics

▶

An array is a variable that stores multiple values of the same type in adjacent memory locations. All elements share one name, distinguished by a numeric subscript (index).

// Definition
int tests[5];              // array of 5 integers
const int SIZE = 5;
int scores[SIZE];          // use named constant for size — best practice

// Subscripts start at 0, end at size-1
tests[0] = 79;             // first element
tests[4] = 84;             // last element (index 4 for a 5-element array)

// Access like any variable
cout << tests[0];
cin >> tests[1];
tests[4] = tests[0] + tests[1];

// Use integer expression as subscript
int i = 3;
cout << tests[i];          // accesses tests[3]

C++ does NOT check array bounds — using an out-of-range subscript silently corrupts other memory, crashes the program, or causes unpredictable bugs. Always stay within [0, size-1]. The classic off-by-one error: starting subscripts at 1 or using <= SIZE instead of < SIZE in a loop.

Array size in bytes = number of elements × bytes per element. A 5-element int array uses 20 bytes (5 × 4).

Array Initialization

▶ 
// Initialize with a list
int tests[5] = {79, 82, 91, 77, 84};

// Partial initialization — remaining elements set to 0
int tests[5] = {79, 82};    // tests[2], [3], [4] = 0

// Implicit size — compiler counts the list
int quizzes[] = {12, 17, 15, 11};   // size = 4

// Global arrays: all elements initialized to 0 automatically
// Local arrays: uninitialized — must set values before use!

Initialization list cannot exceed the array size. If you provide fewer values than the size, the remaining elements are zero-initialized. Use named constants for size declarators — if you need to change the array size later, you only change one line.

Loop Processing

▶

Loops are the primary tool for processing arrays. Use the regular for loop when you need the subscript, and the range-based for loop (C++11) when you don't.

const int SIZE = 5;
int numbers[SIZE];

// Regular for loop — use when you need the index
for (int i = 0; i < SIZE; i++)
    numbers[i] = 99;

// Range-based for loop (C++11) — read-only
for (int val : numbers)
    cout << val << endl;

// Range-based for loop with reference — modifies original
for (int &val : numbers)
    cin >> val;

// auto keyword with range-based for
for (auto &val : numbers)
    cin >> val;

The range-based for loop automatically knows the array size and stays in bounds — no counter, no off-by-one errors. However, it ONLY works with the full array — it can't process a partial array or process elements in reverse. For those cases, use the regular for loop. Without the & reference, the range variable is a copy — modifying it does NOT change the array.

Common Array Operations

▶ 
// Sum and average
double sum = 0;
for (int i = 0; i < SIZE; i++)
    sum += scores[i];
double average = sum / SIZE;

// Find highest value (seed with first element)
int highest = numbers[0];
for (int i = 1; i < SIZE; i++)
    if (numbers[i] > highest)
        highest = numbers[i];

// Find lowest value
int lowest = numbers[0];
for (int i = 1; i < SIZE; i++)
    if (numbers[i] < lowest)
        lowest = numbers[i];

// Copy array element-by-element (cannot use = on whole array)
for (int i = 0; i < SIZE; i++)
    newTests[i] = tests[i];
// newTests = tests;   // WRONG — will not work!

// Compare two arrays element-by-element
bool equal = true;
for (int i = 0; i < SIZE; i++)
    if (first[i] != second[i])
        equal = false;

Always seed min/max searches with the first element (index 0), not 0 or INT_MAX — that way the code works correctly regardless of whether the values are positive, negative, or mixed. Don't confuse tests[i]++ (increment the element) with tests[i++] (increment the index, leave element unchanged).

Arrays as Function Arguments

▶

Pass an array to a function using just the array name. Always pass the size separately — the function has no other way to know how many elements to process.

// Prototype: empty [] for array parameter
void showScores(int [], int);

// Header: parameter name + []
void showScores(int tests[], int size)
{
    for (int i = 0; i < size; i++)
        cout << tests[i] << endl;
}

// Call: just the array name (no [] or subscript)
showScores(tests, SIZE);

Arrays are NOT passed by value — they behave like reference parameters. The function receives a pointer to the original array, so changes made inside the function affect the original. This is unlike regular variables which get copied. If you don't want the function to modify the array, declare the parameter as const: void showScores(const int [], int);

Two-Dimensional Arrays

▶

A 2D array is like a table — rows and columns. Two size declarators: first is rows, second is columns. Two subscripts to access an element.

const int ROWS = 4, COLS = 3;
int exams[ROWS][COLS];

// Access: [row][col]
exams[2][2] = 86;

// Initialization — row by row
int exams[2][2] = { {84, 78},
                    {92, 97} };

// Process with nested loops
for (int row = 0; row < ROWS; row++)
    for (int col = 0; col < COLS; col++)
        cout << exams[row][col] << " ";

// Sum all elements
int total = 0;
for (int row = 0; row < ROWS; row++)
    for (int col = 0; col < COLS; col++)
        total += numbers[row][col];

// Sum each row (e.g., student averages)
for (int row = 0; row < NUM_STUDENTS; row++)
{
    double rowTotal = 0;
    for (int col = 0; col < NUM_SCORES; col++)
        rowTotal += scores[row][col];
    cout << "Average: " << rowTotal / NUM_SCORES;
}

// Function prototype — must specify all dimensions except first
void getExams(int [][COLS], int rows);

Parallel Arrays

▶

Parallel arrays are two or more arrays where elements at the same subscript are related. They can be of different types.

const int SIZE = 5;
int    id[SIZE];        // student ID
double average[SIZE];   // course average
char   grade[SIZE];     // letter grade

// Elements at index i all belong to the same student
for (int i = 0; i < SIZE; i++)
{
    cout << "Student ID: " << id[i]
         << " Average: "   << average[i]
         << " Grade: "     << grade[i] << endl;
}

Parallel arrays are a simple but effective way to associate related data before you learn about structs or classes. The shared subscript is the relationship — id[2], average[2], and grade[2] all describe the same student.

STL Vectors

▶

Vectors are from the Standard Template Library — they work like arrays but grow automatically as needed. Requires #include <vector>.

#include <vector>

// Declare vectors
vector<int> scores;           // empty, size 0
vector<int> scores(30);       // 30 elements
vector<int> scores(30, 0);    // 30 elements, all initialized to 0
vector<int> finals(scores);   // copy of another vector
vector<int> numbers {10, 20, 30, 40}; // C++11 initializer list

// Add / remove elements
scores.push_back(75);    // add to end
scores.pop_back();       // remove from end
scores.clear();          // remove all elements

// Query
int size = scores.size();      // number of elements
bool isEmpty = scores.empty(); // true if size == 0

// Access like an array
scores[0] = 100;
cout << scores[2];

// Range-based for loop
for (int val : scores)
    cout << val << endl;

Vectors solve the biggest problem with arrays — you don't need to know the size at compile time. They grow automatically with push_back(). Use vectors when the number of elements isn't known in advance. Use arrays when the size is fixed and known — arrays have slightly less overhead. Vectors are covered in more depth in Chapter 17.
