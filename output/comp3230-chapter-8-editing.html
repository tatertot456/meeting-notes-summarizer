Editing & Topology Fundamentals

▶

Editing refers to updating existing feature classes or creating new ones — creating new features, modifying spatial geometry, or editing attribute values.

Topology refers to the spatial relationships between features. Ideal topology matches the real world:

  • Roads actually connect at intersections
  • Adjacent polygons share identical boundaries without gaps or overlaps
  • Features within features are contained properly

Schema changes (adding/deleting fields, modifying domains) and editing changes (modifying coordinates or attributes) BOTH place a file lock on the feature class until saved. You CANNOT edit features when there are unsaved schema changes, and vice versa. Always save schema changes before beginning a feature editing session.

Essential editing tips:

  • Always keep a backup copy of your data before editing
  • Use simple maps with only the layers you need — reduces accidental changes and improves performance
  • Save edits often — you cannot Undo actions that occurred before the most recent save
  • Enable time-dependent auto-saving (every X minutes) as an extra safety net
  • If the map CS differs from the data CS, edited coordinates are converted on the fly and stored in the DATA coordinate system

Snapping

▶

Snapping ensures the ends of roads or vertices of polygons connect exactly. When a cursor moves within the snap tolerance of an existing feature, the new feature snaps to it precisely. Without snapping, roads and polygon boundaries may appear connected visually but have gaps that break topology.

Snap TypeSnaps ToUse When
End SnappingNode (endpoint) of an existing line onlyConnecting road endpoints — ensures roads meet at their ends
Edge SnappingAnywhere along an existing lineWhen a line needs to connect at any point along another line
Vertex SnappingOnly to vertices (shape points) of an existing lineConnecting at existing shape points, not arbitrary positions

Snapping controls in ArcGIS Pro:

  • Snapping group on the Edit ribbon — turn on/off, set type and options
  • Popup button below the map for quick access
  • Contents pane → List By Snapping — enable/disable snapping per layer and set priority order
  • Snap tolerance = the distance at which snapping activates (set in snapping options)

Creating Features

▶

The first step is often creating a new feature class with the Create Feature Class tool — specify name, location, geometry type (point/line/polygon), and coordinate system. Type cannot be changed later.

Create Features pane — used to create new features:

  • Contains templates that specify basic settings and construction tools
  • A layer can have multiple templates (e.g., Commercial Buildings and Residential Buildings) that automatically assign appropriate default attributes
  • Select a template and construction tool to begin creating
  • Active Template mode shows the settings and attributes for the current template

Sketches — provisional features used during creation or modification:

  • Show vertices and segments between them before they are finalized
  • Construction toolbar: modify segment type, create corners or curves, trace existing features, finish or cancel the sketch
  • A sketch is finalized (committed) when you double-click or press F2

Modifying Features & Attributes

▶

Modify Features pane — tools for changing existing features:

  • Edit individual vertices of a feature
  • Move, scale, rotate, or align features
  • Reshape a feature by re-digitizing a section (not the whole feature)
  • Many tools have an Edges tab for topology-aware editing

Attributes pane — aids editing attribute values:

  • Shows currently selected features and their attributes
  • Click a feature to edit its attributes individually
  • Right-click a feature for a context menu with useful functions
  • Click the layer name to change a value in ALL selected features simultaneously

Polygon Topology & Map Topology

▶

Adjacent polygons (parcels, land cover) should share identical boundaries with no gaps or overlaps. Three techniques for maintaining topologically correct adjacent polygons:

Autocomplete Polygon Tool

Start a new polygon by clicking INSIDE an existing polygon, digitize the new boundary, end INSIDE the existing polygon. Creates two polygons with perfectly coincident shared boundaries. The new section must completely enclose an area.

Split Tool

Draw a line across an existing polygon to split it into two polygons with coincident shared boundaries. If multiple polygons are selected, the split applies to all of them simultaneously.

Map Topology

Tracks shared boundaries (edges) and connected points (nodes). Edit ribbon → Manage Edits. When editing with the Edges tab, both adjacent polygons update together — no gap is created. Essential for maintaining correct topology during edits.

Topology-Aware Line Editing

Lines connect at nodes. When using Edges tab in Modify tools, moving a node keeps all connected lines attached. "Stretch Geometry Proportionally" option moves connected lines smoothly instead of creating abrupt bends.

If you edit one polygon without using map topology, the adjacent polygon does NOT update — creating a gap or overlap. Always use the Autocomplete Polygon tool, Split tool, or map topology (Edges tab) when editing adjacent polygons. Regular editing of individual polygons will break topological relationships.

Annotation

▶

Annotation is a set of labels stored as features in a geodatabase feature class. Unlike dynamic labels (which regenerate automatically), annotation is persistent and individually editable. Each label has attributes for font, size, style, position, and angle.

Advantages over Dynamic Labels

Each label can be moved, rotated, or reformatted individually. Can be used across many maps and projects. Unplaced annotation can be created for labels that don't fit and edited later.

Reference Scale

Annotation is given a reference scale so labels maintain consistent size relative to map features. At reference scale: labels appear at assigned size. Zoom out: labels shrink. Zoom in: labels grow.

Creating annotation:

  • Set up dynamic labels to appear as desired at the reference (conversion) scale
  • Convert dynamic labels to annotation stored in a geodatabase feature class
  • Labels that don't fit become unplaced annotation — edit their position manually
  • Use the Create Features pane to create individual annotation features from scratch
  • Construction tools: horizontal, straight, or curved annotation following existing features
