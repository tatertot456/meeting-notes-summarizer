The cin Object

▶

cin is the standard input object. Like cout, it requires the iostream header. Use the stream extraction operator >> to read input from the keyboard into variables. cin automatically converts input to match the variable's data type.

int height;
cout << "How tall is the room? ";  // always prompt before cin
cin >> height;

// Read multiple values at once (separated by spaces when entered)
cin >> height >> width;

Always display a prompt with cout BEFORE every cin statement — otherwise the user stares at a blank screen with no idea what to type. When reading multiple values, the user separates them with spaces. Order matters: first value entered → first variable, second → second, and so on.

Mathematical Expressions & Order of Operations

▶

An expression can be a literal, a variable, or a mathematical combination of constants and variables. Expressions can appear in assignments, cout statements, and anywhere a value is expected.

area = 2 * PI * radius;
cout << "border is: " << 2 * (l + w);

Order of operations (precedence, highest to lowest):

  1. Unary negation (-), right to left
  2. *, /, % — left to right
  3. +, - — left to right
2 + 2 * 2 - 2   = 4   // * first: 2 + 4 - 2 = 4
(2 + 2) * 2 - 2 = 6   // parentheses first: 4*2-2 = 6
2 + 2 * (2 - 2) = 2   // parentheses first: 2+2*0 = 2
(2 + 2) * (2 - 2) = 0 // both parentheses: 4*0 = 0

Algebraic expressions must be explicitly written out: Area = lw becomes Area = l * w;. There is no exponentiation operator in C++ — use pow(s, 2) from the cmath library instead of s². Parentheses are your best tool for controlling evaluation order.

Type Conversion & Casting

▶

Operations are performed between operands of the same type. When types differ, C++ automatically converts one — this is called type coercion.

Type hierarchy (highest to lowest): long double → double → float → unsigned long → long → unsigned int → int

Coercion rules:

  • char, short, unsigned short are automatically promoted to int
  • When operating on different types, the lower type is promoted to the higher
  • With =, the right side is converted to the type of the left side variable

Integer division trap: if both operands are int, / returns only the whole number quotient. To force floating point division, cast one operand: static_cast<double>(y2-y1) / (x2-x1). Overflow occurs when a value is too large for its variable type — the value wraps around unpredictably.

// static_cast: preferred modern C++ style
double m;
m = static_cast<double>(y2 - y1) / (x2 - x1);

// See the ASCII value of a char
char ch = 'C';
cout << ch << " is " << static_cast<int>(ch);  // C is 67

// C-style cast (older, still supported)
cout << (int)ch;

// Prestandard C++ cast (older, still supported)
cout << int(ch);

Assignment & Combined Assignment Operators

▶

The = operator assigns values right to left. You can chain assignments:

x = y = z = 5;   // assigns 5 to z, then y, then x (right to left)

Combined assignment operators provide shorthand for modifying a variable:

OperatorExampleEquivalent To
+=sum += 1sum = sum + 1
-=sum -= 1sum = sum - 1
*=sum *= 2sum = sum * 2
/=sum /= 2sum = sum / 2
%=sum %= 2sum = sum % 2

Formatting Output

▶

Requires #include <iomanip>. Stream manipulators control how output is displayed.

ManipulatorEffectPersists?
setw(x)Print in a field at least x spaces wideNo — next value only
fixedUse decimal notation for floating-pointYes
setprecision(x)With fixed: x digits after decimal. Without: x significant digitsYes
showpointAlways show decimal point for floatsYes
#include <iomanip>

cout << setw(10) << "Name" << setw(10) << "Score";

double price = 14.5;
cout << fixed << setprecision(2) << price;  // displays 14.50

setw() only affects the next value printed — you must repeat it for each value. fixed, setprecision(), and showpoint stay in effect until changed — set them once and they apply to all subsequent floating-point output.

String & Character Input

▶

Using cin >> for strings causes problems — it stops at whitespace, so "John Smith" would only read "John". Use getline instead:

string fullName;
getline(cin, fullName);   // reads the entire line including spaces

Reading a single character:

char ch;
cin >> ch;       // skips whitespace — won't read spaces/tabs/enter
cin.get(ch);     // reads the very next character, including whitespace

Mixing cin >> and cin.get() in the same program causes subtle bugs — the enter key left in the buffer by cin >> gets read by cin.get() as a character. Use cin.ignore() to skip leftover characters: cin.ignore(); skips the next character, cin.ignore(10, '\n'); skips up to 10 characters or until a newline.

String operations:

string state = "Texas";
int size = state.length();         // get length

string greeting1 = "Hello, ";
string name = "World";
string result = greeting1 + name;  // concatenate with +
greeting1 += name;                 // concatenate with +=

Mathematical Library Functions

▶

cmath functions (#include <cmath>)

Take double input, return double. Common functions: pow(base, exp), sqrt(x), abs(x), ceil(x), floor(x), sin(x), cos(x), log(x).

cstdlib functions (#include <cstdlib>)

rand(): returns random int 0 to max int. Same sequence every run. srand(x): seeds the generator with x — use srand(time(0)) for different sequences each run.

#include <cmath>
double area = pow(s, 2);     // s squared
double hyp = sqrt(a*a + b*b);

#include <cstdlib>
#include <ctime>
srand(time(0));              // seed with current time
int roll = rand() % 6 + 1;  // random number 1-6

Hand Tracing & Case Study

▶

Hand tracing means acting as the computer — stepping through each statement one by one and recording variable values in a trace chart. It's the most reliable way to find logic and math errors without running the program.

Case study — crate calculator: General Crates Inc. needs a program to calculate volume, cost, charge, and profit for custom wooden crates.

// Formulas used:
volume = length * width * height;
cost   = volume * 0.23;
charge = volume * 0.50;
profit = charge - cost;

// Program structure follows IPO:
// Input:   get length, width, height from user
// Process: calculate volume, cost, charge, profit
// Output:  display all four calculated values

The IPO pattern (Input → Process → Output) is the foundation of most programs. Notice the case study follows it exactly: gather all inputs first, then do all calculations, then display all results. Mixing these phases makes programs harder to read and debug.
