A Brief History of GIS

▶

GIS evolved from manual map-making to powerful digital spatial analysis platforms over roughly 150 years.

YearMilestone
1854John Snow maps cholera outbreak in London — removes pump handle to stop spread. First spatial disease analysis.
Early 1960sCanada Land Inventory → Canada Geographic Information System (CGIS) — first true digital GIS.
1969Ian McHarg's Design with Nature introduces map overlay for suitability analysis.
1970sU.S. Census Bureau's DIME (Dual Independent Map Encoding) introduces urban street network topology.
1990sMainstream desktop GIS — affordable hardware brings GIS to organizations of all sizes.
TodayIntegration with satellite imagery, GPS, web mapping, social media, and cloud computing.

Types of Map Data & GIS Data Models

▶

Discrete Data

Objects exist in defined locations. Examples: lookout tower (point), stream (line), snail habitat (polygon). Best represented with the vector model.

Continuous Data

Data exists everywhere — no hard boundaries. Examples: elevation, temperature, precipitation. Best represented with the raster model.

Vector Data Model — stores geographic features as points, lines, and polygons:

  • Features are stored as map objects with precise coordinates
  • A feature class is a collection of similar features stored together (e.g., all rivers, all state boundaries)
  • Each feature is linked to an attribute table by a unique integer ID
  • Best for discrete features: boundaries, roads, buildings

Raster Data Model — divides the map into a grid of cells (pixels):

  • Each cell stores a single numeric value (elevation, pixel color, temperature)
  • Best for continuous data: elevation models, aerial/satellite photography
  • Resolution = cell size — smaller cells = more detail but larger file size

Choose your model based on your data type. Vector is precise for discrete objects and has small file sizes for complex boundaries. Raster handles continuous surfaces naturally but can be large. Many GIS workflows combine both — a vector road network overlaid on a raster elevation model, for example.

Map Scale & Generalization

▶

Map scale is the ratio of distance on the map to the corresponding distance on the ground. It is dimensionless — works in any unit (cm, inches, mm).

1:100,000 means 1 cm on map = 100,000 cm (1 km) on the ground Large scale = zoomed in, high detail (1:1,000 — 1 cm = 10 m) Small scale = zoomed out, low detail (1:1,000,000 — 1 cm = 10 km) Estimating precision from scale: A 1-point line is ~0.001 feet thick on paper Map scale ÷ 1,000 ≈ precision in feet Example: 1:24,000 scale → precision ≈ 24 feet

Generalization simplifies features for display at smaller scales. Small-scale maps show less detail; large-scale maps show more.

Source scale is the scale at which data were originally captured — it determines the inherent detail and accuracy of the dataset:

  • Data should not be displayed at scales much different from the source scale
  • Two datasets of the same features may not align if captured at different source scales
  • Larger source scale = finer sampling resolution = more accurate for large-scale display

Source scale mismatch is a real problem. Oregon county boundaries from two different data sources may not align at all. Always check the source scale metadata before combining datasets. Displaying data at a scale far finer than the source scale creates a false impression of precision that doesn't exist in the data.

Data Quality

▶

No dataset is perfect. Quality is defined as fitness for a particular purpose — the same dataset may be inadequate for one use but perfectly acceptable for another. Users have an ethical and legal responsibility to assess fitness.

Quality DimensionQuestionExample Issue
Geometric AccuracyIs it where it says it is?Road mapped from satellite may be off by 10m from GPS survey. Both have errors.
Thematic AccuracyAre the attribute values correct?How was canopy cover measured? What are the error sources in the measurements?
Logical (Topological) ConsistencyDo relationships make sense?Do roads actually connect at intersections? Do shared county/district boundaries align exactly?
ResolutionHow finely was data sampled?Spatial: GPS point spacing. Temporal: daily vs. decadal. Thematic: measuring in meters vs. km.
PrecisionHow repeatable is the measurement?20 GPS readings at same spot have 5m std dev → precision ≈ ±10m. Not the same as accuracy!

Precision ≠ Accuracy. A GPS that always reports the same wrong location is precise but inaccurate. A measurement that averages to the right value but varies widely is accurate but imprecise. GIS data can be highly precise (many decimal places) but geometrically inaccurate. Always evaluate both independently.

Data Acquisition & Citation

▶

Sources for GIS data:

  • Government agency websites (USGS, Census Bureau, state agencies)
  • Digitizing from paper maps or orthophotos
  • Creating from satellite imagery or drone flights
  • Converting GPS tracks, survey data, street addresses, or coordinate text files

Map images and satellite photos must undergo georeferencing (geometric transformation) before use in GIS. All new and existing data must be edited to correct digitizing and topological errors.

Citing GIS data — professional and ethical responsibility:

Format:
Data set name (Year published) [source type]. Producer name,
producer contact information. Resource URL: [Date accessed].

Examples:
National Hydrography Dataset (2015) [downloaded file].
  United States Geological Survey on the National Map Viewer.
  URL: http://viewer.nationalmap.gov/viewer/ [July 23, 2015].

USA Topo Maps (2009) [map service]. Esri on ArcGIS Online.
  URL: http://server.arcgisonline.com/arcgis/services/
  USA_Topo_Maps/MapServer [January 1, 2012].

Use Zotero to manage GIS data citations! A good citation enables anyone to find and obtain the original source. This matters because datasets get updated, moved, or deprecated — documenting exactly what you used and when protects the reproducibility of your analysis.

ArcGIS Pro Interface

▶

ArcGIS has evolved from Arc/Info → ArcMap → ArcGIS Pro. It is available at three license levels: Basic, Standard, and Advanced (increasing cost and functionality).

Projects

ArcGIS Pro organizes work into projects containing: maps, tools, databases, layouts, styles, folders, models, tasks, and locators.

Panes

Contain commands and settings. Movable and stackable. May have tabs and panels. Key panes: Contents, Catalog, Geoprocessing, Symbology.

Ribbon

Buttons and functions organized into logical groups. Some ribbons always visible; others appear contextually (when a layer is selected, etc.).

Contents Pane Panels

List by Drawing Order, Source, Selection, Editing, Snapping, Labeling, Charts — different views of the layer stack.

Geoprocessing Pane

Access toolboxes and tools for advanced analysis and processing. Extends capabilities far beyond the GUI menus.

Catalog Pane

Four tabs: Project (local resources), Portal (ArcGIS Online), Favorites (saved locations), Notifications (software updates, etc.).

Key tools:

  • Summary Statistics — calculates statistics from table attributes; user chooses fields and statistic types; creates new output table
  • Select Layer By Attribute — selects features based on table values; highlights selection in both table and map
  • Scale Ranges — layer property that shows/hides a layer at specific scales (detailed symbols at large scale, generalized at small)
  • Full Extent — customizable; can be set to a layer, current display, or world default

Warnings vs. Errors: Warnings indicate potential issues but allow the tool to run. Errors prevent the tool from running at all. Always read both — warnings may indicate your output is technically produced but meaningless. The Help system is web-based: breadcrumb trail + search box + tabbed sections.

GIS Services & Web Maps: GIS servers provide data over the Internet to multiple platforms. Clients consume only what they need. Enables real-time updates. Portal tab in Catalog pane accesses web maps, feature layers, imagery layers, and map image layers from ArcGIS Online or other servers.

Layers, Groups & Symbology

▶

Source data becomes a layer when added to a map. Layers store display properties and settings — the same source file can appear very differently in different maps.

Layers

Store properties and settings for display. Each layer's symbols and settings can be independently customized. Layer properties do NOT change the source data.

Group Layers

Multiple layers combined together. Layers can be toggled on/off individually or as a group. Can be saved as layer files to quickly transfer display settings to new data.

Symbology Pane

Gallery tab: choose a symbol from available styles. Properties tab: fine-tune the chosen symbol's settings (color, size, outline, fill, etc.).

Pane Management

Panes and views can be positioned: Adjacent, Stacked, or Floating. Use the docking icon to drag and reposition. Views (maps, layouts) are separate from panes.
