Θ (Theta) Notation — Tight Bound

▶

Θ(g(n)) describes a tight bound — the function grows exactly as fast as g(n), up to constant factors.

Formal definition: f(n) = Θ(g(n)) if there exist positive constants c₁, c₂, and n₀ such that:

0 ≤ c₁·g(n) ≤ f(n) ≤ c₂·g(n) for all n ≥ n₀

Θ means "sandwiched between" — g(n) is both an upper and lower bound on f(n). When we say T(n) = Θ(n²), the running time grows neither faster nor slower than n² asymptotically.

O (Big-O) Notation — Upper Bound

▶

O(g(n)) describes an upper bound — the function grows no faster than g(n).

Formal definition: f(n) = O(g(n)) if there exist positive constants c and n₀ such that:

0 ≤ f(n) ≤ c·g(n) for all n ≥ n₀

Important note: O is not the same as Θ. Saying an algorithm is O(n²) means it runs in at most n² time — it might be faster. Θ(n²) means it is exactly that tight.

O notation is most commonly used in practice because we care most about worst-case upper bounds.

Ω (Omega) Notation — Lower Bound

▶

Ω(g(n)) describes a lower bound — the function grows at least as fast as g(n).

Formal definition: f(n) = Ω(g(n)) if there exist positive constants c and n₀ such that:

0 ≤ c·g(n) ≤ f(n) for all n ≥ n₀

Theorem 3.1: f(n) = Θ(g(n)) if and only if f(n) = O(g(n)) AND f(n) = Ω(g(n)). Theta is the intersection of O and Ω.

Ω is useful for proving lower bounds — e.g., any comparison-based sort requires Ω(n log n) comparisons.

Little-o and Little-ω Notation

▶

o (little-o) — Strict Upper Bound

f(n) = o(g(n)) means f grows strictly slower than g. The ratio f(n)/g(n) → 0 as n → ∞. Example: n = o(n²).

ω (little-omega) — Strict Lower Bound

f(n) = ω(g(n)) means f grows strictly faster than g. Example: n^2.0001 = ω(n²). But n² ≠ ω(n²).

Analogy with real numbers: O is like ≤, Ω is like ≥, Θ is like =, o is like <, ω is like >. Little notations are strict — no equality allowed.
