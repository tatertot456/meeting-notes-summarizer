Insertion Sort

▶

Insertion Sort works the way you sort a hand of playing cards — pick up one card at a time and insert it into its correct position among already-sorted cards.

INSERTION-SORT(A)
for j = 2 to A.length
    key = A[j]
    i = j - 1
    while i > 0 and A[i] > key
        A[i+1] = A[i]
        i = i - 1
    A[i+1] = key

Best Case

Array already sorted — inner while loop never executes. Runtime: Θ(n).

Worst Case

Array reverse sorted — every element must shift. Runtime: Θ(n²).

Loop Invariants & Correctness

▶

A loop invariant is a property that is true before and after each iteration of a loop. It is the formal tool for proving an algorithm correct.

Three conditions must hold:

  • Initialization: The invariant is true before the first iteration
  • Maintenance: If true before an iteration, it remains true after
  • Termination: When the loop ends, the invariant gives us a useful property that proves correctness

For Insertion Sort: the invariant is that A[1..j-1] always contains the original elements of A[1..j-1] in sorted order. At termination (j = n+1), the entire array is sorted.

Analyzing Insertion Sort

▶

We count the number of primitive operations each line executes as a function of n (input size):

  • The outer for loop runs n-1 times
  • The inner while loop runs between 0 and j-1 times depending on input
  • Summing over all iterations gives a quadratic function in the worst case

Order of Growth: we care about the leading term only. Θ(n²) means the running time grows proportionally to n² as n gets large — constants and lower-order terms are ignored.

Why best/worst case matters: guaranteed performance bounds tell you the limits of what an algorithm can do on any input, not just typical ones.

Merge Sort & Divide and Conquer

▶

Merge Sort uses the divide and conquer paradigm:

  • Divide: Split the array into two halves
  • Conquer: Recursively sort each half
  • Combine: Merge the two sorted halves into one sorted array
MERGE-SORT(A, p, r)
if p < r
    q = floor((p + r) / 2)
    MERGE-SORT(A, p, q)
    MERGE-SORT(A, q+1, r)
    MERGE(A, p, q, r)

Merge Sort runtime: T(n) = 2T(n/2) + Θ(n) → Θ(n log n). Dramatically better than Insertion Sort's Θ(n²) for large inputs.

The key insight: merging two sorted arrays of size n/2 takes Θ(n) time, and we do this at log n levels of recursion.

Solving Recursion

▶

Recursive algorithms produce recurrence relations — equations that describe T(n) in terms of T on smaller inputs. Three methods to solve them:

Substitution Method

Guess the form of the solution, then use mathematical induction to prove it correct.

Recursion Tree Method

Draw the recursion as a tree, sum costs at each level to find total cost.

Master Theorem

A formula-based approach for recurrences of the form T(n) = aT(n/b) + f(n). Covered in Chapter 4.

Why It Matters

Solving recurrences tells you the actual runtime of divide-and-conquer algorithms without tracing every recursive call.
