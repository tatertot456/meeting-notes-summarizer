Modular Programming

▶

Modular programming breaks a program into smaller, manageable functions. A function is a named collection of statements that performs a specific task.

Improves Maintainability

Fix or change a function in one place — every caller benefits automatically. No need to hunt through thousands of lines of code.

Simplifies Development

Break complex problems into smaller pieces. Write, test, and debug each piece independently before combining them.

Enables Reuse

Write a function once, call it many times from many places — even across different programs.

Stubs & Drivers

Stub: dummy function that stands in during development. Driver: test function that calls another function with various inputs to verify it works correctly.

Defining and Calling Functions

▶

A function definition has four parts: return type, name, parameter list, and body.

// void function: returns nothing
void printHeading()
{
    cout << "Monthly Sales\n";
}

// int function: returns an integer
int sum(int num1, int num2)
{
    return num1 + num2;
}

// Calling functions
printHeading();           // call with no arguments
int total = sum(5, 3);    // call with arguments, capture return value

When a function is called, execution jumps to the function body. After the function ends (via return or reaching the last }), execution resumes in the calling function at the exact point of the call. main() can call any number of functions, and functions can call other functions.

Function Prototypes

▶

The compiler must know a function's name, return type, and parameter types before it can be called. A prototype provides this information without the full body.

// Prototype: like the header with a semicolon instead of body
void printHeading();
int sum(int, int);           // parameter names optional in prototype
bool isValid(int, double);

int main()
{
    printHeading();          // compiler knows about it from prototype
    int total = sum(5, 3);
}

// Full definitions can appear anywhere after the prototypes
void printHeading()
{
    cout << "Monthly Sales\n";
}

int sum(int num1, int num2)
{
    return num1 + num2;
}

Place prototypes near the top of the file, before main(). Without a prototype or prior definition, calling a function causes a compiler error. With prototypes, you can organize function definitions in any order — most programmers put main() first and definitions after.

Parameters & Pass by Value

▶

Arguments are values passed at the call site. Parameters are the variables inside the function that receive those values. By default, C++ passes by value — a COPY is made.

// Function with parameters
void displayValue(int num)       // num is the parameter
{
    cout << "Value: " << num << endl;
    num = 99;    // only changes the LOCAL copy — caller unaffected
}

// Multiple parameters
void showData(int x, double y, char z)
{
    cout << x << " " << y << " " << z;
}

// Calling with arguments
int val = 5;
displayValue(val);    // passes a COPY of val — val still 5 after call
showData(10, 3.14, 'A');

Pass by value means the function gets its own copy of the data. Changes to the parameter inside the function have NO effect on the original argument. Arguments are promoted/demoted as needed to match parameter types. The number and order of arguments must exactly match the parameter list.

Return Values

▶ 
// return a computed value
int sum(int num1, int num2)
{
    return num1 + num2;   // can return an expression directly
}

// return a bool
bool isEven(int num)
{
    return (num % 2 == 0);   // returns true or false
}

// Use return values in calling code
int total = sum(5, 3);               // assign to variable
cout << sum(10, 20);                 // use in cout
if (isEven(val))                     // use in condition
    cout << "Even\n";

// Early return: exit function before reaching the end
void checkAge(int age)
{
    if (age < 0)
    {
        cout << "Invalid age\n";
        return;    // exits function immediately
    }
    cout << "Age is " << age;
}

The return type in the prototype and definition must match. A void function doesn't need a return statement — it ends at the closing brace. A return statement in a void function simply exits early. Statements after a return never execute.

Scope, Lifetime & Global Variables

▶

Local Variables

Defined inside a function. Only accessible within that function. Created when function starts, destroyed when it ends. NOT automatically initialized — must be set by the programmer.

Global Variables

Defined outside all functions. Accessible by any function defined after them. Auto-initialized to 0/NULL. Avoid — make debugging very difficult. Use global CONSTANTS instead.

Static Local Variables

Declared with static keyword. Retain their value between function calls. Initialized only once (first call). Default value is 0.

Global Constants

The one legitimate use of global scope. const double PI = 3.14159; defined globally makes it available to all functions without passing it as a parameter.

const double TAX_RATE = 0.0675;  // global constant — OK

void countCalls()
{
    static int count = 0;    // retains value between calls
    count++;
    cout << "Called " << count << " times\n";
}

// Two functions can have variables with the same name
void funcA() { int x = 10; }   // this x
void funcB() { int x = 20; }   // different x — completely separate

Default Arguments & exit()

▶

Default arguments are automatically used when an argument is omitted from the call. Must be declared in the prototype, must appear at the END of the parameter list.

// Default arguments in prototype
int getSum(int, int = 0, int = 0);   // last two have defaults

// Valid calls
sum = getSum(num1);           // uses 0 for 2nd and 3rd
sum = getSum(num1, num2);     // uses 0 for 3rd only
sum = getSum(num1, num2, num3); // no defaults used

// INVALID — can't skip middle arguments
sum = getSum(num1, , num3);   // ERROR

// INVALID — defaultless parameters must come first
int getSum(int = 0, int, int);  // ERROR
// exit() — terminate program from any function
#include <cstdlib>

if (criticalError)
{
    cout << "Fatal error — terminating\n";
    exit(EXIT_FAILURE);   // or exit(0) for success
}

Reference Parameters (Pass by Reference)

▶

A reference parameter is an alias for the original variable — changes inside the function affect the original. Declared with &. Allows a function to "return" multiple values.

// & in prototype and header
void getDimensions(int& length, int& width);

void getDimensions(int& length, int& width)
{
    cout << "Enter length: ";
    cin >> length;    // modifies the ORIGINAL variable
    cout << "Enter width: ";
    cin >> width;     // modifies the ORIGINAL variable
}

int main()
{
    int l, w;
    getDimensions(l, w);   // l and w are changed by the function
    cout << l << " x " << w;
}

Rules for reference parameters: must use & in BOTH prototype and header. The argument passed must be a variable — not a literal or expression (5 or num1+num2 cannot be passed by reference). Only use reference parameters when the function needs to modify the original — use pass by value otherwise.

Function Overloading

▶

Overloaded functions share the same name but have different parameter lists. The compiler selects the correct version based on the argument types and count at the call site.

// Four overloaded versions of getDimensions
void getDimensions(int);             // 1 int
void getDimensions(int, int);        // 2 ints
void getDimensions(int, double);     // int + double
void getDimensions(double, double);  // 2 doubles

// Compiler picks the right one automatically
int length, width;
double base, height;

getDimensions(length);           // calls version 1
getDimensions(length, width);    // calls version 2
getDimensions(length, height);   // calls version 3
getDimensions(height, base);     // calls version 4

Overloading lets you use one intuitive name for related operations on different data types — like how the + operator works on both ints and doubles. The compiler distinguishes versions by their signature (parameter count and types) — NOT by return type alone.
