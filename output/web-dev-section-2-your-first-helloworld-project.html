What Is Blazor WebAssembly?

▶

Rather than writing raw HTML, modern web development uses frameworks — sets of libraries that handle common tasks in a simpler, more powerful format. Frameworks combine HTML, JavaScript, and CSS with built-in components and utilities. They became popular because they make websites more sophisticated and developers more productive.

Blazor is Microsoft's web framework with a key advantage: it uses C# instead of JavaScript (most of the time).

Runs .NET in the Browser

Blazor WebAssembly executes C# code directly in the browser — no server round-trips needed for most interactions.

Full-Stack Capable

Handles both client-side (what the user sees) and server-side (data, logic) components in one framework.

Rich Component Library

Comes with a wide set of free (or mostly free) prebuilt components — grids, charts, forms, and more.

Platform Independent

Works across operating systems and browsers. Performance-focused by design — a common pain point for web apps.

Project Setup

▶

We use Visual Studio Community Edition (free). Make sure ASP.NET components are installed via the Visual Studio installer.

  1. Download and install Visual Studio Community Edition
  2. Run the installer and select the ASP.NET and web development workload
  3. Create a new project → select Blazor Web App
  4. Language: C# | Platform: Windows (or macOS) | Framework: Blazor
  5. Give your project a name and choose a save location
  6. Go with the basic default settings

Visual Studio includes a local emulator that sets up a listening port on your computer and handles all web communication locally. This means you won't need to use your Azure account during development — saving your $100 credit for deployment only.

Project Structure — File Roles

▶

Each file in a Blazor project has a specific purpose. Understanding them is essential before writing any code:

FilePurpose
app.cssGlobal stylesheet — styles that apply across the entire application
MainLayout.razorDefines consistent look and feel — universal headers, footers, and navigation
App.razorMain routing component — all user requests come here first, then get directed to the right page
Routes.razorCentralized routing declaration — establishes all endpoints for all Razor components
_Imports.razorShared import file — global imports available to all components (like a global using file)
Home.razorLanding page — what loads when a user visits the root URL. Uses @page "/"
Error.razorError handling page — properly catches and displays exceptions so they aren't silently missed
appsettings.jsonApplication configuration — connection strings, API URLs, and other settings
Program.csMain hosting module — sets up Blazor environment, registers services, configures rendering, builds and runs the app

Blazor pairs .razor files with .css files automatically: if a file named MainLayout.css exists in the same directory as MainLayout.razor, Blazor connects them. The .razor file holds HTML (and C# code), the .css file holds component-specific styles. This keeps styling scoped and organized.

Pages folder contains:

  • Counter.razor — demonstrates event-driven controls (buttons)
  • Error.razor — displays error conditions
  • Home.razor — your landing/home page
  • Weather.razor — demonstrates data binding with a grid/table

Blazor App Walkthrough

▶

The default Blazor app demonstrates three core web development patterns:

Home Page (Landing Page)

A typical modern site has a central display area and navigation. Navigation links to other pages, display area shows content. Links may also appear at top and/or bottom.

Counter Page (Event-Driven)

Demonstrates a button's Click event bound to a C# method. A paragraph updates dynamically when clicked — no server round-trip needed. Shows @page directive, render mode, dynamic variables, and event handling all in one file.

Weather Page (Data Binding)

A typical grid display using an HTML table. Each row is bound to a data element in a collection. Each column is bound to a specific field in a forecast object. C# code in the same file creates the data and stores it in a collection.

Standard Layout

Central display panel + navigation bar on left + about pane on top. This is a standard, industry-recognized way to organize a web application.

Key Blazor Features

▶

Hot Reload — make on-the-fly changes without restarting the app:

  1. Make changes to your code
  2. Click the Hot Reload button in Visual Studio
  3. Watch changes appear in the running app immediately

Page Routing — every page is defined by a human-readable route:

@page "/"          // Home page — root URL
@page "/counter"   // Counter page
@page "/weather"   // Weather page

// A page can have multiple routes
@page "/store"
@page "/shop"

// Routes can be dynamic and bound to variables
@page "/product/{id}"

// Or optional
@page "/item/{id?}"

Routes look like file paths but are just names — they're more readable than traditional URLs and can be bound to variables, making navigation flexible and intuitive for both developers and users.

Layouts

▶

Layouts define common content that wraps all your pages — headers, footers, navigation — so you write it once and it appears everywhere automatically.

  • Layouts are special components stored in the Layouts folder
  • One layout handles the body/content area of the page
  • One layout handles the navigation pane
  • The top and left panes you see in the default app are both layouts

Blazor splits a page into three sections using HTML elements:

<div> — Navigation Bar

The left sidebar with links to other pages. Defined in the navigation layout. Appears on every page automatically.

<div> — Top Pane

The header area at the top of the page. Typically contains app name, branding, or global controls.

<article> — Content Area

The main display area where each page's unique content renders. This is what changes as you navigate between pages.

Why Layouts Matter

Without layouts, you'd copy/paste navigation and header code into every single page. Layouts follow the DRY principle — Don't Repeat Yourself. Change the nav once, it updates everywhere.

Understanding how layouts work requires understanding how HTML works — which is exactly what comes next. The three-section structure (nav / top / content) is the foundation of virtually every modern web application you'll build.
