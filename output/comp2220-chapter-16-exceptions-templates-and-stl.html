Exceptions

▶

Exceptions indicate that something unexpected has occurred. They allow a program to deal with errors in a controlled, structured manner rather than crashing or producing bad output.

throw

Sends a signal that an error has occurred. Followed by an argument — the exception object or value being thrown.

try

Followed by a block {}. Wraps code that might throw an exception. If an exception is thrown, execution immediately exits the try block.

catch

Followed by a block {}. Detects and processes exceptions from the preceding try block. Takes a parameter matching the type of exception thrown.

Exception

An object or value that signals an error. Can be a primitive (int, string) or a class object. Carried from the throw point to the catch block.

// Function that throws an exception
int totalDays(int days, int weeks)
{
    if (days < 0 || days > 7)
        throw "invalid number of days";   // throw a C-string
    else
        return (7 * weeks + days);
}

// Calling code uses try/catch
try
{
    totDays = totalDays(days, weeks);
    cout << "Total days: " << totDays;
    // If exception thrown above, these lines don't execute
}
catch (const char *msg)    // catch matches thrown type
{
    cout << "Error: " << msg;
}

// Throwing different types
throw 42;                  // throw an int
throw string("bad input"); // throw a string
throw MyException("msg");  // throw a class object

Flow of Control & Nested try Blocks

▶

What happens when an exception is thrown:

  1. The throwing function terminates immediately (does not return)
  2. The try block is immediately exited
  3. Any objects created in the try block are destroyed (stack unwinding)
  4. C++ searches catch blocks in order for one matching the thrown type
  5. First matching catch block executes
  6. If NO matching catch is found, the program terminates

Once an exception is thrown, the program CANNOT return to the throw point. The function that threw it terminates. All intermediate functions between the throw and the matching catch also terminate — this is called "unwinding the stack." This is why proper cleanup (destructors, smart pointers) matters so much with exceptions.

// An exception is NOT caught if:
// 1. Thrown from outside a try block
// 2. No catch block matches the thrown type
// Result: program terminates (abort)

// Nested try blocks — exceptions can be passed up
try                           // outer try
{
    try                       // inner try
    {
        throw 42;
    }
    catch (int e)
    {
        cout << "Inner caught: " << e << endl;
        throw;    // re-throw same exception to outer level
    }
}
catch (int e)
{
    cout << "Outer caught: " << e << endl;
}

Exception Classes

▶

Custom exception classes allow you to carry detailed error information from the throw point to the catch block.

// Simple exception class — just signals an error
class NegativeBalance { };

// Exception class with data
class InvalidInput
{
private:
    string message;
    int badValue;
public:
    InvalidInput(string msg, int val)
        : message(msg), badValue(val) { }

    string getMessage() const { return message; }
    int getBadValue() const { return badValue; }
};

// Class with nested exception class
class BankAccount
{
public:
    // Nested exception class
    class InsufficientFunds
    {
    private:
        double amount;
    public:
        InsufficientFunds(double a) : amount(a) { }
        double getAmount() const { return amount; }
    };

    void withdraw(double amount)
    {
        if (amount > balance)
            throw InsufficientFunds(amount);  // throw object
        balance -= amount;
    }
private:
    double balance = 0.0;
};

// Catching class exceptions
try
{
    account.withdraw(1000.00);
}
catch (BankAccount::InsufficientFunds &e)
{
    cout << "Can't withdraw $" << e.getAmount() << endl;
}

Catch by reference (const ExceptionType&) not by value — avoids slicing and unnecessary copying. The thrown value doesn't need to be used in the catch block — you just need the type to match. A class can have multiple nested exception classes for different error conditions.

Function Templates

▶

A function template is a pattern for a function that works with many data types. Type parameters are left generic — the compiler generates specific code when the function is called.

// Template prefix: T is a type parameter
template <class T>
T times10(T num)
{
    return 10 * num;
}

// Compiler generates the right version automatically
int ival = 3;
double dval = 2.55;
cout << times10(ival);   // compiler creates times10(int) → 30
cout << times10(dval);   // compiler creates times10(double) → 25.5

// Multiple type parameters
template <class T1, class T2>
double mpg(T1 miles, T2 gallons)
{
    return miles / gallons;
}
cout << mpg(250, 10.5);    // T1=int, T2=double

// Overloaded templates — must have different parameter lists
template <class T>
T sumAll(T num) { return num; }

template <class T1, class T2>
T1 sumAll(T1 num1, T2 num2) { return num1 + num2; }

A function template generates NO actual code — it's purely a pattern. Code is only generated when the template function is called with specific types. All type parameters in the template prefix MUST be used in the function definition. When passing class objects to templates, make sure all operators used in the template (like < or +) are defined or overloaded in that class.

How to write a template: write the function normally for a specific type first, then replace the type name with T and add the template prefix.

Class Templates

▶

Classes can also be templates. Type information is supplied when defining objects — unlike function templates where the compiler deduces the type.

// Class template definition
template <class T>
class Grade
{
private:
    T score;
public:
    Grade(T s) : score(s) { }
    void setGrade(T s) { score = s; }
    T getGrade() const { return score; }
};

// Define objects — must specify the type explicitly
Grade<int>    testList[20];      // holds int scores
Grade<double> quizList[20];     // holds double scores
Grade<string> letterGrades[5];  // holds string grades

// Use like ordinary objects
testList[0].setGrade(95);
cout << testList[0].getGrade();

// Class template inheritance
template <class T>
class Rectangle { ... };

template <class T>
class Square : public Rectangle<T>   // use T everywhere base is named
{
public:
    Square(T side) : Rectangle<T>(side, side) { }
};

Unlike function templates where the compiler deduces T from arguments, class templates require explicit type specification at object definition: Grade<int> not just Grade. The type parameter T must be used everywhere the base class name appears in derived class templates.

The Standard Template Library (STL)

▶

The STL is a library of templates for frequently used data structures and algorithms. Two key components: containers and iterators.

Sequence Containers

Organize data sequentially like arrays. Include vector (dynamic array), deque (double-ended queue), and list (doubly-linked list).

Associative Containers

Use keys for fast data access. Include set (unique keys), multiset (duplicate keys), map (key-value pairs), and multimap.

Iterators

Generalization of pointers for accessing container elements. Types: forward (++), bidirectional (++ and --), random-access, input (cin), output (cout).

Algorithms

Function templates for common operations on containers. Requires #include <algorithm>. Examples: sort, find, count, reverse, min_element, max_element.

#include <vector>
#include <algorithm>

// vector — the most commonly used STL container
vector<int> nums = {5, 2, 8, 1, 9, 3};

// Iterator — like a pointer to an element
vector<int>::iterator it;
for (it = nums.begin(); it != nums.end(); it++)
    cout << *it << " ";   // dereference like a pointer

// Range-based for (simpler)
for (int n : nums)
    cout << n << " ";

// STL algorithms
sort(nums.begin(), nums.end());        // sort ascending
auto pos = find(nums.begin(), nums.end(), 8);  // find 8
int count = count(nums.begin(), nums.end(), 5); // count 5s
reverse(nums.begin(), nums.end());    // reverse order

The STL dramatically reduces the amount of code you need to write. Instead of implementing your own sorting, searching, and data structure code, you use well-tested, optimized templates. Containers + Iterators + Algorithms work together: algorithms operate on containers through iterators, making them independent of the specific container type.
