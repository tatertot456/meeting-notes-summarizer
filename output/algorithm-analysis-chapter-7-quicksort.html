Quicksort Description

▶

Quicksort is a divide-and-conquer sorting algorithm. It is one of the most widely used sorting algorithms in practice due to its excellent average-case performance and cache efficiency.

  • Divide: Partition the array around a pivot element
  • Conquer: Recursively sort the two subarrays on either side of the pivot
  • Combine: Nothing — the array is sorted in place

After partitioning, the pivot is in its final sorted position. All elements to its left are ≤ pivot, all elements to its right are ≥ pivot — forever.

The Quicksort Algorithm

▶ 
QUICKSORT(A, p, r)
if p < r
    q = PARTITION(A, p, r)
    QUICKSORT(A, p, q-1)
    QUICKSORT(A, q+1, r)

PARTITION(A, p, r)
x = A[r]          // pivot
i = p - 1
for j = p to r-1
    if A[j] ≤ x
        i = i + 1
        swap A[i] with A[j]
swap A[i+1] with A[r]
return i + 1

PARTITION maintains four regions:

  • A[p..i]: elements ≤ pivot
  • A[i+1..j-1]: elements > pivot
  • A[j..r-1]: not yet examined
  • A[r]: the pivot

Quicksort Correctness

▶

PARTITION's correctness is proved via a loop invariant on the four regions. At the start of each iteration:

  • All elements in A[p..i] are ≤ x (the pivot)
  • All elements in A[i+1..j-1] are > x
  • A[r] = x (pivot is untouched)

After PARTITION places the pivot in its final position q, QUICKSORT recursively sorts A[p..q-1] and A[q+1..r]. By induction, these subarrays become sorted — and since the pivot is already in place, the whole array is sorted.

Quicksort Efficiency

▶

Worst Case — O(n²)

Pivot is always the smallest or largest element (already-sorted input). Produces maximally unbalanced partitions: T(n) = T(n-1) + Θ(n) → Θ(n²).

Best Case — O(n log n)

Pivot always splits array exactly in half. T(n) = 2T(n/2) + Θ(n) → Θ(n log n). Same recurrence as merge sort.

Average Case — O(n log n)

Over all permutations of input, expected runtime is Θ(n log n). The constant is very small — faster in practice than merge sort.

In-Place

Sorts using only O(log n) stack space for recursion. No auxiliary array needed — unlike merge sort.

Randomized Quicksort

▶

Instead of always using A[r] as pivot, randomly select a pivot before partitioning:

RANDOMIZED-PARTITION(A, p, r)
i = RANDOM(p, r)
swap A[i] with A[r]
return PARTITION(A, p, r)

With randomized pivot selection, no particular input causes worst-case behavior. An adversary cannot construct a bad input because they don't know which element will be chosen as pivot. Expected runtime is Θ(n log n) regardless of input.

This is the version used in practice. The expected number of comparisons is exactly 2n·ln(n) ≈ 1.386·n log₂(n).
