What Is a Recurrence?

▶

A recurrence is an equation that describes a function in terms of its value on smaller inputs. Divide-and-conquer algorithms naturally produce recurrences.

T(n) = 2T(n/2) + Θ(n)   ← Merge Sort
T(n) = T(n-1) + Θ(n)    ← Insertion Sort (worst case)
T(n) = 7T(n/2) + Θ(n²)  ← Strassen's Algorithm

Conventions: T(1) = Θ(1) as the base case. We ignore floors and ceilings when they don't affect the asymptotic answer. We assume T(n) is constant for small n.

Strassen's Matrix Multiplication

▶

Naive matrix multiplication of two n×n matrices requires Θ(n³) operations. Strassen's algorithm reduces this by using 7 recursive multiplications instead of 8.

Naive Approach

T(n) = 8T(n/2) + Θ(n²) → Θ(n³). Three nested loops, 8 subproblems.

Strassen's Approach

T(n) = 7T(n/2) + Θ(n²) → Θ(n^log₂7) ≈ Θ(n^2.81). One fewer multiplication per level.

The key insight: additions are cheaper than multiplications. Strassen replaces one recursive multiplication with several additions — saving a factor of n^0.19 asymptotically, which matters enormously for large matrices.

Substitution Method

▶

The substitution method has two steps:

  • Guess the form of the solution (e.g., T(n) = O(n log n))
  • Prove the guess correct using mathematical induction

Example for T(n) = 2T(n/2) + n:

  • Guess: T(n) ≤ cn log n
  • Assume T(n/2) ≤ c(n/2)log(n/2)
  • Substitute: T(n) ≤ 2c(n/2)log(n/2) + n = cn·log(n/2) + n = cn·log n - cn + n ≤ cn log n when c ≥ 1

Guessing is the hard part. Good guesses come from recursion trees, similar recurrences you've seen before, or the Master Theorem.

Recursion Tree Method

▶

Draw the recursion as a tree where each node represents the cost of a subproblem:

  • Root cost = work at top level (non-recursive part)
  • Each level's total cost = work done across all subproblems at that depth
  • Sum all levels to get total cost

For T(n) = 2T(n/2) + n:

  • Level 0: n, Level 1: n/2 + n/2 = n, Level 2: n/4×4 = n...
  • There are log n levels, each costing n → total = n log n

The recursion tree is best used to generate a good guess for the substitution method — it gives intuition but isn't a formal proof on its own.

Master Theorem

▶

For recurrences of the form T(n) = aT(n/b) + f(n) where a ≥ 1, b > 1:

CaseConditionSolution
Case 1f(n) = O(n^(log_b a - ε))T(n) = Θ(n^log_b a)
Case 2f(n) = Θ(n^log_b a)T(n) = Θ(n^log_b a · log n)
Case 3f(n) = Ω(n^(log_b a + ε))T(n) = Θ(f(n))

Intuition: compare f(n) to n^log_b(a). If the recursive work dominates → Case 1. If they're equal → Case 2 (add a log factor). If f(n) dominates → Case 3.

Examples:

  • T(n) = 2T(n/2) + n: a=2, b=2, log₂2=1, f(n)=n=Θ(n¹) → Case 2 → Θ(n log n)
  • T(n) = 9T(n/3) + n: a=9, b=3, log₃9=2, f(n)=n=O(n^(2-1)) → Case 1 → Θ(n²)
  • T(n) = 7T(n/2) + n²: a=7, b=2, log₂7≈2.81, f(n)=n²=O(n^(2.81-ε)) → Case 1 → Θ(n^2.81)
